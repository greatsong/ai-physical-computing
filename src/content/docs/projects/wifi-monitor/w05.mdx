1. **추가할 기능 1** (필수): 한 줄 설명
2. **추가할 기능 2** (선택): 한 줄 설명
3. **추가할 기능 3** (도전): 한 줄 설명
4. **내일 발표에서 보여주고 싶은 것**

<Aside type="tip" title="기능 정의서가 중요한 이유">
무작정 코딩을 시작하면 방향을 잃기 쉽습니다. "뭘 만들지"를 먼저 정하고 AI에게 요청하면, 프롬프트도 구체적이 되고 결과도 좋아집니다. 기능 정의서는 여러분의 **내비게이션**이에요.
</Aside>

## 3단계: 자유 개발

기능 정의서에 적은 첫 번째 기능부터 시작하세요. AI에게 기능 정의서를 보여주고 "이 기능을 추가해줘"라고 요청하면 됩니다.

### 프롬프트 전략: 점진적 요청

코드가 길어지면 AI에게 전체를 한 번에 보내기 어렵습니다. **단계별로 나눠서 요청**하세요.

<Aside type="tip" title="점진적 요청이 중요한 이유">
한 번에 "전부 바꿔줘"라고 하면 AI가 엉뚱한 결과를 줄 수 있습니다. 대신 이렇게 해보세요:

1. **계획 요청** — "이 기능을 추가하려면 어떤 부분을 수정해야 해?"
2. **부분 수정** — "그러면 `make_page()` 함수만 수정한 코드를 보여줘"
3. **에러 해결** — "이 에러 메시지가 나왔어. 해결해줘"
4. **반복** — 작동하면 다음 기능으로!

이 사이클을 반복하면 코드를 안정적으로 발전시킬 수 있습니다.
</Aside>

### 한 기능이 완성되면

반드시 **실행해서 스마트폰으로 확인**하세요. Pico의 IP 주소로 접속해서 새 기능이 잘 작동하는지 직접 눈으로 보세요. 작동하면 다음 기능으로 넘어갑니다!

### 코드가 너무 길어졌을 때

5차시쯤 되면 코드가 꽤 길어져 있을 거예요. AI에게 전체 코드를 보내기 힘들다면:

- **수정할 함수만 보내세요.** "이건 내 코드의 `make_page()` 함수야. 여기에 경고 배너를 추가해줘" 처럼 범위를 좁혀서 요청하세요.
- **"이건 MicroPython이야"** 라고 꼭 알려주세요. AI가 일반 Python 코드를 줄 수 있거든요.
- 메모리 에러(`ENOMEM`)가 나면, HTML을 간소화하거나 CSS를 줄여보세요.

## 4단계: 갤러리 워크

개발 시간이 끝나면, 자리에서 일어나 친구들의 작품을 체험해보세요.

1. 스마트폰 브라우저를 열고, **다른 친구의 Pico IP 주소**로 접속합니다.
2. **5명 이상 방문**하세요.
3. 가장 인상적인 기능이 뭐였는지 기억해두세요.
4. 마음에 드는 기능이 있으면, 그 친구에게 "어떻게 만들었어?"라고 물어보세요.

<Aside type="note" title="영감 얻기">
다른 사람의 작품에서 영감을 받는 것도 개발자의 중요한 능력입니다. "베끼기"가 아니라 "배우기"예요!
</Aside>

## 5단계: 발표 준비

다음 시간에 마지막 발표가 있습니다. 아래 구성을 참고해서 준비하세요.

| 순서 | 시간 | 내용 |
|------|------|------|
| 시연 | 1분 | 스마트폰 화면을 보여주며 핵심 기능 시연 |
| 설명 | 30초 | "이 기능을 왜 만들었는지" 한 문장 |
| 과정 공유 | 30초 | 가장 잘된 프롬프트 or 가장 어려웠던 점 |
| 질문/피드백 | 30초 | 친구들에게 질문 1개 받기 |

<Aside type="caution" title="코드 저장 필수!">
오늘 수업이 끝나면 코드를 꼭 저장해두세요. 다음 시간 발표에서 그대로 사용합니다. Thonny에서 Pico에 저장되어 있는지 확인하세요.
</Aside>

## 셀프 체크리스트

오늘 활동을 돌아보세요.

- [ ] 기능 정의서를 작성하고, 최소 1개 기능을 구현했다
- [ ] AI에게 점진적으로 요청하는 방법을 사용했다 (계획 → 수정 → 디버깅)
- [ ] 구현한 기능이 스마트폰에서 정상 작동한다
- [ ] 친구 5명 이상의 Pico에 접속해서 작품을 체험했다
- [ ] 다음 시간 발표에서 보여줄 내용을 정했다
- [ ] 코드를 Pico에 저장했다
