## 발표 전 체크리스트

발표 시작 전에 아래 4가지를 확인하세요.

1. **Pico 전원 연결** → Wi-Fi 연결 확인
2. **스마트폰에서 대시보드 접속** → 기능 작동 확인
3. **시연 순서 메모** — 어떤 화면을 먼저 보여줄지 정해두기
4. **핵심 프롬프트 1~2개 기억해두기** — 과정 공유할 때 사용

<Aside type="tip" title="서버가 안 될 때">
Pico USB를 뽑았다 다시 꽂고, Thonny에서 main.py를 다시 실행하세요. 그래도 안 되면 이전 차시의 안정 버전으로 되돌리세요. 발표에서는 "이런 기능을 만들려고 했다"는 과정 설명도 가치가 있습니다.
</Aside>

## 코드 제출 방법

발표가 끝나면 코드를 제출하세요.

1. **Thonny**에서 Pico의 `main.py`를 엽니다.
2. **파일 → 다른 이름으로 저장 → "이 컴퓨터"** 를 선택합니다.
3. 바탕화면 등에 저장한 뒤, **Google Classroom에 제출**합니다.

<Aside type="caution" title="저장 위치 주의!">
"이 컴퓨터"를 선택하지 않으면 Pico에만 저장됩니다. 꼭 **"이 컴퓨터"**를 눌러야 PC에 파일이 생겨요.
</Aside>

## 프로젝트 전체 셀프 체크리스트

6주간의 프로젝트를 돌아보며 체크해보세요.

- [ ] Pico에 MicroPython 웹서버를 만들고 스마트폰에서 접속할 수 있다
- [ ] RSSI 값을 실시간으로 측정하고 대시보드에 표시할 수 있다
- [ ] 차트(꺾은선, 막대 등)로 데이터를 시각화할 수 있다
- [ ] 측정 위치별 신호 세기를 지도 형태로 표현할 수 있다
- [ ] AI에게 구체적인 프롬프트를 작성하여 원하는 코드를 얻을 수 있다
- [ ] AI가 준 코드가 안 될 때, 에러 메시지를 활용해서 디버깅할 수 있다
- [ ] "왜 이 기능을 만들었는지"를 설명할 수 있다
- [ ] AI의 한계를 알고, 직접 판단이 필요한 순간을 구분할 수 있다
- [ ] 실제 설치를 위한 엔지니어링 제약조건을 생각해볼 수 있다
- [ ] 코드를 PC에 저장하고 제출할 수 있다
