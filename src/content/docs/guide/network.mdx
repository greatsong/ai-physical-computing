# 네트워크 이해하기

> **핵심 질문:** "Pico가 측정한 온도는 어떻게 내 스마트폰 화면에 나타나는 걸까?"

<p style="color: var(--sl-color-gray-3);">

## 이 장에서 배우는 것

이 장을 마치면:
- LAN, IP 주소, SSID 같은 용어를 **아파트 비유로 설명**할 수 있다
- 센서 데이터가 Pico를 거쳐 브라우저까지 도착하는 **전체 경로를 설명**할 수 있다
- STA 모드와 AP 모드의 **차이를 한 문장으로 설명**할 수 있다

:::tip[센서 없이 읽을 수 있어요!]
이 장은 **개념 이해**에 집중합니다. Pico나 센서가 없어도 괜찮아요. 읽고 나면 앞으로의 실습이 훨씬 쉬워집니다.
:::

## 1단계: 이미 알고 있는 것에 이름 붙이기

### 왜 이걸 배우나요?

네트워크를 처음 배운다고요? 사실은 **이미 알고 있어요.**

- 학교 Wi-Fi에 폰을 연결해서 유튜브를 본 적 있죠?
- 무선 프린터로 과제를 출력한 적 있죠?
- 선생님이 노트북 화면을 전자칠판에 미러링하는 거 봤죠?

이 모든 것이 **네트워크** 위에서 일어나는 일입니다. "모르는 것을 배우는 게" 아니에요. **여러분이 매일 하는 행동에 정확한 이름을 붙이는 것**입니다.

:::note[잠깐, 생각해보기 🤔]
"그러면 Wi-Fi랑 LAN이랑 같은 건가요?"

아파트 비유로 설명하면 — **LAN은 아파트 단지 자체**고, **Wi-Fi는 단지에 들어가는 방법** 중 하나예요. 랜선(유선)으로 들어가도 같은 단지, Wi-Fi(무선)로 들어가도 같은 단지!
:::

## 2단계: 우리 교실의 네트워크

### 왜 이걸 배우나요?

교실에는 생각보다 많은 기기가 네트워크에 연결되어 있어요. 이 구조를 이해하면 **Pico가 만든 웹 페이지를 누가 볼 수 있는지** 바로 알 수 있습니다.

### 교실을 위에서 내려다보면

아래 다이어그램에서 **빛나는 점**을 따라가 보세요. 데이터가 공유기를 거쳐 각 기기로 흘러가는 모습입니다!

<ClassroomNetwork />

### 💡 핵심 규칙: 같은 LAN = 서로 직접 대화 가능

이 규칙이 왜 중요할까요?

| 상황 | 가능 여부 | 이유 |
|------|:---------:|------|
| 같은 Wi-Fi에 연결된 노트북 → 프린터 출력 | ✅ | 같은 LAN (같은 단지) |
| 같은 Wi-Fi에 연결된 스마트폰 → Pico 대시보드 접속 | ✅ | 같은 LAN (같은 단지) |
| 학교 Wi-Fi 기기 → 집 Wi-Fi 기기 | ❌ | 다른 LAN (다른 단지) |
| Pico 대시보드를 학교 밖에서 접속 | ❌ | 다른 LAN — 인터넷 필요 |

:::tip[이게 바로 우리 프로젝트의 핵심!]
Pico가 만든 웹 페이지를 **같은 Wi-Fi에 연결된 기기**라면 누구나 볼 수 있어요. 노트북, 스마트폰, 전자칠판 모두요! 바로 이 "같은 LAN" 규칙 덕분입니다.
:::

## 4단계: Pico의 두 얼굴 — STA vs AP

### 왜 이걸 배우나요?

지금까지는 Pico가 학교 Wi-Fi에 접속하는 경우만 봤어요. 하지만 Pico는 **반대로 자기가 Wi-Fi를 만들 수도** 있습니다! 이 두 가지 모드를 이해하면 프로젝트마다 왜 다른 방식을 쓰는지 자연스럽게 이해됩니다.

### STA 모드 vs AP 모드

**쉽게 말하면**:
- **STA** = "남의 Wi-Fi에 **들어가는** 것" (손님)
- **AP** = "내가 Wi-Fi를 **만드는** 것" (사장님)

### STA 모드 예시 — 학교 Wi-Fi에 합류

```
학교 공유기 (AP, School_WiFi)
    │
    ├── 📺 전자칠판     192.168.1.10
    ├── 💻 노트북       192.168.1.20
    ├── 📱 스마트폰     192.168.1.30
    └── 🔧 Pico (STA)  192.168.1.100  ← 웹 서버 운영!
                │
                🌡️ 센서 (전선 연결)
```

브라우저에 `192.168.1.100` 입력 → Pico의 대시보드가 뜬다!

### AP 모드 예시 — Pico가 직접 Wi-Fi를 만듦

```
🔧 Pico (AP, "DragonRadar")  192.168.4.1
    │
    ├── 📱 내 폰        192.168.4.2
    └── 📱 짝꿍 폰      192.168.4.3

    ❌ 인터넷 없음 (독립 네트워크!)
    ✅ 학교 Wi-Fi 없어도 동작!
```

:::note[왜 AP 모드가 필요할까?]
운동장이나 복도처럼 **학교 Wi-Fi가 안 닿는 곳**에서도 Pico를 사용하고 싶을 때! Pico가 직접 Wi-Fi를 만들면 공유기 없이도 동작합니다.
:::

### ⚠️ 자주 하는 실수

**실수 1: Pico는 2.4GHz Wi-Fi만 됩니다!**

요즘 공유기는 5GHz도 지원하지만, Pico의 Wi-Fi 칩은 **2.4GHz만** 가능합니다.

```
공유기 설정에서 보이는 Wi-Fi:
  School_WiFi_5G     ← ❌ Pico 연결 불가!
  School_WiFi        ← ✅ 이걸 사용!
```

**실수 2: STA 모드에서 IP 주소 확인 안 하기**

Pico가 Wi-Fi에 접속하면 공유기가 IP를 줍니다. 이 IP는 **매번 바뀔 수 있어요!**
코드 실행 후 Shell에 출력되는 IP를 반드시 확인하세요.

```python
# Pico 코드에서 IP 확인
ip = wlan.ifconfig()[0]
print(f"내 IP: {ip}")  # 이걸 꼭 확인!
```

**실수 3: 폰이 학교 Wi-Fi가 아닌 데이터(LTE/5G)에 연결**

Pico 대시보드가 안 열리면, **스마트폰이 학교 Wi-Fi에 연결되어 있는지** 확인하세요!
모바일 데이터를 끄고, Wi-Fi에 연결해야 합니다.

  <div class="check-a"><strong>STA</strong>는 남이 만든 Wi-Fi에 접속(카페 손님), <strong>AP</strong>는 자기가 Wi-Fi를 만듦(카페 사장님). STA는 인터넷 가능, AP는 독립 네트워크.</div>
  <div class="check-q">Q. 코스 A의 센서 프로젝트에서 Pico는 어떤 모드를 사용하나요?</div>
  <div class="check-a"><strong>STA 모드</strong>입니다. 학교 Wi-Fi에 접속해서, 같은 네트워크의 브라우저들에게 센서 데이터를 제공합니다.</div>

  <div class="check-a">Pico는 <strong>2.4GHz만</strong> 지원합니다. 공유기 설정에서 2.4GHz 네트워크 이름(SSID)을 확인하고 그걸로 연결하세요.</div>

## 핵심 용어 정리

| 용어 | 뜻 | 비유 |
|------|-----|------|
| **LAN** | 같은 공유기에 연결된 기기들의 모임 | 같은 아파트 단지 |
| **AP** | Wi-Fi 신호를 쏘는 쪽 (공유기, 또는 AP 모드 Pico) | 카페 사장님 |
| **STA** | Wi-Fi 신호를 받는 쪽 (폰, 노트북, STA 모드 Pico) | 카페 손님 |
| **IP 주소** | 네트워크 안에서 각 기기의 고유 주소 (192.168.1.xxx) | 아파트 호수 |
| **SSID** | Wi-Fi 이름 ("School_WiFi", "DragonRadar" 등) | 아파트 단지 이름 |
| **웹 서버** | IP로 접속하면 웹 페이지를 보내주는 프로그램 | 카페 사장님 |
| **JSON** | 기계가 읽기 쉬운 데이터 형식 `{"temp":26.5}` | 정해진 양식의 편지 |
| **fetch()** | 브라우저가 서버에 데이터를 요청하는 함수 | 카페에서 "하나 더 주세요!" |
