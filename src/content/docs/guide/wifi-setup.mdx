**핵심:** 같은 Wi-Fi에 연결된 기기끼리는 서로 접속할 수 있어요. Pico가 웹서버를 돌리면, 노트북 브라우저에서 Pico의 IP 주소를 입력해 대시보드를 볼 수 있습니다!

<Aside type="caution" title="Pico 2 WH는 2.4GHz 전용!">
Pico 2 WH의 Wi-Fi 칩은 **2.4GHz만 지원**합니다. 5GHz 전용 네트워크에는 연결할 수 없어요. 공유기 설정에서 **2.4GHz 대역의 SSID**를 확인하세요.
</Aside>

## wifi_config.py 만들기

Pico가 Wi-Fi에 접속하려면, Wi-Fi 이름(SSID)과 비밀번호가 필요합니다. 이 정보를 `wifi_config.py` 파일에 저장해두면, 다른 코드에서 불러와 쓸 수 있어요.

```python
# wifi_config.py
# Pico에 저장해두면 다른 코드에서 import해서 쓸 수 있어요
WIFI_SSID = "Wi-Fi이름"
WIFI_PASSWORD = "비밀번호"
```

<Aside type="tip" title="Thonny에서 Pico에 파일 저장하는 방법">
1. Thonny 메뉴 → 파일 → 다른 이름으로 저장
2. "어디에 저장할까요?" → **"Raspberry Pi Pico"** 선택
3. 파일 이름 입력 (`wifi_config.py`)
4. 저장 완료!
</Aside>

## 환경별 Wi-Fi 설정

<Tabs>
  <TabItem label="학교에서">
    학교 Wi-Fi는 **비밀번호가 설정**되어 있어요. 선생님이 SSID와 비밀번호를 알려줄 거예요. Pico는 **2.4GHz만 지원**하므로 5GHz 전용 네트워크는 사용할 수 없습니다.

    ```python
    # wifi_config.py — 학교 버전
    WIFI_SSID = "학교Wi-Fi이름"        # 선생님이 알려준 SSID
    WIFI_PASSWORD = "학교Wi-Fi비밀번호"  # 선생님이 알려준 비밀번호
    ```

    <Aside type="tip" title="IP를 고정하고 싶다면">
    DHCP가 자동으로 IP를 바꾸기 때문에, 매번 IP가 달라질 수 있어요. 아래 **정적 IP 설정** 섹션을 참고하세요.

    ```python
    # wifi_config.py — 학교 정적 IP 버전
    WIFI_SSID = "학교Wi-Fi이름"
    WIFI_PASSWORD = "학교Wi-Fi비밀번호"

    STATIC_IP = "172.30.6.10"    # 다른 기기와 안 겹치는 번호
    SUBNET    = "255.255.0.0"    # 학교 네트워크는 보통 /16 대역
    GATEWAY   = "172.30.0.1"     # 기본 게이트웨이
    DNS       = "8.8.8.8"
    ```
    </Aside>

    **학교 IP 대역 확인법:** Thonny 셸에서 연결 후 `wlan.ifconfig()`를 실행하면 자동 할당된 IP, 서브넷, 게이트웨이를 확인할 수 있어요.
  </TabItem>
  <TabItem label="집에서">
    집에 가져가서 실습하려면 `wifi_config.py`만 바꾸면 돼요! 집 공유기의 **2.4GHz SSID**를 확인하세요 (5GHz와 이름이 다를 수 있어요).

    **1단계: 기본 연결 (이것만 하면 바로 동작!)**

    ```python
    # wifi_config.py — 집 버전
    WIFI_SSID = "우리집Wi-Fi이름"
    WIFI_PASSWORD = "우리집비밀번호"
    ```

    이렇게만 바꾸고 실행하면 Pico가 자동으로 IP를 받아요. Thonny 셸에 뜨는 IP로 접속하면 끝!

    **2단계 (선택): 내 네트워크 정보 확인하기**

    정적 IP를 설정하려면 먼저 집 네트워크 정보를 알아야 해요. **가장 쉬운 방법**은 Pico를 먼저 연결한 뒤 Thonny 셸에서 확인하는 거예요:

    ```python
    # Thonny 셸에서 실행 (Pico가 Wi-Fi 연결된 상태에서)
    >>> wlan.ifconfig()
    ('192.168.0.15', '255.255.255.0', '192.168.0.1', '168.126.63.1')
    #   ↑ 내 IP          ↑ 서브넷           ↑ 게이트웨이       ↑ DNS
    ```

    <Aside type="caution" title="Pico에 접속이 안 될 때">
    노트북/스마트폰이 Pico와 **다른 네트워크**에 연결되어 있으면 접속할 수 없어요. 둘 다 **같은 Wi-Fi**에 연결되어 있는지 확인하세요.
    </Aside>

    공유기마다 IP 대역이 다르므로 **반드시 직접 확인**하세요:

    | 공유기 | 흔한 대역 | 서브넷 |
    |--------|----------|--------|
    | iptime | `192.168.0.x` | `255.255.255.0` |
    | KT/SKT/LG | `172.30.x.x` 또는 `192.168.x.x` | 다양함 |
    | TP-Link, ASUS | `192.168.0.x` 또는 `192.168.1.x` | `255.255.255.0` |

    Pico 없이 PC/스마트폰에서 확인하는 방법:
    - **Windows:** `Win+R` → `cmd` → `ipconfig` 입력 → **IPv4 주소**, **서브넷 마스크**, **기본 게이트웨이** 확인
    - **Mac:** Option 키를 누른 채 메뉴바 Wi-Fi 아이콘 클릭 → **IP 주소**, **서브넷 마스크**, **라우터** 표시
    - **iPhone:** 설정 → Wi-Fi → 연결된 네트워크 옆 (i) 버튼 → **IP 주소**, **서브넷 마스크**, **라우터**
    - **Android:** 설정 → Wi-Fi → 연결된 네트워크 길게 누르기 → 네트워크 세부정보

    **3단계 (선택): 정적 IP 설정**

    2단계에서 확인한 값을 기반으로 설정합니다:

    ```python
    # wifi_config.py — 집 정적 IP 버전
    # (아래 숫자들은 예시! 반드시 2단계에서 확인한 값으로 바꾸세요)
    WIFI_SSID = "우리집Wi-Fi이름"
    WIFI_PASSWORD = "우리집비밀번호"

    STATIC_IP = "192.168.0.200"    # 내 IP의 마지막 숫자만 200~250으로 변경
    SUBNET    = "255.255.255.0"    # 서브넷은 그대로 복사
    GATEWAY   = "192.168.0.1"      # 게이트웨이도 그대로 복사
    DNS       = "8.8.8.8"          # 구글 DNS (고정값)
    ```

    <Aside type="tip" title="STATIC_IP는 어떻게 정하나요?">
    `wlan.ifconfig()`로 확인한 **내 IP의 마지막 숫자만 바꾸면** 돼요. 예를 들어 자동 할당이 `192.168.0.15`였다면, 정적 IP는 `192.168.0.200`처럼 **200~250 사이** 높은 번호를 쓰세요. 이 범위는 보통 다른 기기가 쓰지 않아요.
    </Aside>

    `main.py`는 수정할 필요 없어요! `wifi_config.py`만 교체하면 어디서든 동일하게 작동합니다.
  </TabItem>
</Tabs>

- **네트워크 주소** (앞 3자리): 우리 아파트 **단지** 이름
- **기기 번호** (마지막 1자리): 우리 집 **호수**
- **서브넷 마스크**: "앞 몇 자리까지가 단지 이름인지" 알려주는 규칙
- **게이트웨이**: 아파트 단지의 **정문** (공유기 주소)

:::note[같은 단지 = 같은 네트워크]
IP 주소의 네트워크 부분(앞 3자리)이 같으면 "같은 네트워크"예요. 같은 네트워크에 있는 기기끼리만 직접 접속할 수 있습니다!
:::

## 정적 IP 설정하기 (선택)

기본적으로 Pico는 Wi-Fi에 연결할 때마다 **자동으로 IP를 받습니다** (DHCP). 이 IP는 매번 바뀔 수 있어요.

**항상 같은 주소로 접속하고 싶다면** 정적 IP를 설정하세요.

### 1단계: 현재 네트워크 정보 확인

Pico를 Wi-Fi에 연결한 뒤 Thonny 셸에서:

```python
>>> wlan.ifconfig()
('192.168.0.15', '255.255.255.0', '192.168.0.1', '168.126.63.1')
#   ↑ 내 IP        ↑ 서브넷         ↑ 게이트웨이     ↑ DNS
```

### 2단계: wifi_config.py에 정적 IP 추가

```python
# wifi_config.py — 정적 IP 버전
WIFI_SSID = "Wi-Fi이름"
WIFI_PASSWORD = "비밀번호"

STATIC_IP = "192.168.0.200"    # 마지막 숫자만 200~250으로 변경
SUBNET    = "255.255.255.0"    # 1단계에서 확인한 서브넷 그대로
GATEWAY   = "192.168.0.1"      # 1단계에서 확인한 게이트웨이 그대로
DNS       = "8.8.8.8"          # 구글 DNS (고정값)
```

### 3단계: main.py에서 정적 IP 적용

Wi-Fi 연결 직후에 한 줄을 추가합니다:

```python
wlan.connect(WIFI_SSID, WIFI_PASSWORD)
while not wlan.isconnected():
    time.sleep(1)

# ↓ 이 줄을 추가하면 IP 고정!
wlan.ifconfig((STATIC_IP, SUBNET, GATEWAY, DNS))

my_ip = wlan.ifconfig()[0]
print(f"IP 고정 완료: {my_ip}")
```

<Aside type="tip" title="정적 IP는 선택사항!">
DHCP 자동 할당으로도 충분합니다. 처음에는 자동 IP로 시작하고, 매번 IP가 바뀌는 게 불편해질 때 정적 IP를 설정하세요.
</Aside>

## 접속 체크리스트

브라우저에서 Pico에 접속이 안 될 때, 이 3가지를 순서대로 확인하세요:

<div class="net-check">
  <div class="check-q">1. 내 노트북/스마트폰이 Pico와 같은 Wi-Fi에 연결되어 있나요?</div>
  <div class="check-a">모바일 데이터(LTE/5G)가 아닌 **Wi-Fi**에 연결되어 있어야 해요. 그리고 Pico와 **같은** Wi-Fi여야 합니다.</div>
  <div class="check-q">2. 주소가 http:// 인가요? (https:// 가 아닌지 확인!)</div>
  <div class="check-a">브라우저가 자동으로 <code>https://</code>로 바꾸는 경우가 있어요. 반드시 <code>http://192.168.x.x</code>로 접속하세요.</div>
  <div class="check-q">3. Pico의 IP 주소가 정확한가요?</div>
  <div class="check-a">Thonny 셸에 표시된 IP 주소를 다시 한번 확인하세요. 숫자 하나라도 다르면 접속이 안 됩니다.</div>
</div>

## 흔한 문제 & 해결

| 증상 | 해결 방법 |
|------|----------|
| Wi-Fi 연결 실패 | `wifi_config.py`의 SSID/비밀번호 확인, **2.4GHz** SSID인지 확인 |
| IP가 매번 바뀜 | 정적 IP 설정 (위 가이드 참조) |
| 브라우저 접속 불가 | 같은 Wi-Fi 확인, `http://` 직접 입력, IP 주소 재확인 |
| `EADDRINUSE` 에러 | Pico USB를 뺐다 다시 꽂고 코드 재실행 |
| 집에서 연결 안 됨 | `wifi_config.py`를 집 SSID/비밀번호로 교체, 2.4GHz 확인 |
