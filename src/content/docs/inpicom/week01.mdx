## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. WH 모델은 헤더 핀이 미리 납땜되어 있어 바로 사용할 수 있습니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. 양쪽 끝에 4핀 커넥터가 달려 있습니다. |
| MQ2 가스 센서 | 공기 중의 가연성 가스(LPG, 알코올, 연기 등)를 감지하는 아날로그 센서입니다. 내부에 소형 히터가 있어 가스 분자에 반응합니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

MQ2 가스 센서를 Grove 케이블로 Shield의 **A0 포트**(아날로그)에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[예열 시간 안내]
MQ2 센서는 내부에 히터가 있어서 전원을 켠 후 **1~2분** 예열이 필요합니다. 연결 직후 값이 불안정해도 정상이니 기다려주세요.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH를 Grove Shield 위에 올려 꽂습니다.** Pico의 핀과 Shield의 소켓을 맞춰 끝까지 눌러주세요.
2. **Micro USB 케이블로 컴퓨터와 연결합니다.** Pico의 Micro USB 포트에 케이블을 꽂으면 전원이 들어옵니다.
3. **MQ2 센서를 Grove 케이블로 A0 포트에 꽂습니다.** Shield에 "A0"이라고 적힌 포트를 찾아 연결하세요.

:::tip
USB를 먼저 연결해서 MQ2 예열을 시작하세요. 코드를 작성하는 동안 예열이 끝납니다.
:::

### Step 2. 첫 코드 작성 (10분)

1. 컴퓨터에서 **Thonny**를 열어주세요.
2. 아래 [기본 코드](#기본-코드)를 Thonny 편집기에 입력합니다.
3. 상단의 **실행(Run)** 버튼을 클릭합니다.
4. 하단 **Shell** 창에 가스 센서 값이 1초마다 출력되는지 확인하세요.

:::note[생각해보기]
센서 가까이에 입김을 '후~' 불어보세요. 값이 어떻게 변하나요? 손소독제(에탄올)를 가까이 대면 어떤가요?
:::

### Step 3. 플로터로 실시간 그래프 보기 (20분)

숫자만 보면 변화가 느껴지지 않습니다. Thonny의 플로터 기능으로 실시간 그래프를 그려봅시다.

1. Thonny 메뉴에서 **View > Plotter**를 선택합니다. 화면 하단에 그래프 영역이 나타납니다.
2. [도전 코드](#도전-코드-이동-평균-필터)를 입력하고 실행합니다. `print()`로 출력한 숫자가 자동으로 실시간 그래프가 됩니다.
3. 다음 실험을 하면서 그래프 변화를 관찰하세요:
   - 입김 불기 전/후 값 비교
   - 손소독제를 센서 가까이 대기
   - 센서에서 거리별(1cm / 5cm / 10cm) 값 비교
   - 교실 창문 열기 전/후 비교

### Step 4. 토론하기 (10분)

다음 질문을 모둠에서 이야기해보세요.

1. **"센서 값이 가장 크게 변한 실험은?"** 왜 그 물질에 더 강하게 반응했을까요?
2. **"같은 실험을 했는데 모둠별로 값이 다르다면?"** 환경(환기, 위치, 예열 시간)이 어떤 영향을 줄까요?
3. **"이 센서 데이터를 활용하면 어떤 제품을 만들 수 있을까요?"** 구체적으로 제안해봅시다.

### 기본 코드

```python
# === MQ2 가스 센서 기본 읽기 ===
# 센서의 아날로그 값을 읽어서 화면에 출력하는 코드

from machine import ADC, Pin
import time

# --- 센서 설정 ---
# Grove A0 포트 = GP26 (Pico의 ADC 채널 0)
gas_sensor = ADC(Pin(26))

# --- 반복해서 값 읽기 ---
while True:
    value = gas_sensor.read_u16()  # 0~65535 사이 값 읽기
    print(value)                   # 값 출력
    time.sleep(1)                  # 1초 대기
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 4 | `from machine import ADC, Pin` | ADC는 Analog-to-Digital Converter의 약자입니다. 아날로그 신호(연속적인 전압)를 디지털 숫자로 바꿔주는 장치입니다. |
| 9 | `ADC(Pin(26))` | GP26 핀에 연결된 아날로그 센서를 읽겠다는 뜻입니다. Grove Shield의 A0 포트가 GP26에 연결되어 있습니다. |
| 13 | `read_u16()` | 센서 값을 16비트 부호 없는 정수(0~65535)로 읽습니다. 가스 농도가 낮으면 작은 값, 높으면 큰 값이 나옵니다. |
| 14 | `print(value)` | 숫자만 출력하면 Thonny Plotter가 이 값을 자동으로 그래프에 표시합니다. |

### 도전 코드: 이동 평균 필터

센서 값의 노이즈를 줄여서 진짜 변화만 보려면, 여러 번 읽어서 평균을 내는 방법을 사용합니다.

```python
# === MQ2 가스 센서 + 이동 평균 필터 ===
# 10번 읽어서 평균을 구해 노이즈를 줄이는 코드

from machine import ADC, Pin
import time

# --- 센서 설정 ---
gas_sensor = ADC(Pin(26))

# --- 이동 평균 필터 함수 ---
def read_average(sensor, count=10):
    """count번 읽어서 평균을 구하는 함수"""
    total = 0
    for i in range(count):
        total += sensor.read_u16()
        time.sleep_ms(10)  # 10ms 간격으로 읽기
    return total // count  # 정수 나눗셈

# --- 메인 루프 ---
while True:
    avg_value = read_average(gas_sensor, 10)
    print(avg_value)  # Thonny 플로터용 출력
    time.sleep(0.5)   # 0.5초마다 측정
```

:::note[코드 읽어보기]
`read_average()` 함수는 센서를 10번 읽어서 평균을 구합니다. 시험을 한 번만 보면 운이 작용하지만, 열 번 보면 실력이 나오는 것과 같은 원리입니다. `time.sleep_ms(10)`은 밀리초(1/1000초) 단위로 대기합니다. `//`는 정수 나눗셈으로 소수점 이하를 버립니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

공기 중의 가스 분자가
**→** MQ2 센서 내부 히터에 닿아 **전기 저항**을 바꾸고
**→** 저항 변화가 **전압 변화**로 이어지고
**→** ADC가 그 전압을 0~65535 사이의 **디지털 숫자**로 바꾸고
**→** 코드가 그 숫자를 **실시간 그래프**로 시각화했습니다.

<Mermaid client:load chart={`
flowchart LR
    A["가스 분자\n(아날로그)"] --> B["MQ2 센서\n저항 변화"]
    B --> C["전압 신호\n0~3.3V"]
    C --> D["ADC 변환\n0~65535"]
    D --> E["print 출력\n숫자/그래프"]
`} caption="아날로그 신호가 디지털 숫자로 바뀌는 과정" />

눈에 보이지 않는 가스를 숫자로, 그 숫자를 그래프로 바꿨습니다. 가스 누출 탐지기, 공기청정기의 자동 작동, 산업 현장의 유해가스 경보 시스템 — 모두 이 원리의 확장입니다.
