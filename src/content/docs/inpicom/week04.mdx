| Micro USB 케이블 | Pico와 컴퓨터를 연결합니다. |

### 연결 방법

오늘은 **두 개의 I2C 장치**를 연결합니다. Grove Shield에 I2C 포트가 여러 개 있으니 하나씩 꽂으면 됩니다.
- **BME280** → **I2C1 포트**
- **OLED** → **I2C0 포트**

:::tip[I2C의 장점]
I2C 장치는 **주소**로 구분됩니다. BME280(0x76)과 OLED(0x3C)의 주소가 다르기 때문에, 같은 I2C 버스에서 충돌 없이 동시에 동작합니다.
:::

:::note[드라이버 파일 확인]
BME280과 OLED를 사용하려면 Pico에 **bme280.py**와 **ssd1306.py** 드라이버 파일이 저장되어 있어야 합니다. Thonny에서 MicroPython 장치의 파일 목록을 확인하세요.
:::

### Step 1. 연결하기 (5분)

1. **BME280 센서를 I2C1 포트에 연결합니다.**
2. **OLED 디스플레이를 I2C0 포트에 연결합니다.**

4. **I2C 스캔으로 장치 확인** — Thonny에서 다음 코드를 실행하세요:

```python
from machine import I2C, Pin
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
devices = i2c.scan()
print("발견된 장치:", [hex(d) for d in devices])
```

0x3c(OLED)와 0x76 또는 0x77(BME280)이 모두 나오면 성공입니다.

### 기본 코드

```python
# === BME280 기상 데이터 읽기 ===
# 온도, 습도, 기압을 읽어서 출력

from machine import I2C, Pin
import bme280
import time

# --- I2C 설정 ---
i2c = I2C(0, sda=Pin(8), scl=Pin(9))

# --- BME280 센서 초기화 ---
bme = bme280.BME280(i2c=i2c)

# --- 반복 측정 ---
while True:
    temp = bme.values[0]      # 온도 (예: "23.4C")
    humidity = bme.values[2]  # 습도 (예: "45.2%")
    pressure = bme.values[1]  # 기압 (예: "1013.2hPa")

    print(f"온도: {temp}  습도: {humidity}  기압: {pressure}")
    time.sleep(1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 4 | `from machine import I2C, Pin` | I2C는 Inter-Integrated Circuit의 약자로, 선 2개(SDA, SCL)로 여러 장치와 통신하는 방식입니다. |
| 9 | `I2C(0, sda=Pin(8), scl=Pin(9))` | I2C 버스 0번을 사용합니다. SDA(데이터 선)는 GP8, SCL(클럭 선)은 GP9에 연결되어 있습니다. |
| 12 | `bme280.BME280(i2c=i2c)` | BME280 드라이버에 I2C 버스를 전달하여 센서를 초기화합니다. 드라이버가 주소(0x76)를 자동으로 찾습니다. |
| 16 | `bme.values[0]` | BME280 드라이버가 원시 데이터를 사람이 읽을 수 있는 문자열로 변환합니다. 인덱스 0은 온도, 1은 기압, 2는 습도입니다. |

### 도전 코드: OLED 기상 대시보드

BME280에서 읽은 데이터를 OLED 화면에 실시간으로 표시합니다.

```python
# === 미니 기상 스테이션 ===
# BME280 데이터를 OLED에 실시간 표시

from machine import I2C, Pin
import ssd1306
import bme280
import time

# --- I2C 설정 (센서 + OLED 공유) ---
i2c = I2C(0, sda=Pin(8), scl=Pin(9))

# --- 장치 초기화 ---
bme = bme280.BME280(i2c=i2c)
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

# --- 메인 루프 ---
while True:
    temp = bme.values[0]
    pressure = bme.values[1]
    humidity = bme.values[2]

    # 화면 갱신
    oled.fill(0)  # 화면 지우기

    # 제목
    oled.text("= Weather =", 20, 0)

    # 데이터 표시
    oled.text(f"Temp: {temp}", 0, 20)
    oled.text(f"Humi: {humidity}", 0, 35)
    oled.text(f"Pres: {pressure}", 0, 50)

    oled.show()  # 화면에 반영
    time.sleep(1)
```

:::note[코드 읽어보기]
`oled.fill(0)`은 화면 전체를 검정(0)으로 채워 이전 내용을 지웁니다. `oled.text(문자열, x, y)`는 해당 좌표에 텍스트를 그립니다. 기본 폰트는 8x8 픽셀이라 한 줄에 16글자, 총 8줄까지 표시 가능합니다. `oled.show()`는 LED의 `np.write()`와 같은 역할로, 이 줄이 없으면 화면에 표시되지 않습니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

Pico가 I2C 버스를 통해 BME280에게 "온도 알려줘"라고 **주소(0x76)**를 불러 요청하고
**→** BME280이 온도·습도·기압 데이터를 **디지털 패킷**으로 응답하고
**→** 코드가 그 데이터를 사람이 읽을 수 있는 **문자열**로 변환하고
**→** 같은 I2C 버스를 통해 OLED **주소(0x3C)**로 화면 데이터를 전송했습니다.

<Mermaid client:load chart={`
flowchart LR
    P["Pico\n(마스터)"] -->|"주소 0x76\n데이터 요청"| B["BME280\n온도·습도·기압"]
    B -->|"측정 데이터\n응답"| P
    P -->|"주소 0x3C\n화면 데이터"| O["OLED\n디스플레이"]
    style P fill:#f9f,stroke:#333
`} caption="I2C 버스: 선 2개(SDA+SCL)로 여러 장치가 주소로 대화" />

선 2개만으로 여러 장치가 대화하는 I2C 통신. 스마트폰 안의 가속도 센서, 자이로스코프, 기압 센서가 모두 I2C로 연결되어 있습니다. 여러분은 방금 그 통신 방식을 직접 사용한 것입니다.

#### ADC vs I2C 비교표

| 항목 | ADC (1차시 MQ2) | I2C (4차시 BME280) |
|------|-----------------|-------------------|
| 통신 방식 | 전압 → 숫자 변환 | 주소 기반 디지털 통신 |
| 한 번에 읽는 데이터 | 1개 (가스 농도) | 여러 개 (온도, 습도, 기압) |
| 하나의 포트에 연결 가능한 장치 | 1개 | 여러 개 (주소로 구분) |
| 사용 예시 | 가스, 빛, 소리 센서 | 온습도, 기압, OLED, 가속도 센서 |

:::note[두 통신 방식의 적합한 용도]
ADC는 단순한 아날로그 센서(전압이 연속적으로 변하는 것)에 적합합니다. I2C는 복잡한 디지털 센서(여러 데이터를 정확하게 전달해야 하는 것)에 적합합니다. 실제 제품에서는 두 방식을 함께 사용합니다.
:::
