## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. 3개 필요합니다. |
| BME280 센서 | 온도, 습도, 기압을 측정하는 I2C 센서입니다 (주소: `0x76`). |
| SCD30 CO2 센서 | CO2 농도, 온도, 습도를 측정하는 I2C 센서입니다 (주소: `0x61`). |
| OLED 디스플레이 (0.96") | 128x64 픽셀의 I2C 디스플레이입니다 (주소: `0x3C`). |
| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

BME280을 **I2C1 포트**에, SCD30을 **I2C0 포트**에 연결하세요. OLED는 BME280과 같은 **I2C1 버스**를 공유합니다(주소가 다르므로 충돌 없음).

:::tip[I2C 다중 장치]
I2C 통신은 **주소**로 장치를 구분합니다. BME280(`0x76`)과 OLED(`0x3C`)는 주소가 다르므로 같은 I2C 버스(I2C1)에 함께 연결할 수 있습니다. 하나의 케이블 라인에 여러 장치를 연결하는 것이 I2C의 장점입니다.
:::

:::note[3개 장치 연결]
Shield의 I2C 포트를 2개 사용합니다. BME280과 OLED가 I2C1을 공유하고, SCD30은 별도로 I2C0을 사용합니다. I2C 허브가 있으면 한 포트에 여러 케이블을 연결할 수 있습니다.
:::

## 활동 가이드

### Step 1. 연결 및 I2C 스캔 (5분)

1. **BME280 → I2C1**, **SCD30 → I2C0**에 연결합니다. OLED는 I2C 허브를 통해 I2C1에 함께 연결합니다.
2. **Micro USB 케이블로 컴퓨터와 연결합니다.**
3. Thonny에서 다음 코드를 실행하여 장치가 잘 인식되는지 확인하세요.

```python
from machine import I2C, Pin
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))
print("I2C0:", [hex(d) for d in i2c0.scan()])
print("I2C1:", [hex(d) for d in i2c1.scan()])
```

정상이면 I2C0에 `0x61`(SCD30), I2C1에 `0x3c`(OLED)와 `0x76`(BME280)이 나타납니다.

:::tip
장치가 보이지 않으면 케이블 연결을 확인하세요. Grove 케이블이 끝까지 밀어져 있는지, 포트가 올바른지 확인합니다.
:::

### Step 2. 6개 데이터 동시 읽기 (15분)

1. 아래 [기본 코드](#기본-코드)를 Thonny에 입력하고 실행합니다.
2. Shell 창에서 BME280의 3개 데이터와 SCD30의 3개 데이터를 확인하세요.
3. **교차 검증**: 두 센서의 온도 차이가 2°C 이내인지 확인합니다.

### Step 3. OLED 대시보드 만들기 (20분)

[도전 코드](#도전-코드-oled-환경-대시보드)를 실행하여 OLED 화면에 모든 데이터를 표시하는 대시보드를 만들어보세요.

1. **도전 코드**를 Thonny에 입력하고 실행합니다.
2. OLED 화면에 등급, 점수, CO2, 온도, 습도, 기압이 모두 표시되는지 확인합니다.
3. 다음 실험을 해보세요:
   - **창문을 닫고 5분 기다리기** → CO2가 올라가면서 점수가 내려가는지 관찰
   - **창문을 열어 환기하기** → 점수가 다시 올라가는지 관찰
   - **센서에 입김 불기** → CO2와 습도가 급격히 변하는지 관찰

### 기본 코드

```python
from machine import I2C, Pin
import bme280
import scd30
import time

# --- 하드웨어 설정 ---
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))   # Grove I2C0 (SCD30)
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))   # Grove I2C1 (BME280)

# --- I2C 스캔 ---
print("I2C0:", [hex(d) for d in i2c0.scan()])
print("I2C1:", [hex(d) for d in i2c1.scan()])

# --- 센서 초기화 ---
bme = bme280.BME280(i2c=i2c1)
co2_sensor = scd30.SCD30(i2c0, addr=0x61)
co2_sensor.start_continous_measurement()

print("=== 다중 센서 데이터 수집 ===")
time.sleep(2)   # SCD30 안정화

while True:
    bme_temp = bme.values[0]     # "23.4C"
    bme_hum = bme.values[2]      # "45.2%"
    bme_pres = bme.values[1]     # "1013.2hPa"

    if co2_sensor.data_ready():
        co2, scd_temp, scd_hum = co2_sensor.read_measurement()

        print(f"[BME280] T:{bme_temp} H:{bme_hum} P:{bme_pres}")
        print(f"[SCD30]  CO2:{co2:.0f}ppm T:{scd_temp:.1f}C H:{scd_hum:.1f}%")

        temp_diff = abs(float(bme_temp[:-1]) - scd_temp)
        print(f"[교차검증] 온도차: {temp_diff:.1f}C", end="")
        if temp_diff < 2:
            print(" ✓ 정상")
        else:
            print(" ⚠ 센서 점검 필요")
        print("---")

    time.sleep(2)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 7~8 | `I2C(0, ...)` / `I2C(1, ...)` | 두 개의 I2C 버스를 사용합니다. I2C0에 SCD30, I2C1에 BME280과 OLED가 연결됩니다. |
| 16 | `scd30.SCD30(i2c0, addr=0x61)` | SCD30 센서를 I2C0 버스에서 주소 `0x61`로 초기화합니다. |
| 17 | `start_continous_measurement()` | SCD30에 연속 측정을 시작하라는 명령입니다. 약 2초 간격으로 데이터가 준비됩니다. |
| 27 | `co2_sensor.data_ready()` | SCD30 데이터가 준비되었는지 확인합니다. 아직 준비되지 않았으면 건너뜁니다. |
| 32 | `abs(float(bme_temp[:-1]) - scd_temp)` | 두 센서의 온도 차이 절댓값을 구합니다. 2°C 이내면 정상입니다. |

### 도전 코드: OLED 환경 대시보드

6개 데이터를 종합 판정하고 OLED에 대시보드 형태로 표시합니다.

```python
from machine import I2C, Pin
import ssd1306
import bme280
import scd30
import time

# --- 하드웨어 설정 ---
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))   # SCD30
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))   # BME280 + OLED

bme = bme280.BME280(i2c=i2c1)
co2_s = scd30.SCD30(i2c0, addr=0x61)
co2_s.start_continous_measurement()
oled = ssd1306.SSD1306_I2C(128, 64, i2c1)

time.sleep(2)

# --- 환경 종합 판정 ---
def judge_environment(co2, temp, hum):
    """CO2, 온도, 습도로 100점 만점 환경 점수 산출"""
    score = 0

    # CO2 점수 (40점 만점)
    if co2 < 800:
        score += 40
    elif co2 < 1000:
        score += 30
    elif co2 < 1500:
        score += 15

    # 온도 점수 (30점 만점)
    if 20 <= temp <= 26:
        score += 30
    elif 18 <= temp <= 28:
        score += 20
    else:
        score += 5

    # 습도 점수 (30점 만점)
    if 40 <= hum <= 60:
        score += 30
    elif 30 <= hum <= 70:
        score += 20
    else:
        score += 5

    # 등급 판정
    if score >= 80:
        return score, "Excellent!"
    elif score >= 60:
        return score, "Good"
    elif score >= 40:
        return score, "Fair"
    else:
        return score, "Poor-Ventilate!"

# --- 메인 ---
print("=== 교실 환경 대시보드 ===")

while True:
    if co2_s.data_ready():
        co2, scd_t, scd_h = co2_s.read_measurement()
        bme_t = float(bme.values[0][:-1])
        bme_p = float(bme.values[1][:-3])
        bme_h = float(bme.values[2][:-1])

        # 센서 퓨전: 두 센서의 평균
        avg_temp = (bme_t + scd_t) / 2
        avg_hum = (bme_h + scd_h) / 2

        score, grade = judge_environment(co2, avg_temp, avg_hum)

        # OLED 표시
        oled.fill(0)
        oled.text(f"{grade}", 0, 0)
        oled.text(f"{score}pt", 96, 0)
        oled.hline(0, 10, 128, 1)

        oled.text(f"CO2:{co2:.0f}ppm", 0, 14)
        oled.text(f"T:{avg_temp:.1f}C", 0, 26)
        oled.text(f"H:{avg_hum:.0f}%", 64, 26)
        oled.text(f"P:{bme_p:.0f}hPa", 0, 38)

        # CO2 막대 그래프
        oled.hline(0, 50, 128, 1)
        bar_w = min(int(co2 / 2000 * 120), 120)
        oled.rect(4, 54, 120, 8, 1)
        oled.fill_rect(4, 54, bar_w, 8, 1)

        oled.show()

        print(f"[{grade} {score}pt] CO2:{co2:.0f} T:{avg_temp:.1f} H:{avg_hum:.0f}")

    time.sleep(2)
```

:::note[코드 읽어보기]
`judge_environment` 함수는 CO2(40점), 온도(30점), 습도(30점)로 총 100점 만점의 환경 점수를 산출합니다. 80점 이상이면 Excellent, 60점 이상이면 Good, 40점 이상이면 Fair, 그 이하는 환기가 필요하다는 뜻입니다. 점수 기준을 여러분의 교실 환경에 맞게 조절해보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

BME280이 **온도, 습도, 기압** 3개를, SCD30이 **CO2, 온도, 습도** 3개를
**→** 총 **6개의 데이터**를 동시에 수집하고
**→** 겹치는 온도·습도는 **교차 검증 + 평균**으로 신뢰도를 높이고
**→** CO2, 온도, 습도를 각각 점수로 바꿔 **100점 만점 종합 판정**을 하고
**→** 128x64 픽셀 화면에 모든 정보를 **한눈에 보이게** 배치했습니다.

구글 네스트 온도 조절기, 공기질 모니터, 스마트 빌딩 관제 시스템 — 모두 여러 센서의 데이터를 통합하고 판정하는 **같은 구조**입니다. 여러분은 방금 IoT 환경 모니터링 시스템의 핵심을 직접 만든 것입니다.

## 마무리

### 오늘 배운 것

- **I2C 다중 장치**: 주소가 다르면 하나의 I2C 버스에 여러 장치를 연결할 수 있습니다.
- **센서 퓨전**: 여러 센서의 데이터를 조합하여 신뢰도를 높이는 방법을 배웠습니다.
- **종합 판정 로직**: CO2, 온도, 습도를 각각 점수화하고 합산하여 환경 등급을 매기는 코드를 작성했습니다.
- **UI 설계**: 128x64 픽셀의 제한된 화면에 정보를 효과적으로 배치하는 방법을 경험했습니다.

### 돌아보기

:::note[9~11차시 심화 통합 프로젝트를 되돌아보며]
**9차시**: BME280 온도 → LED Strip 색상 (매핑)
**10차시**: BME280 기압 → 고도 추정 (변환 공식)
**11차시**: BME280 + SCD30 → 종합 판정 (센서 퓨전)

하나의 센서에서 **어떤 데이터를 어떻게 해석하느냐**에 따라 완전히 다른 프로젝트가 만들어집니다. 이것이 피지컬 컴퓨팅의 매력이자 **문제 정의의 힘**입니다.
