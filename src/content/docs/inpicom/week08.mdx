### 연결 방법

PIR을 **D18 포트**에, MQ2를 **A0 포트**에, RGB LED Stick을 **D16 포트**에 연결하세요.

:::tip[3개 부품 동시 연결]
오늘은 입력 2개(PIR + MQ2) + 출력 1개(LED)를 동시에 사용합니다. 각 부품이 서로 다른 종류의 포트(디지털, 아날로그, 디지털)를 사용하므로 충돌 없이 연결할 수 있습니다.
:::

:::note[MQ2 예열 시간]
MQ2 가스 센서는 전원을 켠 후 1~2분간 예열이 필요합니다. 예열 중에는 값이 불안정할 수 있으니 잠시 기다린 후 테스트하세요.
:::

## 활동 가이드

### Step 1. 연결 + 상태 머신 이해하기 (10분)

1. **PIR을 D18, MQ2를 A0, LED Stick을 D16에 연결합니다.**
2. Micro USB 케이블로 컴퓨터와 연결합니다.
3. 아래 상태 다이어그램을 보며 시스템 동작을 이해합니다.

**상태 머신**이란 시스템이 가질 수 있는 상태를 정의하고, 어떤 조건에서 다른 상태로 전환되는지 설계하는 방법입니다.

우리 경보기의 상태는 세 가지입니다:
- **SLEEP** (절전): LED 1개만 약한 파랑. 가스 측정 안 함. 배터리 절약.
- **ACTIVE** (측정 중): LED 초록~노랑. 가스를 계속 측정.
- **ALERT** (경보): LED 빨강 깜빡임. 가스가 위험 수준.

:::note[게임 캐릭터와 같은 개념]
게임 캐릭터를 생각해보세요. 대기 -> 이동 -> 공격 -> 피격. 각 상태에서 특정 입력이 오면 다음 상태로 바뀝니다. 상태 머신은 같은 개념을 코드로 구현하는 것입니다.
:::

### Step 2. 상태 전환 테스트 (10분)

1. Thonny를 열고 아래 **기본 코드**를 입력합니다.
2. 실행하면 LED 1개가 약한 파란색으로 켜집니다 (SLEEP 상태).
3. **손을 흔들어보세요.** LED가 초록색으로 바뀌면 ACTIVE 상태입니다.
4. **손소독제를 센서 가까이 대보세요.** LED가 빨간색으로 바뀌면 ALERT 상태입니다.

### Step 3. 절전형 경보기 완성 (20분)

1. **도전 코드**를 입력합니다.
2. 다음 시나리오를 순서대로 테스트하세요:
   - 가만히 있기 -> SLEEP 유지 (약한 파랑 1개)
   - 손 흔들기 -> ACTIVE 전환 (초록)
   - 10초간 가만히 있기 -> 자동 SLEEP 복귀 (파랑 1개)
   - 다시 손 흔들기 -> ACTIVE
   - 손소독제 가까이 대기 -> ALERT (빨강 깜빡임)
   - 손소독제 치우기 -> ACTIVE 복귀

3. `TIMEOUT_MS` 값을 바꿔보며 적절한 절전 전환 시간을 찾아보세요.

### Step 4. 배터리 수명 토론 (10분)

모둠에서 다음을 이야기해보세요.

1. **"항상 측정"과 "절전형"의 배터리 수명 차이**를 추정해보세요.
2. **"SLEEP에서 PIR만 체크하면 왜 절전이 될까?"** LED를 끄고 가스 센서를 안 읽는 것이 어떤 차이를 만드나요?
3. **"이 패턴을 다른 곳에 적용한다면?"** 예: 자판기, 신호등, 스마트 가전 등

### 기본 코드

```python
# === 상태 머신 기본 구조 ===
# 세 가지 상태 전환 패턴 학습

from machine import Pin, ADC
from neopixel import NeoPixel
import time

# --- 하드웨어 ---
pir = Pin(18, Pin.IN)
gas = ADC(Pin(26))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 상태 정의 ---
STATE_SLEEP  = "SLEEP"
STATE_ACTIVE = "ACTIVE"
STATE_ALERT  = "ALERT"

current_state = STATE_SLEEP

# --- LED 표시 함수 ---
def show_sleep():
    """절전: LED 1개만 파란색 (나머지 끔)"""
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np[0] = (0, 0, 20)  # 아주 약한 파랑
    np.write()

def show_active():
    """활성: 초록색"""
    for i in range(NUM_LEDS):
        np[i] = (0, 50, 0)
    np.write()

def show_alert():
    """경보: 빨강"""
    for i in range(NUM_LEDS):
        np[i] = (255, 0, 0)
    np.write()

# --- 메인 루프 ---
print("=== 상태 머신 테스트 ===")
show_sleep()

while True:
    if current_state == STATE_SLEEP:
        if pir.value() == 1:
            print("SLEEP -> ACTIVE (움직임 감지)")
            current_state = STATE_ACTIVE
            show_active()

    elif current_state == STATE_ACTIVE:
        value = gas.read_u16()
        if value > 30000:  # 가스 감지
            print(f"ACTIVE -> ALERT (가스: {value})")
            current_state = STATE_ALERT
            show_alert()

    elif current_state == STATE_ALERT:
        value = gas.read_u16()
        if value < 20000:  # 가스 해소
            print("ALERT -> ACTIVE (가스 해소)")
            current_state = STATE_ACTIVE
            show_active()

    time.sleep(0.5)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 15-17 | `STATE_SLEEP`, `STATE_ACTIVE`, `STATE_ALERT` | 시스템이 가질 수 있는 세 가지 상태를 문자열로 정의합니다. |
| 19 | `current_state = STATE_SLEEP` | 시작 상태를 SLEEP(절전)으로 설정합니다. |
| 41 | `if current_state == STATE_SLEEP:` | 현재 상태를 확인하고, 그 상태에서의 동작과 전환 조건을 실행합니다. |
| 42 | `if pir.value() == 1:` | SLEEP 상태에서 움직임이 감지되면 ACTIVE로 전환합니다. |
| 48 | `if value > 30000:` | ACTIVE 상태에서 가스가 높으면 ALERT로 전환합니다. 이 임계값은 환경에 따라 조절하세요. |

### 도전 코드: 절전형 경보기 완성

```python
# === 절전형 가스 경보기 ===
# PIR 감지 시에만 MQ2 측정, 타임아웃 시 절전 복귀

from machine import Pin, ADC
from neopixel import NeoPixel
import time

# --- 하드웨어 ---
pir = Pin(18, Pin.IN)
gas = ADC(Pin(26))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 설정 ---
GAS_WARN = 20000
GAS_ALERT = 35000
TIMEOUT_MS = 10000   # 10초 무감지 시 절전

# --- 상태 ---
STATE_SLEEP  = 0
STATE_ACTIVE = 1
STATE_ALERT  = 2
state = STATE_SLEEP
last_motion = 0

# --- 이동 평균 (노이즈 제거) ---
def read_avg(sensor, n=10):
    """센서 값을 n번 읽어 평균을 구합니다"""
    total = 0
    for _ in range(n):
        total += sensor.read_u16()
        time.sleep_ms(10)
    return total // n

# --- LED 함수들 ---
def led_sleep():
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np[0] = (0, 0, 15)  # 상태 표시: 약한 파랑 1개
    np.write()

def led_active(gas_val):
    """가스 농도에 따라 초록~노랑 그라데이션"""
    ratio = min(gas_val / GAS_ALERT, 1.0)
    r = int(255 * ratio)
    g = int(255 * (1 - ratio * 0.5))
    for i in range(NUM_LEDS):
        np[i] = (r, g, 0)
    np.write()

def led_alert_blink():
    """빨강 깜빡임"""
    for i in range(NUM_LEDS):
        np[i] = (255, 0, 0)
    np.write()
    time.sleep(0.15)
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)
    np.write()
    time.sleep(0.15)

# --- 메인 루프 ---
print("=== 절전형 가스 경보기 ===")
led_sleep()

while True:
    motion = pir.value()
    now = time.ticks_ms()

    if motion:
        last_motion = now

    # === 상태별 처리 ===
    if state == STATE_SLEEP:
        if motion:
            state = STATE_ACTIVE
            print("SLEEP -> ACTIVE (활성화!)")
        else:
            time.sleep(0.5)  # 절전: 느리게 체크
            continue

    if state == STATE_ACTIVE:
        gas_val = read_avg(gas)
        led_active(gas_val)

        if gas_val >= GAS_ALERT:
            state = STATE_ALERT
            print(f"ACTIVE -> ALERT (가스: {gas_val})")

        # 타임아웃: 일정 시간 움직임 없으면 절전
        elapsed = time.ticks_diff(now, last_motion)
        if elapsed > TIMEOUT_MS:
            state = STATE_SLEEP
            led_sleep()
            print("ACTIVE -> SLEEP (절전 복귀)")

    elif state == STATE_ALERT:
        led_alert_blink()
        gas_val = read_avg(gas)

        if gas_val < GAS_WARN:
            state = STATE_ACTIVE
            print(f"ALERT -> ACTIVE (경보 해제, 가스: {gas_val})")

    print(f"[{['SLEEP','ACTIVE','ALERT'][state]}] motion={motion}")
    time.sleep(0.3)
```

:::note[코드 읽어보기]
`continue`는 루프의 나머지를 건너뛰고 처음으로 돌아가는 명령입니다. SLEEP 상태에서는 가스 측정이 필요 없으니 `continue`로 바로 다음 루프로 갑니다. `read_avg()` 함수는 센서 값을 10번 읽어 평균을 구해 **노이즈를 줄입니다**. `led_active()`는 가스 농도에 비례해 초록에서 노랑으로 LED 색상이 서서히 변합니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

**3개의 부품**이 하나의 시스템으로 통합되었습니다:
- PIR이 **"사람이 있나?"**를 판단하고
- MQ2가 **"가스가 위험한가?"**를 측정하고
- LED가 **"지금 상태가 뭔가?"**를 눈으로 보여주고
- **상태 머신**이 이 모든 것을 **논리적으로 연결**했습니다.

센서 하나를 읽는 것에서 시작해서, 여러 센서의 데이터를 조합하고, 상태에 따라 다르게 동작하는 **시스템**을 만든 것입니다. 실제 스마트 빌딩의 화재 경보기, 공장의 안전 시스템, 자동차의 에어백 시스템 모두 이 상태 머신 패턴으로 설계됩니다.

## 마무리

### 오늘 배운 것

- **상태 머신 설계**: SLEEP, ACTIVE, ALERT 세 가지 상태와 전환 조건을 설계했습니다. 이 패턴은 자판기, 신호등, 게임 등 어디서나 사용됩니다.
- **3개 부품 통합**: PIR(디지털 입력) + MQ2(아날로그 입력) + LED(디지털 출력)를 하나의 시스템으로 조합했습니다.
- **절전 로직**: 사람이 없을 때 센서 측정을 멈추고 LED를 최소화하여 배터리를 절약하는 방법을 구현했습니다.
- **이동 평균**: 센서 노이즈를 줄이기 위해 여러 번 읽어 평균을 구하는 `read_avg()` 기법을 사용했습니다.

### 상태 머신 패턴 정리

이 패턴을 기억하세요. 앞으로 어떤 시스템을 만들 때도 적용할 수 있습니다:

1. **상태를 정의한다** (예: SLEEP, ACTIVE, ALERT)
2. **각 상태에서의 동작을 정한다** (예: LED 표시, 센서 읽기 여부)
3. **상태 전환 조건을 정한다** (예: PIR 감지, 가스 임계값, 타임아웃)
