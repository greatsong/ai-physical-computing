## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. |
| Grove 케이블 (4핀) x 2 | 센서와 LED를 각각 연결하는 케이블이 **2개** 필요합니다. |
| MQ2 가스 센서 | 가연성 가스를 감지하는 아날로그 센서입니다. A0 포트에 연결합니다. |
| Grove RGB LED Stick | NeoPixel 방식의 10개짜리 LED 바입니다. D16 포트에 연결합니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결합니다. |

### 연결 방법

오늘은 **두 개의 부품**을 동시에 연결합니다.
- **MQ2 가스 센서** → **A0 포트** (아날로그)
- **RGB LED Stick** → **D16 포트** (디지털)

:::tip[예열 먼저!]
MQ2 센서를 A0에 먼저 꽂고 USB를 연결하세요. LED 코드를 준비하는 동안 예열이 완료됩니다.
:::

:::note[포트 종류가 다릅니다]
A0은 아날로그 포트(센서용), D16은 디지털 포트(LED용)입니다. 센서와 출력 장치가 서로 다른 포트를 사용하므로 충돌 없이 동시에 동작합니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **MQ2 센서를 A0 포트에 연결합니다.** 1차시에서 사용한 것과 같은 포트입니다.
2. **RGB LED Stick을 D16 포트에 연결합니다.** 2차시에서 사용한 것과 같은 포트입니다.
3. **USB를 연결하여 예열을 시작합니다.** MQ2는 1~2분 예열이 필요합니다.

:::tip
예열하는 동안 임계값 개념을 이해하는 시간으로 활용하세요.
:::

### Step 2. 첫 코드 — 검출 시스템 v1 (15분)

1. Thonny를 열고 아래 [기본 코드](#기본-코드)를 입력합니다.
2. **실행(Run)** 을 클릭합니다.
3. LED가 **초록색**으로 켜지면 성공입니다.
4. 센서에 입김을 불거나 손소독제를 가까이 대서 **색이 바뀌는지** 확인하세요.

:::note[임계값 튜닝]
Shell에 출력되는 센서 값을 30초간 관찰하세요. 이것이 '평상시 기준값'입니다. THRESHOLD_WARN을 `기준값 + 5000`, THRESHOLD_ALERT을 `기준값 + 15000`으로 조정하면 환경에 맞는 검출기가 됩니다.
:::

### Step 3. 실습 — 경보 애니메이션 추가 (20분)

[도전 코드](#도전-코드-경보-애니메이션)를 실행하여 상태별 애니메이션을 확인해보세요.
- **안전**: 은은한 초록 (고정)
- **주의**: 노란색 숨쉬기 (서서히 밝아졌다 어두워짐)
- **경보**: 빨간색 깜빡임 (급박한 느낌)

임계값을 자기 환경에 맞게 조정하고, 입김/손소독제로 3단계 전환을 테스트해보세요.

### Step 4. 토론하기 (10분)

다음 질문을 모둠에서 이야기해보세요.

1. **"임계값을 어떻게 설정했나요?"** 모둠별로 다르다면 왜 다를까요? 환기, 위치, 예열 시간의 영향을 이야기해봅시다.

3. **"더 나은 검출기를 만들려면 뭐가 필요할까요?"** Wi-Fi 알림, 소리 경보, 다중 센서 등 아이디어를 제안해봅시다.

### 기본 코드

```python
# === 전자담배 검출 시스템 v1 ===
# MQ2 센서 값에 따라 LED 색상이 바뀌는 검출기

from machine import ADC, Pin
from neopixel import NeoPixel
import time

# --- 하드웨어 설정 ---
gas_sensor = ADC(Pin(26))        # A0: MQ2 가스 센서
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS) # D16: RGB LED Stick

# --- 임계값 설정 ---
# 먼저 평상시 센서 값을 확인한 뒤 조정하세요
THRESHOLD_WARN = 20000   # 이 이상이면 주의 (노랑)
THRESHOLD_ALERT = 35000  # 이 이상이면 경보 (빨강)

# --- 이동 평균 함수 (1차시 복습) ---
def read_average(sensor, count=10):
    total = 0
    for i in range(count):
        total += sensor.read_u16()
        time.sleep_ms(10)
    return total // count

# --- LED 전체를 한 색으로 설정 ---
def set_all_leds(r, g, b):
    for i in range(NUM_LEDS):
        np[i] = (r, g, b)
    np.write()

# --- 메인 루프 ---
print("=== 검출 시스템 가동 ===")
print(f"주의 임계값: {THRESHOLD_WARN}")
print(f"경보 임계값: {THRESHOLD_ALERT}")

while True:
    value = read_average(gas_sensor)

    if value >= THRESHOLD_ALERT:
        set_all_leds(255, 0, 0)      # 빨강
        status = "경보"
    elif value >= THRESHOLD_WARN:
        set_all_leds(255, 255, 0)    # 노랑
        status = "주의"
    else:
        set_all_leds(0, 255, 0)      # 초록
        status = "안전"

    print(f"{status} | 센서값: {value}")
    time.sleep(0.5)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 9 | `ADC(Pin(26))` | 1차시에서 배운 MQ2 센서 설정입니다. A0 포트(GP26)에서 아날로그 값을 읽습니다. |
| 11 | `NeoPixel(Pin(16), NUM_LEDS)` | 2차시에서 배운 LED 설정입니다. D16 포트(GP16)에 연결된 10개 LED를 제어합니다. |
| 15-16 | `THRESHOLD_WARN / ALERT` | 임계값(threshold)은 상태를 구분하는 기준선입니다. 환경에 따라 조정이 필요합니다. |
| 36-42 | `if / elif / else` | 센서 값을 임계값과 비교하여 세 가지 상태로 분류합니다. 위에서부터 순서대로 검사합니다. |

### 도전 코드: 경보 애니메이션

안전할 때는 차분하게, 위험할 때는 급박하게. 상태별로 다른 애니메이션을 적용합니다.

```python
# === 전자담배 검출 시스템 v2 (경보 애니메이션) ===
# 안전=초록 고정, 주의=노랑 숨쉬기, 경보=빨강 깜빡임

from machine import ADC, Pin
from neopixel import NeoPixel
import time

# --- 하드웨어 설정 ---
gas_sensor = ADC(Pin(26))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 임계값 설정 ---
THRESHOLD_WARN = 20000
THRESHOLD_ALERT = 35000

# --- 함수 모음 ---
def read_average(sensor, count=10):
    total = 0
    for i in range(count):
        total += sensor.read_u16()
        time.sleep_ms(10)
    return total // count

def set_all_leds(r, g, b):
    for i in range(NUM_LEDS):
        np[i] = (r, g, b)
    np.write()

def blink_red():
    """빨강 깜빡임 - 위험 경보 효과"""
    set_all_leds(255, 0, 0)
    time.sleep(0.15)
    set_all_leds(0, 0, 0)
    time.sleep(0.15)

def breathe_yellow(step):
    """노랑 숨쉬기 - step(0~19)에 따라 밝기 변화"""
    if step < 10:
        brightness = step * 25
    else:
        brightness = (19 - step) * 25
    set_all_leds(brightness, brightness, 0)

# --- 메인 루프 ---
print("=== 검출 시스템 v2 ===")
breath_step = 0

while True:
    value = read_average(gas_sensor)

    if value >= THRESHOLD_ALERT:
        blink_red()
        status = "경보"
    elif value >= THRESHOLD_WARN:
        breathe_yellow(breath_step)
        breath_step = (breath_step + 1) % 20
        status = "주의"
        time.sleep(0.1)
    else:
        set_all_leds(0, 50, 0)  # 은은한 초록
        status = "안전"
        breath_step = 0
        time.sleep(0.5)

    print(f"{status} | 값: {value}")
```

:::note[코드 읽어보기]
`blink_red()` 함수는 빨강으로 켜고 0.15초 대기, 끄고 0.15초 대기합니다. 약 0.3초마다 깜빡이는 급박한 효과입니다. `breathe_yellow(step)`은 step이 0~9일 때 점점 밝아지고 10~19일 때 점점 어두워지는 삼각파 패턴입니다. `% 20`은 나머지 연산으로, 19 다음에 다시 0으로 돌아가게 합니다. 안전 상태의 `(0, 50, 0)`은 눈이 편한 어두운 초록입니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

1차시에서 만든 **입력**(센서 읽기)과 2차시에서 만든 **출력**(LED 제어)이 합쳐져서, 가운데에 **판단**(if/elif/else)이 들어간 **완성된 시스템**이 되었습니다.

이 **입력→처리→출력** 구조는 모든 자동화 시스템의 기본입니다. 자동문(적외선 센서 → 판단 → 모터), 에어컨(온도 센서 → 판단 → 압축기), 자율주행차(카메라/라이다 → AI 판단 → 조향) — 규모만 다를 뿐 구조는 동일합니다. 여러분은 방금 그 구조를 직접 설계하고 구현한 것입니다.
