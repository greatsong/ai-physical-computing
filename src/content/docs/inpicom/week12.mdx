## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) x4 | 센서와 Shield를 연결하는 케이블입니다. 오늘은 4개의 부품을 연결하므로 4개가 필요합니다. |
| Grove MQ2 가스 센서 | 가연성 가스(연기, LPG, 알코올 등)를 감지하는 아날로그 센서입니다. ADC 방식으로 값을 읽습니다. |
| Grove SCD30 CO2 센서 | 공기 중 CO2 농도, 온도, 습도를 동시에 측정하는 I2C 센서입니다. |
| Grove OLED 디스플레이 0.96" | 128x64 픽셀의 소형 화면입니다. I2C 방식으로 통신합니다. |
| Grove RGB LED Stick | NeoPixel LED 10개가 일렬로 배치된 모듈입니다. 디지털 핀으로 제어합니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

오늘은 **4개의 부품**을 동시에 연결합니다. 포트 종류가 서로 다르므로 주의하세요.

| 부품 | 포트 | 통신 방식 |
|------|------|-----------|
| MQ2 가스 센서 | **A0** | ADC (아날로그) |
| SCD30 CO2 센서 | **I2C0** | I2C (주소: `0x61`) |
| OLED 디스플레이 | **I2C1** | I2C (주소: `0x3C`) |
| RGB LED Stick | **D16** | 디지털 (NeoPixel) |

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 4개 모두 같은 방식입니다.
:::

:::caution[I2C 포트 구분]
SCD30과 OLED 모두 I2C 센서이지만, 서로 다른 I2C 버스에 연결합니다. SCD30은 **I2C0**, OLED는 **I2C1**입니다. 바꿔 꽂으면 코드에서 인식하지 못합니다.
:::

## 활동 가이드

### Step 1. 4부품 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.**
2. **MQ2 가스 센서**를 Grove 케이블로 **A0 포트**에 연결합니다.
3. **SCD30 CO2 센서**를 **I2C0 포트**에 연결합니다.
4. **OLED 디스플레이**를 **I2C1 포트**에 연결합니다.
5. **RGB LED Stick**을 **D16 포트**에 연결합니다.
6. Micro USB 케이블로 컴퓨터와 연결합니다.

:::tip
4개의 케이블이 서로 엉키지 않도록 정리해두면 나중에 디버깅이 편합니다.
:::

### Step 2. 이중 센서 비교 실험 (15분)

먼저 MQ2와 SCD30을 동시에 읽으며 **반응 패턴의 차이**를 관찰합니다.

1. 아래 [기본 코드](#기본-코드)를 Thonny에 입력하고 실행하세요.
2. 다음 실험을 해보세요.

| 실험 | MQ2 반응 | CO2 반응 | 결론 |
|------|----------|----------|------|
| 입김 불기 | 약간 상승 | 크게 상승 | CO2만 높음 |
| 손소독제 가까이 대기 | 크게 상승 | 변화 없음 | MQ2만 높음 |
| 입김 + 손소독제 동시 | 크게 상승 | 크게 상승 | 둘 다 높음 |

:::note[생각해보기]
손소독제에는 CO2가 없지만 알코올 가스가 있습니다. 입김에는 CO2가 있지만 가연성 가스는 적습니다. 전자담배 연기에는 둘 다 포함됩니다. 이 차이가 이중 검증의 핵심입니다.
:::

### Step 3. 이중 검증 로직 이해하기 (10분)

아래 다이어그램은 MQ2와 SCD30 두 센서의 값을 조합하여 4가지 상태를 판정하는 로직입니다.

### Step 4. 풀버전 시스템 구동 및 테스트 (20분)

1. [도전 코드](#도전-코드-이중-검증--oled--led-경보)를 Thonny에 입력하고 실행하세요.
2. 처음 10초간 **기준값 보정(Calibrating)** 이 진행됩니다. 이 동안 센서 근처에 아무것도 대지 마세요.
3. 보정이 끝나면 OLED에 상태가 표시됩니다.
4. 다음을 시도하며 반응을 관찰하세요.

| 시도 | OLED 표시 | LED 색 |
|------|-----------|--------|
| 아무것도 안 함 | `[SAFE]` | 초록 (약하게) |
| 손소독제 가까이 | `[CHEMICAL]` | 주황 |
| 입김 불기 | `[VENTILATE]` | 노랑 |
| 입김 + 손소독제 동시 | `[DETECTED]` | 빨강 점멸 |

:::note[생각해보기]
검출 횟수(`Detect: N`)가 누적되는 것을 관찰하세요. 이 로그 기능은 "증거"가 됩니다. 실제 학교 화장실 모니터링 장치에도 이런 로그가 기록됩니다.
:::

### 기본 코드

```python
# === MQ2 + SCD30 동시 읽기 ===
# 두 센서의 반응 패턴을 비교합니다

from machine import I2C, Pin, ADC
import scd30
import time

# --- 하드웨어 ---
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))   # SCD30 (I2C0)
gas = ADC(Pin(26))                        # MQ2 (A0 = GP26)
co2_sensor = scd30.SCD30(i2c0, addr=0x61)
co2_sensor.start_continous_measurement()

def read_mq2_avg(n=10):
    """MQ2 값을 n번 읽어 평균을 구합니다 (노이즈 감소)"""
    total = 0
    for _ in range(n):
        total += gas.read_u16()
        time.sleep_ms(10)
    return total // n

print("=== 이중 센서 모니터 ===")
print("MQ2(ADC)와 SCD30(I2C)를 동시에 관찰합니다")
print("손소독제 vs 입김의 차이를 비교해보세요!")
time.sleep(3)

while True:
    mq2_val = read_mq2_avg()

    if co2_sensor.data_ready():
        co2, temp, hum = co2_sensor.read_measurement()
        print(f"MQ2: {mq2_val:5d} | CO2: {co2:6.0f}ppm | T:{temp:.1f}C")

    time.sleep(2)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 9 | `I2C(0, sda=Pin(8), scl=Pin(9))` | Grove Shield의 I2C0 포트를 사용합니다. SCD30 센서가 여기에 연결됩니다. |
| 10 | `ADC(Pin(26))` | A0 포트(GP26)에 연결된 MQ2 가스 센서를 아날로그로 읽습니다. |
| 12 | `start_continous_measurement()` | SCD30에 연속 측정 모드를 시작하라고 명령합니다. 약 2초마다 새 데이터가 준비됩니다. |
| 14~18 | `read_mq2_avg()` | MQ2 값을 10번 읽어 평균을 구합니다. 아날로그 센서는 노이즈가 있으므로 평균을 내면 더 안정적입니다. |
| 27 | `co2_sensor.data_ready()` | SCD30이 새 데이터를 준비했는지 확인합니다. 준비되지 않았으면 건너뜁니다. |
| 29 | `f"MQ2: {mq2_val:5d}"` | `:5d`는 5자리 정수로 출력하라는 포맷입니다. 숫자 폭이 맞춰져 비교하기 쉽습니다. |

### 도전 코드: 이중 검증 + OLED + LED 경보

기본 코드의 센서 읽기에 판정 로직, OLED 표시, LED 경보를 추가한 풀버전입니다.

```python
# === 전자담배 검출기 풀버전 ===
# MQ2 + SCD30 이중 검증 + OLED + LED Stick

from machine import I2C, Pin, ADC
from neopixel import NeoPixel
import ssd1306
import scd30
import time

# --- 하드웨어 ---
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))   # SCD30 (I2C0)
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))   # OLED (I2C1)
gas = ADC(Pin(26))                        # MQ2 (A0)
co2_sensor = scd30.SCD30(i2c0, addr=0x61)
co2_sensor.start_continous_measurement()
oled = ssd1306.SSD1306_I2C(128, 64, i2c1)
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))  # RGB LED Stick (D16)

# --- 임계값 ---
MQ2_THRESHOLD = 25000
CO2_BASELINE = 600
CO2_SPIKE = 200

# --- 상태 ---
co2_baseline = CO2_BASELINE
detection_count = 0

def read_mq2_avg(n=10):
    total = 0
    for _ in range(n):
        total += gas.read_u16()
        time.sleep_ms(10)
    return total // n

def set_leds(r, g, b):
    for i in range(NUM_LEDS):
        np[i] = (r, g, b)
    np.write()

def blink_alert():
    for _ in range(3):
        set_leds(255, 0, 0)
        time.sleep(0.1)
        set_leds(0, 0, 0)
        time.sleep(0.1)

def judge(mq2, co2):
    """이중 검증 판정"""
    global detection_count
    mq2_high = mq2 > MQ2_THRESHOLD
    co2_high = co2 > (co2_baseline + CO2_SPIKE)

    if mq2_high and co2_high:
        detection_count += 1
        return "DETECTED", (255, 0, 0)
    elif mq2_high:
        return "CHEMICAL", (255, 165, 0)
    elif co2_high:
        return "VENTILATE", (255, 255, 0)
    else:
        return "SAFE", (0, 50, 0)

# --- 기준값 보정 ---
oled.fill(0)
oled.text("Calibrating...", 0, 28)
oled.show()
print("기준값 보정 중... (10초)")
time.sleep(5)

total_co2 = 0
count = 0
for _ in range(5):
    if co2_sensor.data_ready():
        c, _, _ = co2_sensor.read_measurement()
        total_co2 += c
        count += 1
    time.sleep(2)
if count > 0:
    co2_baseline = total_co2 / count
print(f"CO2 기준값: {co2_baseline:.0f}ppm")

# --- 메인 루프 ---
print("=== 검출기 풀버전 가동 ===")

while True:
    mq2_val = read_mq2_avg()

    if co2_sensor.data_ready():
        co2, temp, hum = co2_sensor.read_measurement()
        status, color = judge(mq2_val, co2)

        # LED 표시
        if status == "DETECTED":
            blink_alert()
        else:
            set_leds(*color)

        # OLED 표시
        oled.fill(0)
        oled.text(f"[{status}]", 0, 0)
        oled.text(f"MQ2: {mq2_val}", 0, 14)
        oled.text(f"CO2: {co2:.0f}ppm", 0, 26)
        oled.text(f"T:{temp:.1f}C H:{hum:.0f}%", 0, 38)
        oled.text(f"Detect: {detection_count}", 0, 52)
        oled.show()

        print(f"[{status}] MQ2:{mq2_val} CO2:{co2:.0f} 검출:{detection_count}회")

    time.sleep(2)
```

:::note[코드 읽어보기]
`judge()` 함수가 이중 검증의 핵심입니다. MQ2와 CO2 값을 각각 임계값과 비교한 뒤, **둘 다 높을 때만** "DETECTED"를 반환합니다. `MQ2_THRESHOLD`와 `CO2_SPIKE` 값을 여러분 환경에 맞게 조절해보세요. 기준값 보정 단계에서 현재 교실의 CO2 수준을 자동으로 측정합니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

공기 중의 가스 분자가
**→** MQ2 센서에서 **ADC 값**으로 변환되고
**→** 동시에 SCD30이 **CO2 농도**를 측정하고
**→** Pico가 두 값을 조합하여 **4가지 상태로 판정**하고
**→** OLED에 **숫자와 상태**를 표시하고
**→** LED Stick이 **색으로 위험도**를 알려줍니다.

센서 하나로는 손소독제와 전자담배를 구분할 수 없었습니다. 그러나 서로 다른 원리의 센서 두 개를 **조합(센서 퓨전)** 하자, 구분이 가능해졌습니다. 자율주행차가 카메라, 라이다, 레이더를 함께 사용하는 이유, 화재 경보기가 연기와 열을 동시에 감지하는 이유가 바로 이것입니다.

3. **임계값을 더 낮추면** 민감도는 올라가지만 오탐이 늘 수 있습니다. 어떤 값이 적절할까요?

:::note[정밀도와 재현율]
오탐을 줄이면(정밀도 향상) 진짜 위험을 놓칠 수 있고(재현율 하락), 민감하게 반응하면(재현율 향상) 오탐이 늘어납니다(정밀도 하락). 이 균형을 맞추는 것이 센서 시스템 설계의 핵심입니다.
:::

## 마무리

### 오늘 배운 것

- **센서 퓨전**: 서로 다른 원리의 센서 두 개(ADC + I2C)를 조합하여 오탐을 줄이는 이중 검증 로직을 설계했습니다.
- **4부품 통합**: MQ2, SCD30, OLED, RGB LED Stick 네 개의 부품을 하나의 시스템으로 연결하고 제어했습니다.
- **자동 보정**: 환경마다 다른 CO2 기준값을 시작 시 자동으로 측정하여 보정하는 방법을 배웠습니다.
- **상태 기록**: 검출 횟수를 누적 기록하여 증거를 남기는 로그 개념을 경험했습니다.

### 다음 시간 미리보기

:::
