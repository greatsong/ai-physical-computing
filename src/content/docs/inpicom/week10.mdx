## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |

| BME280 센서 | 온도, 습도, 기압을 동시에 측정하는 I2C 센서입니다. 오늘은 **기압** 데이터를 핵심으로 활용합니다. |
| OLED 디스플레이 (0.96") | 128x64 픽셀의 작은 화면으로, 고도와 층수를 표시합니다. I2C 방식으로 통신합니다. |

| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

BME280을 **I2C1 포트**에, OLED를 **I2C0 포트**에, RGB LED Stick을 **D16 포트**에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다.
:::

:::note[포트 확인]
BME280과 OLED는 모두 I2C 장치이지만, **서로 다른 I2C 포트**에 연결합니다. BME280은 I2C1, OLED는 I2C0입니다. I2C 주소가 다르면 같은 버스에 연결할 수도 있지만, 오늘은 포트를 나누어 안정성을 높입니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.**
2. **BME280 → I2C1**, **OLED → I2C0**, **RGB LED Stick → D16** 에 각각 연결합니다.
3. **Micro USB 케이블로 컴퓨터와 연결합니다.**

:::tip
OLED 화면은 연결하자마자 켜지지 않습니다. 코드가 실행되어야 화면에 내용이 표시됩니다.
:::

### Step 2. 기압과 고도의 관계 이해하기 (10분)

높이 올라가면 기압이 낮아집니다. 이 관계를 수식으로 표현하면 기압 데이터만으로 고도를 추정할 수 있습니다.

핵심은 **기준점 보정**입니다. 시작할 때 현재 기압을 측정해서 0m로 설정하면, 그 다음부터는 기압 변화만으로 상대 고도를 계산할 수 있습니다.

:::note[기압과 고도]
기압이 약 0.12 hPa 떨어지면 약 1m 올라간 것입니다. 1층(약 3m)을 오르면 기압이 약 0.36 hPa 떨어집니다. BME280은 이 미세한 변화를 감지할 수 있습니다.
:::

### Step 3. 기본 고도계 실행하기 (15분)

1. Thonny를 열고 아래 [기본 코드](#기본-코드)를 입력합니다.
2. **실행 전 5초간 Pico를 움직이지 마세요.** 기준점 보정이 진행됩니다.
3. Shell 창에 기압, 상대 고도, 층수가 출력되는지 확인하세요.
4. **Pico를 들고 높이 들어올려 보세요.** 고도가 변하나요?

:::note[생각해보기]
책상 위와 바닥에서 고도 차이가 나타나나요? 의자 위에 올라서면 어떤가요? BME280이 감지할 수 있는 최소 높이 변화는 얼마일까요?
:::

### Step 4. 계단 실험하기 (20분)

[도전 코드](#도전-코드-oled-고도계--led-층수-표시기)를 실행하여 OLED와 LED가 동시에 동작하는 고도계를 만든 후, 계단을 오르내리며 실험합니다.

1. **도전 코드**를 Thonny에 입력하고 실행합니다.
2. Pico를 들고 **계단을 1층 올라가 보세요.** OLED에 표시되는 층수를 확인합니다.
3. 다시 내려오면 0층으로 돌아오는지 확인합니다.

### 기본 코드

```python
from machine import I2C, Pin
import bme280
import time

# --- 하드웨어 설정 ---
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))   # Grove I2C1 (BME280)
bme = bme280.BME280(i2c=i2c1)

# --- 고도 계산 함수 ---
def calc_altitude(pressure_hpa, sea_level=1013.25):
    """기압(hPa)으로 고도(m) 추정"""
    return 44330.0 * (1.0 - (pressure_hpa / sea_level) ** 0.1903)

# --- 기준점 보정 ---
print("기준점 보정 중... 5초간 가만히 계세요.")
total = 0
for i in range(10):
    p_str = bme.values[1]       # "1013.25hPa"
    p = float(p_str[:-3])       # 숫자만 추출
    total += p
    time.sleep(0.5)
base_pressure = total / 10
base_altitude = calc_altitude(base_pressure)
print(f"기준 기압: {base_pressure:.2f} hPa")

# --- 실시간 측정 ---
print("\n=== 고도 측정 시작 ===")
print("계단을 오르내려 보세요!")

while True:
    p_str = bme.values[1]
    pressure = float(p_str[:-3])
    altitude = calc_altitude(pressure)
    relative = altitude - base_altitude
    floor_num = relative / 3.0

    print(f"기압: {pressure:.1f}hPa | 상대고도: {relative:+.1f}m | 약 {floor_num:+.1f}층")
    time.sleep(1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 6 | `I2C(1, sda=Pin(6), scl=Pin(7))` | Grove Shield의 I2C1 포트에 연결된 BME280을 사용합니다. |
| 11 | `44330.0 * (1.0 - ...)` | 국제 기압-고도 공식입니다. 대기의 온도와 밀도를 고려한 근사식으로, 외울 필요 없이 함수로 사용하면 됩니다. |
| 16~22 | 기준점 보정 블록 | 10번 측정의 평균을 기준 기압으로 설정합니다. 여러 번 측정하면 오차가 줄어듭니다. |
| 19 | `p_str[:-3]` | `"1013.25hPa"`에서 마지막 3글자(`hPa`)를 제거하고 숫자만 추출합니다. |
| 31 | `relative / 3.0` | 상대 고도를 층수로 변환합니다. 일반 건물 1층 높이가 약 3m입니다. |

### 도전 코드: OLED 고도계 + LED 층수 표시기

OLED에 기압, 고도, 층수를 표시하고, LED Stick으로 현재 층을 시각적으로 나타냅니다.

```python
from machine import I2C, Pin
from neopixel import NeoPixel
import ssd1306
import bme280
import time

# --- 하드웨어 설정 ---
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))   # Grove I2C1 (BME280)
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))   # Grove I2C0 (OLED)
bme = bme280.BME280(i2c=i2c1)
oled = ssd1306.SSD1306_I2C(128, 64, i2c0)
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)          # Grove D16

def calc_altitude(p, sea=1013.25):
    return 44330.0 * (1.0 - (p / sea) ** 0.1903)

# --- 기준점 보정 ---
oled.fill(0)
oled.text("Calibrating...", 0, 28)
oled.show()

total = 0
for _ in range(10):
    total += float(bme.values[1][:-3])
    time.sleep(0.5)
base_alt = calc_altitude(total / 10)

def update_floor_led(floor_num):
    """층수에 따라 LED 표시"""
    for i in range(NUM_LEDS):
        np[i] = (0, 0, 0)

    led_idx = max(0, min(9, int(floor_num + 0.5)))

    if led_idx < 4:
        np[led_idx] = (0, 0, 200)      # 저층: 파랑
    elif led_idx < 7:
        np[led_idx] = (0, 200, 0)      # 중층: 초록
    else:
        np[led_idx] = (200, 0, 0)      # 고층: 빨강

    for j in range(led_idx):
        np[j] = (10, 10, 10)           # 궤적 표시

    np.write()

# --- 메인 ---
while True:
    p = float(bme.values[1][:-3])
    t = bme.values[0]
    alt = calc_altitude(p) - base_alt
    floor_num = alt / 3.0

    update_floor_led(floor_num)

    oled.fill(0)
    oled.text("= Altimeter =", 15, 0)
    oled.text(f"Press: {p:.1f}hPa", 0, 16)
    oled.text(f"Alt: {alt:+.1f} m", 0, 30)
    oled.text(f"Floor: {floor_num:+.1f} F", 0, 44)
    oled.text(f"Temp: {t}", 0, 56)
    oled.show()

    time.sleep(0.5)
```

:::note[코드 읽어보기]
`update_floor_led` 함수는 현재 층에 해당하는 LED 하나를 밝게 켜고, 그 아래 LED들은 흐릿하게 켜서 궤적을 보여줍니다. 저층은 파랑, 중층은 초록, 고층은 빨강으로 색이 바뀝니다. 학교 건물의 실제 층고를 알면 `alt / 3.0`의 3.0을 바꿔보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

공기의 무게(기압)가
**→** BME280에서 **hPa 숫자**로 측정되고
**→** 기준점과의 **차이**를 구하고
**→** 국제 기압-고도 공식으로 **미터 단위 높이**로 변환하고
**→** 3m로 나누어 **층수**를 추정하고
**→** OLED와 LED로 **눈에 보이게** 표시했습니다.

비행기의 고도계, 스마트워치의 층수 측정, 등산앱의 고도 표시 — 모두 이 원리의 확장입니다. 같은 BME280 센서를 9차시에는 온도→색상에, 오늘은 기압→고도에 사용했습니다. **같은 센서, 다른 해석, 전혀 다른 프로젝트** — 이것이 데이터 활용의 힘입니다.

3. 출발점으로 돌아왔을 때 고도가 **정확히 0m인가요?** 아니라면 왜 그럴까요?
4. **오차를 줄이려면** 어떤 방법이 있을까요?

:::note[보정(Calibration)의 중요성]
기준점 보정을 하지 않으면, 해수면 기준 고도가 나옵니다. 날씨가 바뀌면 기압도 바뀌므로, 장시간 사용하면 오차가 커집니다. 전문 고도계는 GPS와 함께 써서 주기적으로 보정합니다. **모든 측정에는 기준이 필요하고, 기준을 어떻게 잡느냐가 정확도를 결정합니다.**
:::
