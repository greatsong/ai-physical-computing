## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. |
| Grove RGB LED Stick | WS2813 칩 기반의 10개짜리 LED 바입니다. 각 LED를 개별적으로 제어할 수 있고, 빨강(R)·초록(G)·파랑(B) 빛을 섞어 1600만 가지 이상의 색을 만들 수 있습니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결합니다. |

### 연결 방법

RGB LED Stick을 Grove 케이블로 Shield의 **D16 포트**(디지털)에 연결하세요.

:::tip[IN 방향 확인]
LED Stick에는 **IN**과 **OUT** 표시가 있습니다. **IN 쪽이 Shield에 연결**되어야 합니다. 반대로 꽂으면 LED가 켜지지 않습니다.
:::

:::note[포트 찾기]
RGB LED Stick은 디지털 신호로 통신합니다. 반드시 **"D16"이라고 적힌 포트**에 연결하세요. D16 포트는 Pico의 GP16 핀에 연결되어 있습니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.**
2. **RGB LED Stick을 Grove 케이블로 D16 포트에 연결합니다.** LED Stick의 **IN** 쪽이 Shield를 향하도록 꽂으세요.
3. **Micro USB 케이블로 컴퓨터와 연결합니다.**

:::tip
LED Stick의 앞면에서 작은 글씨로 IN/OUT이 적혀 있습니다. 잘 안 보이면 돋보기를 사용하거나, 케이블을 한 방향으로 꽂아 테스트한 뒤 안 되면 뒤집어 보세요.
:::

### Step 2. 첫 코드 — 색상 바꾸기 (10분)

1. **Thonny**를 열고 아래 [기본 코드](#기본-코드)를 입력합니다.
2. **실행(Run)** 버튼을 클릭합니다.
3. 1초마다 색이 바뀌는지 확인하세요: 빨강 → 초록 → 파랑 → 노랑 → 보라 → 청록 → 흰색.

:::note[생각해보기]
빨강 (255, 0, 0)과 초록 (0, 255, 0)을 합치면 무슨 색이 될까요? 물감에서는 갈색이 되지만, 빛에서는 **노란색**이 됩니다. 빛의 혼합과 물감의 혼합은 반대입니다.
:::

### Step 3. 실습 — 나만의 패턴 만들기 (25분)

[도전 코드](#도전-코드-나만의-조명-쇼)의 패턴 A~C 중 마음에 드는 것을 골라 색상과 속도를 수정해보세요.

**창작 미션:**
- **기본**: 패턴 하나를 골라 색상/속도를 자기 취향으로 수정
- **도전**: 패턴 2개를 이어붙여 자기만의 '조명 쇼' 만들기
- **고수**: 새로운 패턴 함수를 직접 작성하기 (예: 별 반짝임, 불꽃놀이)

작품에 이름을 붙여보세요. 실제 조명 디자이너들도 패턴에 이름을 붙입니다.

### Step 4. 토론하기 (10분)

각자 만든 작품을 보여주고 다음을 이야기해보세요.

1. **"같은 LED인데 왜 느낌이 다를까?"** 색상, 속도, 밝기가 주는 감정적 인상의 차이를 이야기해봅시다.
2. **"RGB (255, 255, 255)와 (50, 50, 50)은 둘 다 흰색인데 뭐가 다를까?"** 밝기 조절은 어떻게 하나요?
3. **"이 LED를 실생활 어디에 활용할 수 있을까?"** 알림, 장식, 안전 표시 등 구체적으로 제안해봅시다.

### 기본 코드

```python
# === RGB LED Stick: 전체 색상 제어 ===
# LED 10개를 한 번에 같은 색으로 켜는 코드

from machine import Pin
from neopixel import NeoPixel
import time

# --- LED 설정 ---
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))

# --- 색상 사전 ---
RED    = (255, 0, 0)
GREEN  = (0, 255, 0)
BLUE   = (0, 0, 255)
YELLOW = (255, 255, 0)
PURPLE = (128, 0, 128)
CYAN   = (0, 255, 255)
WHITE  = (255, 255, 255)
OFF    = (0, 0, 0)

# --- 전체 LED를 한 색으로 설정 ---
def fill(color):
    for i in range(NUM_LEDS):
        np[i] = color
    np.write()

# --- 색상 퍼레이드 ---
colors = [RED, GREEN, BLUE, YELLOW, PURPLE, CYAN, WHITE]
names  = ["빨강", "초록", "파랑", "노랑", "보라", "청록", "흰색"]

for color, name in zip(colors, names):
    fill(color)
    print(f"{name} 점등!")
    time.sleep(1)

fill(OFF)
print("끝!")
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 5 | `from neopixel import NeoPixel` | NeoPixel은 LED를 개별 제어하는 프로토콜입니다. MicroPython에 기본 내장되어 있습니다. |
| 10 | `NeoPixel(Pin(16), NUM_LEDS, timing=(...))` | GP16 핀에 연결된 10개짜리 LED를 제어하겠다는 뜻입니다. D16 포트가 GP16에 연결되어 있습니다. WS2813 LED는 전용 타이밍이 필요합니다. |
| 24 | `np[i] = color` | i번째 LED에 (R, G, B) 색상을 넣습니다. 0번이 첫 번째, 9번이 마지막입니다. |
| 25 | `np.write()` | 설정한 색상을 LED에 실제로 전송합니다. 이 줄이 없으면 아무 변화도 없습니다. |

### 도전 코드: 나만의 조명 쇼

아래 패턴 중 원하는 것을 골라 색상과 속도를 수정해보세요.

```python
# === 나만의 LED 패턴 만들기 ===

from machine import Pin
from neopixel import NeoPixel
import time

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))

# -- 패턴 A: 숨쉬기 (밝아졌다 어두워지는 효과) --
def breathe(r, g, b, speed=0.03):
    for brightness in range(0, 100, 2):
        scale = brightness / 100
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)
    for brightness in range(100, 0, -2):
        scale = brightness / 100
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)

# -- 패턴 B: 나이트라이더 (불빛이 왔다갔다) --
def knight_rider(r, g, b, speed=0.08):
    for i in range(NUM_LEDS):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[i] = (r, g, b)
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)
        np.write()
        time.sleep(speed)
    for i in range(NUM_LEDS - 2, 0, -1):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[i] = (r, g, b)
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)
        np.write()
        time.sleep(speed)

# -- 패턴 C: 경광등 (빨강/파랑 교대) --
def police_light(speed=0.15):
    for _ in range(5):
        for i in range(5):
            np[i] = (255, 0, 0)
        for i in range(5, 10):
            np[i] = (0, 0, 255)
        np.write()
        time.sleep(speed)
        for i in range(5):
            np[i] = (0, 0, 255)
        for i in range(5, 10):
            np[i] = (255, 0, 0)
        np.write()
        time.sleep(speed)

# === 원하는 패턴을 실행하세요! ===

for _ in range(3):
    breathe(0, 100, 255)    # 하늘색 숨쉬기

for _ in range(3):
    knight_rider(255, 0, 0) # 빨간 나이트라이더

police_light(0.1)           # 경광등

# 끄기
for i in range(NUM_LEDS):
    np[i] = (0, 0, 0)
np.write()
```

:::note[코드 읽어보기]
`int()`는 소수점을 버리는 함수입니다. `r * scale`이 127.5 같은 소수가 나올 수 있는데, RGB 값은 정수만 허용합니다. `r//4`는 정수 나눗셈으로 밝기를 1/4로 줄여 잔상 효과를 만듭니다. 색상과 속도 값을 자유롭게 바꿔보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

코드가 (R, G, B) 숫자 세 개를
**→** NeoPixel 프로토콜로 **직렬 데이터**로 변환하고
**→** 데이터 선 1개를 통해 10개 LED에 **순서대로** 전달하고
**→** 각 LED의 빨강·초록·파랑 칩이 **빛을 혼합**하여
**→** 우리 눈에 원하는 **색상**으로 보입니다.

## 데이터로 생각하기

RGB 값과 실제로 보이는 색의 관계를 탐구해봅시다.

#### RGB 색상 실험 기록표

| R 값 | G 값 | B 값 | 예상한 색 | 실제로 본 색 | 비고 |
|------|------|------|-----------|-------------|------|
| 255 | 0 | 0 | 빨강 | | |
| 0 | 255 | 0 | 초록 | | |
| 255 | 255 | 0 | ? | | |
| 255 | 0 | 255 | ? | | |
| 255 | 165 | 0 | ? | | |
| 100 | 100 | 100 | ? | | |

기록표를 채운 뒤 다음 질문에 답해보세요.

1. **(255, 255, 0)과 (128, 128, 0)**은 둘 다 노란색 계열인데, 차이는 무엇인가요?
2. **흰색 (255, 255, 255)을 더 부드럽게** 만들려면 어떤 값을 쓰면 될까요?
3. 여러분이 만든 **가장 마음에 드는 색**의 RGB 값은 무엇인가요?

:::note[빛의 혼합 vs 물감의 혼합]
빛은 더할수록 밝아집니다(가산 혼합). 빨강 + 초록 = 노랑, 세 색 모두 최대 = 흰색. 물감은 반대로 더할수록 어두워집니다(감산 혼합). 스마트폰, TV, 모니터 모두 빛의 혼합(RGB)으로 색을 만듭니다.
:::

## 마무리

### 오늘 배운 것

- **NeoPixel 제어**: `np[i] = (R, G, B)`로 색을 넣고, `np.write()`로 적용하는 두 단계를 배웠습니다.
- **RGB 색상 모델**: 빨강·초록·파랑 세 값(0~255)의 조합으로 모든 색을 표현할 수 있습니다.
- **애니메이션 원리**: `for` 반복문과 `time.sleep()`을 조합하면 LED 패턴이 움직이는 효과를 만들 수 있습니다.
- **입력 vs 출력**: 1차시의 센서가 '읽는' 장치였다면, LED는 '표현하는' 장치입니다.
