## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) x 2 | 센서와 Shield를 연결하는 케이블입니다. |
| Grove Speech Recognizer | 미리 등록된 영어 명령어를 인식하는 음성인식 센서입니다. UART 방식으로 통신합니다. |
| Grove RGB LED Stick (10 LED) | 10개의 풀컬러 LED가 달린 스틱입니다. NeoPixel 방식으로 각 LED를 개별 제어할 수 있습니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

Speech Recognizer를 **UART0 포트**에, RGB LED Stick을 **D16 포트**에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[UART 포트란?]
UART는 두 장치가 **1:1로 대화**하는 통신 방식입니다. TX(보내기)와 RX(받기) 두 선이 **교차 연결**됩니다. Shield에서 "UART"라고 적힌 포트를 찾아 연결하세요. GP0(TX)과 GP1(RX)에 연결되어 있습니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.**
2. **Speech Recognizer를 Grove 케이블로 UART0 포트에 연결합니다.**
3. **RGB LED Stick을 Grove 케이블로 D16 포트에 연결합니다.**
4. **Micro USB 케이블로 컴퓨터와 연결합니다.**

:::tip
음성인식 센서는 **조용한 환경**에서, **10cm 이내**로 가까이 대고, **또박또박** 발음해야 잘 인식됩니다. 주변이 시끄러우면 인식률이 떨어집니다.
:::

### Step 2. 음성 인식 테스트 (10분)

1. Thonny를 열고 아래 **기본 코드**를 입력합니다.
2. 실행 후 센서에 대고 "Turn on the light"이라고 말해보세요.
3. Shell 창에 어떤 번호가 출력되는지 확인합니다.
4. 다른 명령어도 시도해보세요: "Turn off the light", "Play music", "Pause" 등.

:::note[내장 명령어 목록]
이 센서에는 10개의 영어 명령어가 미리 등록되어 있습니다. 한국어는 인식하지 못합니다. 주요 명령어: 1번 "Turn on the light", 2번 "Turn off the light", 3번 "Play music", 4번 "Pause", 5번 "Next", 6번 "Previous", 7번 "Up", 8번 "Down".
:::

### Step 3. 음성 제어 무드등 (20분)

1. **도전 코드**를 입력합니다.
2. "Turn on the light" -> 따뜻한 백색 점등을 확인합니다.
3. "Play music" -> 무지개 패턴이 흘러가는지 확인합니다.
4. "Pause" -> 숨쉬기 패턴으로 전환되는지 확인합니다.
5. "Turn off the light" -> 소등되는지 확인합니다.

아직 사용하지 않은 명령어(Next, Previous, Up, Down)에 어떤 LED 동작을 연결하면 좋을지 설계해보세요.

### 기본 코드

```python
# === 음성인식 센서 테스트 ===
# 센서가 인식한 명령어 번호를 확인

from machine import UART, Pin
import time

# --- UART 설정 ---
# Grove UART 포트: TX=GP0, RX=GP1
uart = UART(0, baudrate=9600, tx=Pin(0), rx=Pin(1))

# --- 내장 명령어 목록 ---
commands = {
    1: "Turn on the light",
    2: "Turn off the light",
    3: "Play music",
    4: "Pause",
    5: "Next",
    6: "Previous",
    7: "Up (volume up)",
    8: "Down (volume down)",
    9: "Turn on AC",
    10: "Turn off AC",
}

print("=== 음성인식 테스트 ===")
print("센서에 대고 영어로 말해보세요!")
print("예: 'Turn on the light'")

while True:
    if uart.any():
        data = uart.read()
        if data:
            cmd_id = data[0]  # 첫 바이트가 명령어 번호
            cmd_name = commands.get(cmd_id, f"알 수 없는 명령({cmd_id})")
            print(f"인식됨: #{cmd_id} = {cmd_name}")
    time.sleep(0.1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 8 | `UART(0, baudrate=9600, ...)` | UART 통신을 초기화합니다. baudrate는 초당 데이터 전송 속도(9600bps)로, 센서와 Pico가 같은 속도로 맞춰야 합니다. |
| 8 | `tx=Pin(0), rx=Pin(1)` | TX(보내기)는 GP0, RX(받기)는 GP1에 연결됩니다. UART는 TX-RX가 교차 연결되는 것이 특징입니다. |
| 27 | `uart.any()` | 수신 버퍼에 데이터가 있는지 확인합니다. 센서가 명령어를 인식하면 번호를 보내줍니다. |
| 29 | `data[0]` | 수신된 데이터의 첫 바이트가 명령어 번호입니다. 1번이면 "Turn on the light"입니다. |
| 30 | `commands.get(cmd_id, ...)` | 딕셔너리에서 번호에 해당하는 명령어 이름을 찾습니다. 없으면 기본값을 반환합니다. |

### 도전 코드: 음성 제어 무드등 + 패턴

```python
# === 음성 제어 무드등 – 패턴 포함 ===
# 기본 색상 + 무지개/숨쉬기 애니메이션 패턴

from machine import UART, Pin
from neopixel import NeoPixel
import time

uart = UART(0, baudrate=9600, tx=Pin(0), rx=Pin(1))
NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))

# --- LED 기본 함수 ---
def set_all(r, g, b):
    for i in range(NUM_LEDS):
        np[i] = (r, g, b)
    np.write()

def wheel(pos):
    """색상환: 0~255 위치를 RGB로 변환"""
    if pos < 85:
        return (255 - pos * 3, pos * 3, 0)
    elif pos < 170:
        pos -= 85
        return (0, 255 - pos * 3, pos * 3)
    else:
        pos -= 170
        return (pos * 3, 0, 255 - pos * 3)

# --- 애니메이션 패턴 ---
def rainbow_loop():
    """무지개 1바퀴"""
    for offset in range(256):
        for i in range(NUM_LEDS):
            np[i] = wheel((i * 256 // NUM_LEDS + offset) % 256)
        np.write()
        time.sleep(0.02)

def breathe_loop(r, g, b):
    """숨쉬기 1사이클"""
    for brightness in range(0, 100, 5):
        s = brightness / 100
        set_all(int(r*s), int(g*s), int(b*s))
        time.sleep(0.02)
    for brightness in range(100, 0, -5):
        s = brightness / 100
        set_all(int(r*s), int(g*s), int(b*s))
        time.sleep(0.02)

# --- 상태 관리 ---
current_mode = "off"

def handle(cmd_id):
    global current_mode

    if cmd_id == 1:    # "Turn on the light"
        set_all(255, 200, 100)
        current_mode = "static"
        print("조명 ON (따뜻한 백색)")

    elif cmd_id == 2:  # "Turn off the light"
        set_all(0, 0, 0)
        current_mode = "off"
        print("조명 OFF")

    elif cmd_id == 3:  # "Play music" -> 무지개
        current_mode = "rainbow"
        print("무지개 모드!")

    elif cmd_id == 4:  # "Pause" -> 숨쉬기
        current_mode = "breathe"
        print("숨쉬기 모드")

    elif cmd_id == 5:  # "Next" -> 랜덤 색상
        colors = [(255,0,0),(0,255,0),(0,0,255),
                  (255,255,0),(255,0,255),(0,255,255)]
        import random
        c = random.choice(colors)
        set_all(*c)
        current_mode = "static"
        print(f"랜덤 색상: {c}")

# --- 메인 루프 ---
print("=== 음성 제어 무드등 ===")
set_all(0, 0, 0)

while True:
    # 음성 명령 확인
    if uart.any():
        data = uart.read()
        if data:
            handle(data[0])

    # 현재 모드에 따라 애니메이션 실행
    if current_mode == "rainbow":
        rainbow_loop()
    elif current_mode == "breathe":
        breathe_loop(0, 100, 255)
    else:
        time.sleep(0.1)
```

:::note[코드 읽어보기]
`global current_mode`는 함수 안에서 함수 밖의 변수를 바꿀 때 사용합니다. `current_mode`가 "rainbow"면 메인 루프에서 `rainbow_loop()`를 계속 실행합니다. 무지개 도중에 명령이 안 먹는 경우가 있는데, 이는 `rainbow_loop()`가 실행되는 동안 `uart.any()`를 체크하지 못하기 때문입니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

여러분의 **음성**이
**->** 센서 내부의 마이크에서 **전기 신호**로 바뀌고
**->** 센서의 칩이 신호를 미리 등록된 명령어와 **패턴 매칭**하고
**->** 인식된 명령어 번호가 UART 통신으로 Pico에 **전송**되고
**->** 코드가 번호에 따라 LED 동작을 **선택**하고
**->** RGB LED Stick이 해당 **색상/패턴**으로 반응했습니다.

시리, 빅스비, AI 스피커 모두 이 원리를 확장한 것입니다. 다만 오늘 센서는 10개 명령어를 **로컬**에서 처리하고, AI 스피커는 수만 개 명령어를 **클라우드**에서 처리하는 차이가 있습니다.

## 마무리

### 오늘 배운 것

- **UART 통신**: TX/RX 교차 연결로 두 장치가 1:1 대화하는 세 번째 통신 방식을 배웠습니다.
- **음성 -> 동작 매핑**: 센서가 인식한 명령어 번호를 LED 동작에 연결하는 딕셔너리 구조를 활용했습니다.
- **상태 관리**: `current_mode` 변수로 현재 모드(off/static/rainbow/breathe)를 관리하는 패턴을 경험했습니다.
- **세 가지 통신 비교**: ADC(아날로그), I2C(주소 기반), UART(직렬 대화) 방식의 차이를 이해했습니다.

### 다음 시간 미리보기

:::tip[7차시 "CO2 환기 알리미"]
다음 시간에는 **SCD30 CO2 센서**로 교실 공기질을 측정합니다. CO2 농도에 따라 LED 색상이 바뀌고, OLED 화면에 수치가 표시되는 환기 알리미를 만들어봅니다. I2C에 센서 2개를 동시에 연결하는 방법도 배웁니다.
