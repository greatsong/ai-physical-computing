## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) x3 | 센서와 Shield를 연결하는 케이블입니다. 오늘은 3개의 부품을 연결합니다. |
| Grove Speech Recognizer | 영어 음성 명령을 인식하는 UART 센서입니다. "Up", "Down" 등의 명령어를 알아듣습니다. |
| Grove BME280 | 온도, 습도, 기압을 동시에 측정하는 I2C 센서입니다. |
| Grove OLED 디스플레이 0.96" | 128x64 픽셀의 소형 화면입니다. I2C 방식으로 통신합니다. |
| Micro USB 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

오늘은 **3가지 서로 다른 통신 방식**의 부품을 동시에 연결합니다.

| 부품 | 포트 | 통신 방식 |
|------|------|-----------|
| Speech Recognizer | **UART0** | UART (TX/RX 직렬 통신) |
| BME280 | **I2C1** | I2C (주소: `0x76`) |
| OLED 디스플레이 | **I2C0** | I2C (주소: `0x3C`) |

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다.
:::

:::caution[UART 포트 주의]
Speech Recognizer는 I2C가 아니라 **UART** 포트에 연결해야 합니다. UART 포트는 Shield에서 별도로 표시되어 있습니다. I2C 포트에 꽂으면 음성 인식이 동작하지 않습니다.
:::

## 활동 가이드

### Step 1. 3부품 연결 및 통신 구조 이해 (5분)

1. **Speech Recognizer**를 Grove 케이블로 **UART0 포트**에 연결합니다.
2. **BME280**을 **I2C1 포트**에 연결합니다.
3. **OLED 디스플레이**를 **I2C0 포트**에 연결합니다.
4. Micro USB 케이블로 컴퓨터와 연결합니다.

:::note[통신 방식 비교]
**UART**는 1:1 직렬 통신으로, 송신(TX)과 수신(RX) 두 선을 사용합니다. **I2C**는 버스 통신으로, SDA/SCL 두 선에 여러 장치를 주소로 구분하여 연결합니다. 오늘은 I2C 버스 두 개(I2C0, I2C1)를 사용하여 OLED와 BME280을 분리합니다.
:::

### Step 2. 음성 명령으로 콘솔 출력 전환 (15분)

1. 아래 [기본 코드](#기본-코드)를 Thonny에 입력하고 실행하세요.
2. Shell 창에 환경 데이터가 1초마다 출력됩니다.
3. **"Up!"** 이라고 말해보세요. 화면 모드가 전환됩니다.
4. **"Down!"** 이라고 말해보세요. 이전 모드로 돌아갑니다.

| 음성 명령 | 동작 | 화면 모드 |
|-----------|------|-----------|
| "Up" | 다음 모드 | ALL → TEMP → HUM → PRES → ALL... |
| "Down" | 이전 모드 | 역순 순환 |
| "Turn on the light" | 전체 보기 | ALL로 이동 |

:::tip
음성 인식 센서는 영어 명령만 인식합니다. 또렷하게, 센서에서 20cm 이내에서 말해보세요. 인식되면 센서의 LED가 깜빡입니다.
:::

### Step 3. OLED 화면 모드 구현 (20분)

[도전 코드](#도전-코드-음성-oled-환경-모니터)를 실행하면 4가지 화면 모드가 OLED에 표시됩니다.

| 모드 | 화면 내용 | 특징 |
|------|-----------|------|
| Overview | 온도, 습도, 기압 한눈에 | 요약 정보 |
| Temperature | 온도 + 쾌적도 + 막대 게이지 | 15~35도 범위 시각화 |
| Humidity | 습도 + 상태 + 막대 게이지 | 0~100% 범위 시각화 |
| Pressure | 기압 + 날씨 추정 + 고도 | 과학 공식 활용 |

:::note[생각해보기]
"Up!"이라고 말한 후 화면이 바뀌기까지 약간의 시간이 걸릴 수 있습니다. 이것은 음성인식 처리 시간 + UART 전송 시간 + 메인 루프의 `sleep(1)` 때문입니다. 실제 음성 비서(Siri, 알렉사)도 이런 지연이 있다는 점을 생각해보세요.
:::

### 기본 코드

```python
# === 음성 명령으로 데이터 전환 ===
# "Up"/"Down"으로 표시할 데이터를 변경합니다

from machine import UART, I2C, Pin
import bme280
import time

# --- 하드웨어 ---
uart = UART(0, baudrate=9600, tx=Pin(0), rx=Pin(1))  # Speech (UART0)
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))                # BME280 (I2C1)
bme = bme280.BME280(i2c=i2c1)

# --- 화면 모드 ---
MODE_ALL  = 0
MODE_TEMP = 1
MODE_HUM  = 2
MODE_PRES = 3

current_mode = MODE_ALL
mode_names = ["ALL", "TEMP", "HUMIDITY", "PRESSURE"]

print("=== 음성 제어 모니터 ===")
print("'Up' = 다음 화면, 'Down' = 이전 화면")

while True:
    # 음성 명령 확인
    if uart.any():
        data = uart.read()
        if data:
            cmd = data[0]
            if cmd == 7:     # "Up"
                current_mode = (current_mode + 1) % 4
                print(f"-> 다음: {mode_names[current_mode]}")
            elif cmd == 8:   # "Down"
                current_mode = (current_mode - 1) % 4
                print(f"<- 이전: {mode_names[current_mode]}")
            elif cmd == 1:   # "Turn on the light"
                current_mode = MODE_ALL
                print(f"-> 전체 보기")

    # 센서 데이터 읽기
    temp = bme.values[0]
    hum = bme.values[2]
    pres = bme.values[1]

    # 현재 모드에 따라 출력
    if current_mode == MODE_ALL:
        print(f"[ALL] T:{temp} H:{hum} P:{pres}")
    elif current_mode == MODE_TEMP:
        print(f"[TEMP] {temp}")
    elif current_mode == MODE_HUM:
        print(f"[HUM] {hum}")
    elif current_mode == MODE_PRES:
        print(f"[PRES] {pres}")

    time.sleep(1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 8 | `UART(0, baudrate=9600, tx=Pin(0), rx=Pin(1))` | UART0 포트로 음성 인식 센서와 통신합니다. 9600bps 속도로 데이터를 주고받습니다. |
| 9 | `I2C(1, sda=Pin(6), scl=Pin(7))` | I2C1 포트로 BME280 센서와 통신합니다. OLED가 없는 기본 코드에서는 I2C 하나만 사용합니다. |
| 25 | `uart.any()` | UART 버퍼에 수신된 데이터가 있는지 확인합니다. 음성 명령이 인식되면 데이터가 들어옵니다. |
| 27 | `cmd = data[0]` | 수신된 바이트의 첫 번째 값이 명령 번호입니다. 7은 "Up", 8은 "Down"입니다. |
| 29 | `(current_mode + 1) % 4` | 나머지 연산(`%`)으로 0→1→2→3→0으로 순환합니다. 모드가 3을 넘으면 다시 0으로 돌아옵니다. |
| 39 | `bme.values[0]` | BME280이 반환하는 값 배열에서 온도(인덱스 0), 기압(인덱스 1), 습도(인덱스 2)를 가져옵니다. |

### 도전 코드: 음성 OLED 환경 모니터

기본 코드에 OLED 화면 모드 4가지와 막대 게이지를 추가한 버전입니다.

```python
# === 음성 제어 OLED 환경 모니터 ===
# 음성 명령으로 4가지 화면 모드를 전환합니다

from machine import UART, I2C, Pin
import ssd1306
import bme280
import time

# --- 하드웨어 ---
uart = UART(0, baudrate=9600, tx=Pin(0), rx=Pin(1))  # Speech (UART0)
i2c0 = I2C(0, sda=Pin(8), scl=Pin(9))                # OLED (I2C0)
i2c1 = I2C(1, sda=Pin(6), scl=Pin(7))                # BME280 (I2C1)
bme = bme280.BME280(i2c=i2c1)
oled = ssd1306.SSD1306_I2C(128, 64, i2c0)

current_mode = 0
mode_names = ["Overview", "Temperature", "Humidity", "Pressure"]

# --- 화면 그리기 함수들 ---
def draw_overview(t, h, p):
    oled.fill(0)
    oled.text("= Overview =", 15, 0)
    oled.hline(0, 10, 128, 1)
    oled.text(f"Temp: {t}", 0, 16)
    oled.text(f"Humi: {h}", 0, 30)
    oled.text(f"Pres: {p}", 0, 44)
    oled.text("Say 'Up'/'Down'", 0, 56)
    oled.show()

def draw_temp_detail(temp_c):
    oled.fill(0)
    oled.text("= Temperature =", 5, 0)
    oled.text(f"{temp_c:.1f} C", 35, 14)
    if temp_c < 18:
        status = "Cold!"
    elif temp_c < 20:
        status = "Cool"
    elif temp_c <= 26:
        status = "Comfort"
    elif temp_c <= 28:
        status = "Warm"
    else:
        status = "Hot!"
    oled.text(status, 40, 28)
    bar_w = max(0, min(120, int((temp_c - 15) / 20 * 120)))
    oled.rect(4, 44, 120, 12, 1)
    oled.fill_rect(4, 44, bar_w, 12, 1)
    oled.text("15", 0, 58)
    oled.text("25", 55, 58)
    oled.text("35", 110, 58)
    oled.show()

def draw_hum_detail(hum_pct):
    oled.fill(0)
    oled.text("= Humidity =", 15, 0)
    oled.text(f"{hum_pct:.0f} %", 45, 14)
    if hum_pct < 30:
        status = "Too Dry!"
    elif hum_pct < 40:
        status = "Dry"
    elif hum_pct <= 60:
        status = "Comfort"
    elif hum_pct <= 70:
        status = "Humid"
    else:
        status = "Too Humid!"
    oled.text(status, 35, 28)
    bar_w = max(0, min(120, int(hum_pct / 100 * 120)))
    oled.rect(4, 44, 120, 12, 1)
    oled.fill_rect(4, 44, bar_w, 12, 1)
    oled.text("0%", 0, 58)
    oled.text("50%", 50, 58)
    oled.text("100%", 96, 58)
    oled.show()

def draw_pres_detail(pres_hpa):
    oled.fill(0)
    oled.text("= Pressure =", 12, 0)
    oled.text(f"{pres_hpa:.1f} hPa", 15, 14)
    if pres_hpa > 1020:
        weather = "High P - Clear"
    elif pres_hpa > 1010:
        weather = "Normal"
    elif pres_hpa > 1000:
        weather = "Low P - Cloudy?"
    else:
        weather = "Storm coming?"
    oled.text(weather, 0, 30)
    alt = 44330 * (1 - (pres_hpa / 1013.25) ** 0.1903)
    oled.text(f"Est.Alt: {alt:.0f}m", 0, 46)
    oled.show()

# --- 메인 루프 ---
print("=== 음성 제어 환경 모니터 ===")

while True:
    # 음성 명령 확인
    if uart.any():
        data = uart.read()
        if data:
            cmd = data[0]
            if cmd == 7:     # "Up"
                current_mode = (current_mode + 1) % 4
                print(f"-> {mode_names[current_mode]}")
            elif cmd == 8:   # "Down"
                current_mode = (current_mode - 1) % 4
                print(f"<- {mode_names[current_mode]}")
            elif cmd == 1:   # "Turn on" -> Overview
                current_mode = 0

    # 센서 읽기
    temp_s = bme.values[0]
    hum_s = bme.values[2]
    pres_s = bme.values[1]
    temp_c = float(temp_s[:-1])
    hum_f = float(hum_s[:-1])
    pres_f = float(pres_s[:-3])

    # 현재 모드에 따라 화면 그리기
    if current_mode == 0:
        draw_overview(temp_s, hum_s, pres_s)
    elif current_mode == 1:
        draw_temp_detail(temp_c)
    elif current_mode == 2:
        draw_hum_detail(hum_f)
    elif current_mode == 3:
        draw_pres_detail(pres_f)

    time.sleep(1)
```

:::note[코드 읽어보기]
각 `draw_*` 함수가 하나의 화면 모드를 담당합니다. `oled.fill(0)`으로 화면을 지우고, `oled.text()`로 텍스트를, `oled.rect()`/`oled.fill_rect()`로 막대 게이지를 그린 뒤, `oled.show()`로 화면에 표시합니다. 새로운 화면 모드를 추가하고 싶다면 `draw_` 함수를 하나 더 만들고 `current_mode` 분기를 추가하면 됩니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

여러분의 **목소리**가
**→** Speech Recognizer 내부의 마이크에서 **음파**로 감지되고
**→** 센서 내장 AI가 음파를 **명령 번호**로 변환하고
**→** UART 통신으로 Pico에 **바이트 데이터**로 전달되고
**→** Pico가 명령을 해석하여 **화면 모드를 전환**하고
**→** BME280에서 I2C로 읽은 **환경 데이터**를
**→** OLED에 I2C로 보내 **화면에 표시**했습니다.

입력(음성, UART), 처리(Pico), 센싱(BME280, I2C), 출력(OLED, I2C) — 네 가지 역할이 세 가지 통신 방식으로 연결된 완성형 시스템입니다. 스마트 스피커, 자동차 음성 비서, 음성 검색 — 모두 이 구조의 확장입니다.

| 명령 | 시도 횟수 | 성공 횟수 | 인식률 (%) | 비고 |

:::note[음성 인식의 한계]

:::
