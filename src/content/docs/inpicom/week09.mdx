### 연결 방법

BME280 센서를 **I2C1 포트**에, LED Strip을 **D16 포트**에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[포트 찾기]
BME280은 I2C 센서이므로 **I2C1** 포트에, LED Strip은 디지털 신호로 제어하므로 **D16** 포트에 연결합니다. 두 포트를 헷갈리지 않도록 주의하세요.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.**
2. **BME280 센서를 Grove 케이블로 I2C1 포트에 연결합니다.**
3. **LED Strip을 Grove 케이블로 D16 포트에 연결합니다.**
4. **Micro USB 케이블로 컴퓨터와 연결합니다.**

:::tip
LED Strip은 방향이 있습니다. 케이블이 연결된 쪽이 입력(IN) 쪽입니다. 화살표 방향을 확인하세요.
:::

### Step 2. 매핑 개념 이해하기 (10분)

`map_value` 함수는 값을 한 범위에서 다른 범위로 비례 변환합니다. 온도를 색상으로 바꾸는 핵심 원리입니다.

온도가 15°C일 때 **순수 파랑**, 25°C일 때 **초록**, 35°C일 때 **순수 빨강**이 됩니다. 기상청 날씨 지도에서 파란색이 추운 곳, 빨간색이 더운 곳인 것과 같은 원리입니다.

:::note[생각해보기]
`map_value(25, 15, 35, 0, 255)`의 결과는 몇일까요? 15~35 범위에서 25는 정확히 중간이므로, 0~255 범위에서도 중간인 **127**이 됩니다.
:::

### Step 3. 코드 실행하기 (15분)

1. Thonny를 열고 아래 [기본 코드](#기본-코드)를 입력합니다.
2. **실행(Run)** 버튼을 클릭합니다.
3. Shell 창에 온도와 RGB 값이 출력되는지 확인하세요.
4. **손으로 센서를 감싸면** LED Strip 전체 색상이 서서히 변하는 것을 관찰하세요.

:::note[생각해보기]
손을 댔다가 뗄 때, LED 색상이 바뀌는 데 몇 초 걸리나요? 센서가 주변 온도에 다시 적응하는 시간을 관찰해보세요.
:::

### Step 4. 막대 온도계 도전하기 (20분)

[도전 코드](#도전-코드-60-led-막대-온도계)를 실행하여 온도에 비례해 LED가 채워지는 막대 그래프를 만들어보세요.

### 기본 코드

```python
from machine import I2C, Pin
from neopixel import NeoPixel
import bme280
import time

# --- 하드웨어 설정 ---
i2c = I2C(1, sda=Pin(6), scl=Pin(7))    # Grove I2C1
bme = bme280.BME280(i2c=i2c)
NUM_LEDS = 60
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))  # Grove D16

# --- 핵심 함수: 값 범위 변환 ---
def map_value(value, in_min, in_max, out_min, out_max):
    """값을 한 범위에서 다른 범위로 비례 변환"""
    value = max(in_min, min(in_max, value))
    return int((value - in_min) * (out_max - out_min) / (in_max - in_min) + out_min)

# --- 온도 → RGB 색상 변환 ---
def temp_to_color(temp_c):
    """15°C=파랑, 25°C=초록, 35°C=빨강"""
    if temp_c <= 25:
        r = 0
        g = map_value(temp_c, 15, 25, 0, 255)
        b = map_value(temp_c, 15, 25, 255, 0)
    else:
        r = map_value(temp_c, 25, 35, 0, 255)
        g = map_value(temp_c, 25, 35, 255, 0)
        b = 0
    return (r, g, b)

# --- 전체 LED를 한 색으로 ---
def set_all(color):
    for i in range(NUM_LEDS):
        np[i] = color
    np.write()

# --- 메인 루프 ---
print("=== 온도 시각화 조명 ===")

while True:
    temp_str = bme.values[0]
    temp_c = float(temp_str[:-1])

    color = temp_to_color(temp_c)
    set_all(color)

    print(f"온도: {temp_c:.1f}°C → RGB{color}")
    time.sleep(1)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 7 | `I2C(1, sda=Pin(6), scl=Pin(7))` | Grove Shield의 I2C1 포트입니다. BME280은 I2C 방식으로 통신합니다. |
| 10 | `NeoPixel(Pin(16), NUM_LEDS, timing=(...))` | D16 포트에 연결된 60개 LED를 제어합니다. WS2813 전용 타이밍을 지정합니다. |
| 14 | `max(in_min, min(in_max, value))` | 입력값이 범위를 벗어나면 최소/최대로 제한합니다. 안전장치 역할입니다. |
| 20 | `temp_c <= 25` | 25°C를 기준으로 저온 구간(파랑→초록)과 고온 구간(초록→빨강)을 나눕니다. |
| 35 | `temp_str[:-1]` | BME280이 `"23.4C"` 형태로 반환하므로, 마지막 글자(C)를 제거하고 숫자로 변환합니다. |

### 도전 코드: 60 LED 막대 온도계

온도에 비례하여 LED가 채워지는 막대 그래프입니다. 위치에 따라 파랑→초록→빨강 그라데이션이 표시됩니다.

```python
from machine import I2C, Pin
from neopixel import NeoPixel
import bme280
import time

i2c = I2C(1, sda=Pin(6), scl=Pin(7))
bme = bme280.BME280(i2c=i2c)
NUM_LEDS = 60
np = NeoPixel(Pin(16), NUM_LEDS, timing=(280, 515, 515, 745))

TEMP_MIN = 15.0
TEMP_MAX = 35.0

def map_value(value, in_min, in_max, out_min, out_max):
    value = max(in_min, min(in_max, value))
    return int((value - in_min) * (out_max - out_min) / (in_max - in_min) + out_min)

def update_bar(temp_c):
    """온도에 따라 LED 막대 그래프 표시"""
    num_on = map_value(temp_c, TEMP_MIN, TEMP_MAX, 0, NUM_LEDS)

    for i in range(NUM_LEDS):
        if i < num_on:
            ratio = i / NUM_LEDS
            if ratio < 0.5:
                r = 0
                g = int(255 * ratio * 2)
                b = int(255 * (1 - ratio * 2))
            else:
                r = int(255 * (ratio - 0.5) * 2)
                g = int(255 * (1 - (ratio - 0.5) * 2))
                b = 0
            np[i] = (r, g, b)
        else:
            np[i] = (0, 0, 0)
    np.write()

print("=== LED 막대 온도계 ===")

while True:
    temp_str = bme.values[0]
    temp_c = float(temp_str[:-1])

    update_bar(temp_c)
    num_on = map_value(temp_c, TEMP_MIN, TEMP_MAX, 0, NUM_LEDS)
    print(f"{temp_c:.1f}°C → LED {num_on}개 점등")
    time.sleep(0.5)
```

:::note[코드 읽어보기]
`ratio`는 각 LED의 위치를 0~1 사이 비율로 나타낸 것입니다. 절반(0.5)을 기준으로 앞쪽은 파랑→초록, 뒤쪽은 초록→빨강으로 변합니다. `TEMP_MIN`과 `TEMP_MAX` 값을 여러분의 교실 환경에 맞게 조절해보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

공기 중의 열에너지가
**→** BME280 센서에서 **디지털 온도 데이터**로 바뀌고
**→** `map_value` 함수가 온도를 **RGB 색상 값**으로 변환하고
**→** 60개 LED가 그 색상으로 **동시에 빛나며**
**→** 보이지 않는 온도가 눈에 보이는 **색상**이 되었습니다.

기상청 일기예보 지도의 색깔, 열화상 카메라의 화면, 건물 에너지 효율 등급 표시 — 모두 숫자 데이터를 색상으로 **매핑**하는 시각화 기법입니다. 여러분은 방금 그 원리를 직접 60개의 LED로 구현한 것입니다.

:::note[매핑의 핵심]
같은 온도 데이터도 매핑 방식에 따라 전혀 다른 결과가 됩니다. 온도를 색상으로 바꾸면 시각화, LED 개수로 바꾸면 막대 그래프, 소리 높낮이로 바꾸면 청각화(sonification)가 됩니다. **"무엇을 무엇으로 바꿀 것인가"** — 이것이 데이터 시각화의 핵심 질문입니다.

## 마무리

### 오늘 배운 것

- **매핑(mapping)**: `map_value` 함수로 값의 범위를 변환하는 방법을 배웠습니다. 온도 15~35°C를 색상 0~255로 비례 변환했습니다.
- **온도→색상 변환**: 파랑(추움)→초록(보통)→빨강(더움)의 색상 그라데이션을 코드로 구현했습니다.
- **대형 시각화**: 60개 LED를 활용하여 눈에 띄는 크기의 시각화 출력을 만들어봤습니다.
