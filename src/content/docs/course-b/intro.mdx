## 코스 B의 철학

코스 A에서 여러분은 선생님이 정해준 센서를 연결하고, 주어진 코드를 따라 쳤어요.

코스 B에서는 **여러분이 결정해요.**

- **무엇**을 측정할지 여러분이 정해요
- **어떻게** 만들지 여러분이 설계해요
- **왜** 그것이 의미 있는지 여러분이 설명해요

실패해도 괜찮아요. 오히려 실패에서 더 많이 배워요. 코스 B의 목표는 "완벽한 결과물"이 아니라 **"스스로 시도한 경험"**이에요.

## 코스 B 진행 방식

### 시간 구성 (레벨당 3시간)

| 시간 | 활동 |
|------|------|
| 0:00~0:30 | 프로젝트 선택 및 계획 |
| 0:30~2:00 | 개발 (센서 연결 + 코드 작성 + 대시보드 + 테스트) |
| 2:00~2:30 | 데이터 수집 및 정리 |
| 2:30~3:00 | 산출물 정리 및 발표 준비 |

### 산출물

각 레벨을 완료하면 다음 산출물을 제출해요:

**모든 레벨 공통:**
- **작동하는 코드** -- Pico에서 실행되는 main.py
- **대시보드 스크린샷** -- OLED 또는 PC 대시보드
- **한 줄 설명** -- "이 프로젝트는 ___을(를) 측정하여 ___을(를) 알 수 있는 시스템이에요"

**Lv.2 이상 추가:**
- **바이브 코딩 로그** -- AI에게 요청한 프롬프트와 결과

**Lv.3 이상 추가:**
- **데이터 분석 결과** -- 그래프 + 해석
- **가설 검증 보고서** -- 가설, 데이터, 결론

**Lv.4 추가:**
- **ML 모델 결과** -- 학습 데이터, 정확도, 활용 방안

### 평가 기준

| 기준 | 설명 | 비중 |
|------|------|------|
| **작동** | 코드가 에러 없이 실행되고 대시보드가 동작하는가? | 30% |
| **정확** | 센서 데이터가 합리적이고 대시보드 표시가 정확한가? | 20% |
| **설계** | 코드가 구조적이고, 대시보드가 사용하기 쉬운가? | 20% |
| **창의** | 독창적인 아이디어나 응용이 있는가? | 15% |
| **설명** | 프로젝트를 명확하게 설명할 수 있는가? | 15% |
