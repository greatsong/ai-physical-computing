# 용어 사전

> **모르는 용어가 나오면 여기서 찾아보세요.**

용어는 가나다순으로 정렬되어 있어요.

---

## ㄱ

### GPIO (General Purpose Input/Output)
범용 입출력 핀이에요. Pico에는 26개의 GPIO 핀이 있고, 각 핀을 입력(센서 읽기) 또는 출력(LED 켜기)으로 사용할 수 있어요. Grove Shield가 이 핀들을 Grove 포트로 변환해줘요.

### Grove Shield
Pico 2 WH 위에 장착하는 확장 보드예요. I2C, ADC, Digital, UART 포트를 제공하여, Grove 케이블로 센서를 쉽게 연결할 수 있게 해줘요. 납땜이나 점퍼선 없이 센서를 연결할 수 있어요.

---

## ㄴ

### 노이즈 (Noise)
센서 값에 섞이는 원하지 않는 변동이에요. 예를 들어, 온도 센서가 24.0, 24.3, 23.8, 24.1처럼 흔들리면 이 변동이 노이즈예요. 이동 평균 등의 필터로 줄일 수 있어요.

---

## ㄷ

### 대시보드 (Dashboard)
중요한 정보를 한 화면에 모아서 보기 좋게 표시하는 인터페이스예요. 자동차 계기판, 스마트폰 날씨 앱이 대시보드의 예시예요. 이 교재에서는 OLED 화면이나 PC 화면에 센서 데이터를 실시간으로 표시하는 것을 대시보드라고 불러요.

### 데이터 (Data)
센서가 측정한 값이에요. 숫자, 텍스트, 이미지 등 다양한 형태가 있어요. 이 교재에서는 주로 센서가 출력하는 숫자 데이터를 다뤄요. 데이터 자체는 의미가 없고, 분석과 시각화를 통해 정보가 돼요.

### 딕셔너리 (Dictionary)
Python의 자료구조 중 하나로, `"키": 값` 쌍으로 데이터를 저장해요. `{"name": "DHT20", "port": "I2C"}`처럼 사용해요. 센서 정보, 프로젝트 설정 등 구조화된 데이터를 다룰 때 유용해요.

---

## ㄹ

### 라벨링 (Labeling)
데이터에 이름표를 붙이는 작업이에요. 머신러닝에서 학습 데이터를 만들 때 필요해요. 예를 들어, 소리 데이터에 "박수", "노크", "말소리"라는 라벨을 붙이면, 컴퓨터가 이 라벨을 기준으로 패턴을 학습해요.

### 리스트 (List)
Python의 자료구조 중 하나로, 여러 값을 순서대로 담아요. `[23.1, 24.0, 23.5]`처럼 대괄호로 만들어요. 센서 데이터를 모아두거나, 여러 개의 값을 한 번에 처리할 때 사용해요.

---

## ㅁ

### 머신러닝 (Machine Learning, ML)
데이터에서 패턴을 찾아 새로운 데이터를 예측하는 기술이에요. 사람이 규칙을 직접 만드는 대신, 컴퓨터가 데이터에서 규칙을 자동으로 학습해요. 이 교재에서는 Google Teachable Machine을 사용해요.

### MicroPython
마이크로컨트롤러(Pico 등)에서 실행되는 Python이에요. 일반 Python과 문법은 같지만, `machine` 모듈로 하드웨어를 제어할 수 있어요. `matplotlib` 같은 PC 전용 라이브러리는 사용할 수 없어요.

---

## ㅂ

### 바이브 코딩 (Vibe Coding)
AI에게 자연어로 원하는 프로그램을 설명하면, AI가 코드를 생성해주는 프로그래밍 방식이에요. 예를 들어, "OLED에 온도를 막대그래프로 표시해줘"라고 요청하면 AI가 코드를 작성해요. 생성된 코드를 이해하고 수정할 수 있는 능력이 중요해요.

### 변인 (Variable, 실험 설계에서)
실험에서 변하는 요소예요.
- **독립 변인**: 실험자가 의도적으로 바꾸는 것 (예: 측정 장소)
- **종속 변인**: 독립 변인에 따라 변하는 것 (예: 온도)
- **통제 변인**: 같게 유지해야 하는 것 (예: 측정 시간)

---

## ㅅ

### 시각화 (Visualization)
데이터를 그래프, 차트, 대시보드 등 시각적 형태로 표현하는 것이에요. 숫자 나열보다 패턴을 발견하기 쉬워요. matplotlib, OLED 그래프 등이 시각화 도구예요.

### Serial 통신
USB 케이블을 통해 Pico와 PC 사이에서 데이터를 주고받는 방식이에요. Pico에서 `print()`로 출력한 내용이 serial 통신으로 PC에 전달돼요. PC에서는 `pyserial` 라이브러리로 이 데이터를 받을 수 있어요.

---

## ㅇ

### ADC (Analog-to-Digital Converter)
아날로그 신호(연속적인 전압)를 디지털 값(숫자)으로 변환하는 장치예요. Pico의 ADC는 0~3.3V 전압을 0~65535 사이의 숫자로 바꿔요. 빛 센서, 소리 센서 등 아날로그 센서에 사용해요.

### I2C (Inter-Integrated Circuit)
디지털 통신 규격이에요. SDA(데이터)와 SCL(클럭) 두 선만으로 여러 장치와 통신할 수 있어요. 각 장치는 고유한 주소(예: 0x38)를 가지고 있어요. 온습도 센서, OLED, 가속도 센서 등이 I2C를 사용해요.

### IoT (Internet of Things, 사물인터넷)
센서와 장치들이 인터넷에 연결되어 데이터를 주고받는 기술이에요. 스마트홈, 스마트시티, 산업 IoT 등이 있어요. 이 교재에서 만드는 센서+대시보드 시스템이 IoT의 기초 구조예요.

### 이동 평균 (Moving Average)
최근 N개의 값의 평균을 계산하여 노이즈를 줄이는 방법이에요. 예를 들어, 최근 10개의 센서 값을 평균 내면 급격한 튐 현상이 완화돼요. 주식 차트의 이동 평균선도 같은 원리예요.

### 이상치 (Outlier)
다른 값들과 크게 동떨어진 비정상적인 값이에요. 센서 오류, 일시적 장애 등으로 발생해요. 분석 전에 이상치를 제거하거나 표시해야 정확한 결과를 얻을 수 있어요.

---

## ㅈ

### 조건문 (if/elif/else)
조건에 따라 다른 코드를 실행하는 Python 문법이에요. 센서 값이 기준보다 높으면 경고하고, 그렇지 않으면 정상이라고 표시하는 등의 판단에 사용해요.

---

## ㅊ

### CSV (Comma-Separated Values)
쉼표로 값을 구분하는 텍스트 파일 형식이에요. `온도,습도,시간`처럼 각 줄에 데이터를 저장해요. 엑셀, Google Sheets, Python 등 거의 모든 도구에서 읽을 수 있어서 데이터 교환에 널리 사용돼요.

---

## ㅍ

### 펌웨어 (Firmware)
하드웨어에 내장된 소프트웨어예요. Pico에 MicroPython 펌웨어(.uf2 파일)를 설치하면 Python 코드를 실행할 수 있게 돼요. 펌웨어는 보통 한 번만 설치하면 돼요.

### PWM (Pulse Width Modulation)
디지털 신호의 켜짐/꺼짐 비율을 조절하여 아날로그처럼 동작하게 하는 기술이에요. LED 밝기를 단계적으로 조절하거나, 부저의 음높이를 바꿀 때 사용해요.

---

## ㅎ

특정 작업을 하는 코드 블록이에요. `def` 키워드로 정의하고, 이름으로 호출해요. 같은 작업을 여러 번 반복할 때 함수로 만들면 코드가 깔끔해져요.

```python
def read_temperature():
    # 온도를 읽는 코드
```

## 영문

### Teachable Machine
Google이 만든 브라우저 기반 ML 도구예요. 코딩 없이 이미지, 소리, 자세를 분류하는 ML 모델을 만들 수 있어요. [teachablemachine.withgoogle.com](https://teachablemachine.withgoogle.com)에서 사용해요.

### Thonny
MicroPython 개발에 적합한 IDE(통합 개발 환경)예요. Pico에 코드를 전송하고 실행하고 디버깅할 수 있어요. 파일 관리, 패키지 설치 등의 기능도 제공해요.

### try/except
Python의 에러 처리 문법이에요. `try` 블록 안의 코드에서 에러가 발생하면, 프로그램이 멈추지 않고 `except` 블록의 코드를 실행해요. 장기 측정에서 센서 오류를 처리할 때 필수예요.
