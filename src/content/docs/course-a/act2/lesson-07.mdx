## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH + Grove Shield | 하드웨어 (1막과 동일) |
| 심박 센서 (Pulse Sensor) | 손가락 끝의 혈류 변화를 감지하는 아날로그 센서입니다. A0 포트에 연결합니다. ([센서 도감에서 자세히 보기](/code-human-textbook/appendix/sensor-catalog/)) |
| OLED 디스플레이 (SSD1306) | 128x64 픽셀의 작은 화면. I2C 포트에 연결합니다. |
| AI 챗봇 | Claude, ChatGPT 등 — 바이브 코딩에 사용 |

### 연결 방법

- **심박 센서**: Grove 케이블로 Shield의 **A0 포트**(아날로그, GP26)에 연결
- **OLED**: Grove 케이블로 Shield의 **I2C 포트**(SDA=GP8, SCL=GP9)에 연결

:::tip[센서 사용법]
센서의 감지면(LED가 있는 쪽)에 **검지 손가락 끝**을 가볍게 올려놓으세요. 너무 세게 누르면 혈류가 막혀 신호가 사라집니다.
:::

:::note[주의사항]
- 손가락을 센서에 올린 상태에서 **최대한 움직이지 마세요.** 움직임은 노이즈가 됩니다.
- 손이 차가우면 혈류가 약해 신호가 잘 안 잡힐 수 있습니다. 손을 따뜻하게 비비고 시작하세요.
- 이 센서는 의료용이 아닙니다. **변화의 패턴**을 관찰하는 것이 목적입니다.
:::

## 활동 가이드

### Step 1. AI에게 심박 대시보드 코드 요청하기 (15분)

1. AI 챗봇을 열고 [바이브 코딩 프롬프트](#바이브-코딩-프롬프트) 탭의 예시를 참고하여 프롬프트를 보냅니다.
2. AI가 생성한 코드를 읽어봅니다 — ADC 설정, 피크 감지 로직, OLED 출력 부분을 확인하세요.
3. Thonny에 붙여넣고 실행합니다.
4. **검지 손가락**을 센서에 올리고, OLED에 파형이 나타나는지 확인합니다.

:::tip[코드 리뷰 체크포인트]
- ADC 설정이 `ADC(Pin(26))`인가요? (A0 포트)
- 샘플링 간격이 20ms 정도인가요? (심박 파형을 잡으려면 빠르게 읽어야 합니다)
- 피크 감지에 `THRESHOLD`가 있나요? 본인의 센서값에 맞게 조정이 필요합니다.
:::

### Step 2. 안정 시 심박 측정 (10분)

1. 편안하게 앉은 상태에서 **1분간** 측정합니다.
2. OLED 대시보드에 표시되는 BPM 값을 기록합니다.
3. 파형 그래프가 **규칙적으로 오르락내리락**하는지 확인합니다.
4. BPM이 불안정하면 `THRESHOLD` 값을 조절해보세요.

:::note[BPM이 이상하게 나온다면]
피크 감지의 `THRESHOLD` 값은 사람마다 다릅니다. 기본 코드 탭에서 먼저 raw 값을 확인한 뒤, 피크와 골의 중간값을 `THRESHOLD`로 설정하세요. AI에게 "THRESHOLD를 자동으로 보정하는 기능을 추가해줘"라고 요청해볼 수도 있습니다.
:::

### Step 3. 운동 후 심박 비교 (15분)

1. **제자리 뛰기를 30초간** 합니다.
2. 바로 돌아와서 센서에 손가락을 올리고 측정합니다.
3. **운동 직후 BPM**을 기록합니다.
4. 그대로 앉아서 **1분 후**, **2분 후** BPM을 다시 기록합니다.

| 상태 | BPM | OLED 파형 특징 | 비고 |
|------|-----|--------------|------|
| 안정 시 | | 규칙적, 느린 간격 | |
| 운동 직후 | | 빠른 간격 | |
| 1분 회복 후 | | | |
| 2분 회복 후 | | | |

### Step 4. 토론하기 (10분)

1. **"안정 시와 운동 후 BPM 차이는 얼마인가요?"** 모둠원끼리 비교해봅시다.

### AI에게 보낼 프롬프트

```
Pico 2 WH에 Grove Pulse Sensor(심박 센서)를 A0 포트(GP26)에,
OLED SSD1306을 I2C0(SDA=GP8, SCL=GP9)에 연결했어.

다음 기능을 하는 MicroPython 코드를 만들어줘:

1. 심박 센서를 20ms 간격으로 빠르게 읽기 (ADC)
2. 피크 감지 알고리즘으로 심장 박동 횟수를 세기
3. 10초마다 BPM(분당 심박수)을 계산
4. OLED 대시보드에 표시:
   - 상단: 실시간 심박 파형 그래프 (최근 64개 샘플을 그래프로)
   - 하단: BPM 숫자 + 상태(정상 60~100, 빠름 100+, 느림 60-)
5. Shell에도 BPM을 출력

피크 감지는 THRESHOLD 값을 넘을 때 카운트하고,
같은 피크를 두 번 세지 않도록 MIN_GAP(300ms) 설정해줘.
```

:::tip[프롬프트 수정이 필요할 때]
AI가 생성한 코드의 THRESHOLD 값이 맞지 않으면, 추가로 이렇게 요청하세요:
"처음 3초간 센서값을 읽어서 평균을 구한 뒤, 평균 + 3000을 THRESHOLD로 자동 설정하는 기능을 추가해줘"
:::

### 기본 코드

AI가 생성할 수 있는 완성 코드 예시입니다.

```python
from machine import ADC, Pin, I2C
import ssd1306
import time

# 심박 센서 (A0 포트)
pulse = ADC(Pin(26))

# OLED 설정
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
oled = ssd1306.SSD1306_I2C(128, 64, i2c, addr=0x3C)

# 피크 감지 설정
THRESHOLD = 45000   # 피크 판정 기준 (환경에 맞게 조절)
MIN_GAP = 300       # 피크 간 최소 간격 (ms)

# 파형 그래프용 버퍼 (최근 128개 샘플)
wave_buf = [32] * 128  # OLED 높이 64의 중간값으로 초기화

# 심박 카운트 변수
beat_count = 0
last_beat = 0
above = False
start_time = time.ticks_ms()
bpm = 0

def get_status(bpm):
    if bpm == 0:
        return "..."
    elif bpm < 60:
        return "SLOW"
    elif bpm <= 100:
        return "NORMAL"
    else:
        return "FAST"

def draw_dashboard():
    oled.fill(0)

    # 상단: 파형 그래프 (y: 0~39, 40픽셀 높이)
    for x in range(1, len(wave_buf)):
        y1 = wave_buf[x - 1]
        y2 = wave_buf[x]
        # 0~39 범위로 제한
        y1 = max(0, min(39, y1))
        y2 = max(0, min(39, y2))
        oled.line(x - 1, y1, x, y2, 1)

    # 구분선
    oled.hline(0, 42, 128, 1)

    # 하단: BPM + 상태
    status = get_status(bpm)
    oled.text(f"BPM: {bpm}", 0, 48)
    oled.text(f"[{status}]", 80, 48)

    oled.show()

print("심박 대시보드 시작... 손가락을 센서에 올려주세요.")

sample_count = 0
while True:
    value = pulse.read_u16()
    now = time.ticks_ms()

    # 파형 버퍼 업데이트 (65535 → 0~39 범위로 매핑)
    y = 39 - int(value / 65535 * 39)
    wave_buf.pop(0)
    wave_buf.append(y)

    # 피크 감지
    if value > THRESHOLD and not above:
        if time.ticks_diff(now, last_beat) > MIN_GAP:
            beat_count += 1
            last_beat = now
            above = True
    elif value < THRESHOLD - 2000:
        above = False

    # 10초마다 BPM 계산
    elapsed = time.ticks_diff(now, start_time)
    if elapsed >= 10000:
        bpm = beat_count * 6  # 10초 x 6 = 60초
        print(f"BPM: {bpm} ({get_status(bpm)})")
        beat_count = 0
        start_time = now

    # 5회 샘플링마다 화면 갱신 (너무 자주 그리면 느려짐)
    sample_count += 1
    if sample_count >= 5:
        draw_dashboard()
        sample_count = 0

    time.sleep(0.02)  # 20ms 간격
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 6 | `ADC(Pin(26))` | A0 포트(GPIO 26번)를 아날로그 입력으로 설정합니다. |
| 14 | `THRESHOLD = 45000` | 이 값을 넘으면 "심장이 뛴 것"으로 판단합니다. 사람마다 다르므로 조절이 필요합니다. |
| 18 | `wave_buf = [32] * 128` | OLED 가로 128픽셀에 대응하는 파형 버퍼입니다. 새 값이 들어오면 가장 오래된 값이 빠집니다. |
| 64-65 | `y = 39 - int(...)` | ADC 값(0~65535)을 OLED 그래프 높이(0~39)로 변환합니다. |
| 77 | `bpm = beat_count * 6` | 10초간의 피크 수에 6을 곱하면 60초(1분)의 추정 BPM이 됩니다. |

### 도전 코드: 운동 전후 BPM 데이터 CSV 저장

BPM 데이터를 CSV 파일로 저장하여 운동 전후 비교에 활용합니다.

```python
from machine import ADC, Pin, I2C
import ssd1306
import time

pulse = ADC(Pin(26))
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
oled = ssd1306.SSD1306_I2C(128, 64, i2c, addr=0x3C)

THRESHOLD = 45000
MIN_GAP = 300

beat_count = 0
last_beat = 0
above = False
start_time = time.ticks_ms()
bpm = 0
bpm_history = []

# CSV 파일 생성
filename = "heartbeat.csv"
f = open(filename, "w")
f.write("elapsed_sec,bpm,status\n")

TOTAL_TIME = 180  # 3분간 측정 (초)
program_start = time.ticks_ms()

print(f"심박 측정 시작 ({TOTAL_TIME}초간 기록)")
print("1단계: 안정 시 (0~60초)")
print("2단계: 운동 후 (60~120초)")
print("3단계: 회복 (120~180초)")

def get_status(bpm):
    if bpm == 0:
        return "wait"
    elif bpm < 60:
        return "slow"
    elif bpm <= 100:
        return "normal"
    else:
        return "fast"

while True:
    value = pulse.read_u16()
    now = time.ticks_ms()
    total_elapsed = time.ticks_diff(now, program_start) // 1000

    # 종료 조건
    if total_elapsed >= TOTAL_TIME:
        break

    # 단계 안내
    if total_elapsed == 60:
        oled.fill(0)
        oled.text("RUN NOW!", 20, 28)
        oled.show()
    elif total_elapsed == 90:
        oled.fill(0)
        oled.text("Come back!", 16, 28)
        oled.show()

    # 피크 감지
    if value > THRESHOLD and not above:
        if time.ticks_diff(now, last_beat) > MIN_GAP:
            beat_count += 1
            last_beat = now
            above = True
    elif value < THRESHOLD - 2000:
        above = False

    # 10초마다 BPM 계산 + CSV 저장
    elapsed = time.ticks_diff(now, start_time)
    if elapsed >= 10000:
        bpm = beat_count * 6
        status = get_status(bpm)
        bpm_history.append(bpm)

        # CSV에 기록
        f.write(f"{total_elapsed},{bpm},{status}\n")
        f.flush()

        # OLED 표시
        oled.fill(0)
        phase = "Rest" if total_elapsed < 60 else "Run" if total_elapsed < 120 else "Recovery"
        oled.text(f"[{phase}]", 0, 0)
        oled.text(f"BPM: {bpm}", 0, 16)
        oled.text(f"Time: {total_elapsed}s", 0, 32)
        oled.text(f"[{status.upper()}]", 0, 48)
        oled.show()

        print(f"{total_elapsed}s | BPM:{bpm} | {phase} | {status}")
        beat_count = 0
        start_time = now

    time.sleep(0.02)

f.close()

# 결과 요약
print("\n=== 측정 완료 ===")
print(f"저장 파일: {filename}")
print(f"기록 수: {len(bpm_history)}")
if bpm_history:
    print(f"평균 BPM: {sum(bpm_history) // len(bpm_history)}")
    print(f"최고 BPM: {max(bpm_history)}")
    print(f"최저 BPM: {min(bpm_history)}")

oled.fill(0)
oled.text("DONE!", 40, 16)
oled.text(f"Avg: {sum(bpm_history)//len(bpm_history)} BPM", 0, 32)
oled.text(f"Saved: {filename}", 0, 48)
oled.show()
```

:::note[코드 읽어보기]
이 코드는 3분간 3단계로 심박을 측정합니다: 안정(0~60초) → 운동(60~120초) → 회복(120~180초). 60초가 되면 OLED에 "RUN NOW!"가 표시되고, 90초가 되면 "Come back!"이 표시됩니다. CSV 파일에 10초마다 BPM이 기록되므로, 나중에 9차시 데이터 분석에 활용할 수 있습니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

AI에게 **"심박 대시보드 만들어줘"**라고 프롬프트를 보내고
**→** AI가 **ADC 읽기 + 피크 감지 + OLED 그래프** 코드를 생성하고
**→** 여러분이 **THRESHOLD를 조절**하고
**→** 심장이 뛸 때마다 **혈류 변화가 빛의 반사량을 바꾸고**
**→** ADC가 그 변화를 **숫자로 변환**하고
**→** 코드가 피크를 세어 **BPM을 계산**하고
**→** OLED에 **파형과 숫자가 실시간으로** 표시되었습니다.

병원의 맥박산소측정기, 스마트워치의 심박 센서 — 모두 같은 **광용적맥파(PPG)** 원리를 사용합니다. 여러분은 AI와 협력하여 이 원리를 직접 구현한 것입니다.

:::note[증가율 계산법]
