## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico WH + Grove Shield | 하드웨어 (1막과 동일) |
| 온습도 센서 (DHT20) | I2C 포트에 연결 |
| AI 챗봇 | Claude, ChatGPT 등 — 웹 브라우저에서 접속 |
| Thonny | AI가 생성한 코드를 실행하는 도구 |

:::note[바이브 코딩이란?]
**바이브 코딩(Vibe Coding)**은 AI에게 자연어로 원하는 기능을 설명하면, AI가 코드를 작성해주는 방식입니다. 2025년부터 전 세계 개발자들이 적극적으로 활용하고 있습니다. 코드를 한 줄도 모르는 사람도, 프롬프트만 잘 쓰면 작동하는 프로그램을 만들 수 있습니다.

하지만 **AI가 쓴 코드를 읽고, 이해하고, 수정할 수 있어야** 진짜 실력입니다. 1막에서 직접 코딩한 경험이 여기서 빛을 발합니다.
:::

## 활동 가이드

### Step 1. 첫 번째 바이브 코딩 (15분)

AI 챗봇을 열고, 아래 프롬프트를 입력하세요:

```
Raspberry Pi Pico WH에 Grove Shield를 사용하고 있어.
Grove DHT20 온습도 센서를 I2C0 포트(SDA=GP8, SCL=GP9)에 연결했어.
2초마다 온도와 습도를 측정해서 출력하는 MicroPython 코드를 만들어줘.
```

1. AI가 생성한 코드를 **Thonny에 붙여넣고 실행**합니다.
2. 1차시에서 직접 작성한 코드와 비교해보세요.

:::tip[비교 포인트]
- AI 코드와 내 코드의 **구조**가 비슷한가요?
- AI는 어떤 **주석**을 달았나요?
- AI 코드에 **에러**가 있나요? 있다면 어떻게 고칠 수 있나요?
:::

### Step 2. 프롬프트 업그레이드 (15분)

이번에는 더 구체적인 요청을 해봅니다:

```
위 코드에 다음 기능을 추가해줘:
1. 온도가 28도 이상이면 "더워요!"라고 출력
2. 습도가 70% 이상이면 "습해요!"라고 출력
3. 측정 횟수를 세서 "N번째 측정" 형태로 표시
4. 10번 측정하면 자동으로 종료
```

1. AI가 생성한 코드를 실행합니다.
2. **의도대로 동작하지 않는 부분**이 있나요? 직접 수정해보세요.
3. 수정한 부분을 짝과 공유합니다.

### Step 3. 나만의 프롬프트 도전 (15분)

아래 미션 중 하나를 골라 프롬프트를 작성하고, AI에게 코드를 받아 실행하세요.

| 미션 | 설명 | 힌트 프롬프트 |
|------|------|-------------|
| A. 쾌적도 판정기 | 온도+습도 조합으로 쾌적/보통/불쾌 판정 | "온도 20~26, 습도 40~60이면 쾌적..." |
| B. 최고/최저 기록기 | 측정하면서 최고/최저 온도를 갱신 | "지금까지의 최고 온도와 최저 온도를..." |
| C. CSV 파일 저장 | 측정 데이터를 파일로 저장 | "data.csv 파일에 시간, 온도, 습도를..." |

:::note[좋은 프롬프트의 조건]
1. **하드웨어 정보**를 알려주세요: 어떤 보드, 어떤 센서, 어떤 포트
2. **원하는 기능**을 구체적으로 설명하세요
3. **제약 조건**이 있으면 추가하세요: "MicroPython으로", "라이브러리 없이" 등
4. 한 번에 너무 많이 요청하지 마세요 — **단계적으로** 추가하는 게 좋습니다
:::

### Step 4. 토론: 바이브 코딩의 장단점 (5분)

모둠에서 이야기해봅시다:

1. **바이브 코딩의 장점**은 무엇인가요?
2. **바이브 코딩의 한계**는 무엇인가요?
3. **1막에서 직접 코딩한 경험**이 바이브 코딩에 어떻게 도움이 됐나요?

### AI가 생성할 수 있는 코드 (예시)

아래는 "온습도를 측정하고 쾌적도를 판정해줘"라는 프롬프트에 대한 AI 응답 예시입니다:

```python
from machine import I2C, Pin
import time

# Grove Shield I2C0 포트 설정
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
DHT20_ADDR = 0x38

time.sleep(0.1)  # 센서 초기화 대기

def read_dht20():
    """DHT20에서 온도와 습도를 읽어 반환합니다"""
    i2c.writeto(DHT20_ADDR, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(DHT20_ADDR, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

def judge_comfort(temp, humi):
    """온도와 습도를 기반으로 쾌적도를 판정합니다"""
    if 20 <= temp <= 26 and 40 <= humi <= 60:
        return "쾌적"
    elif temp > 30 or humi > 80:
        return "불쾌"
    elif temp < 18 or humi < 30:
        return "건조/추움"
    else:
        return "보통"

# 메인 루프
count = 0
while True:
    count += 1
    temp, humi = read_dht20()
    comfort = judge_comfort(temp, humi)

    print(f"[{count}] 온도: {temp}°C, 습도: {humi}% → {comfort}")
    time.sleep(2)
```

:::note[AI 코드 읽기 연습]
이 코드를 보면서 확인해보세요:
1. `judge_comfort` 함수의 조건 기준이 적절한가요?
2. 온도 18~20도, 습도 30~40%인 경우는 어떤 판정을 받나요?
3. 기준을 바꾸고 싶다면 어디를 수정해야 하나요?
:::

### 비교: 같은 기능, 다른 과정

| 항목 | 직접 코딩 (1막) | 바이브 코딩 (2막) |
|------|---------------|-----------------|
| 시작 | 빈 파일을 열고 한 줄씩 작성 | AI에게 프롬프트로 설명 |
| 시간 | 코드 구조부터 설계 (15~30분) | 프롬프트 작성 후 즉시 생성 (2~5분) |
| 이해도 | 모든 줄을 직접 써서 깊이 이해 | 생성된 코드를 읽으며 이해 |
| 디버깅 | 자기가 쓴 코드라 원인 파악 쉬움 | AI 코드의 버그는 찾기 어려울 수 있음 |
| 확장 | 구조를 알아야 확장 가능 | "이것도 추가해줘"로 빠르게 확장 |
| **핵심 역량** | 코드 작성 능력 | 프롬프트 작성 + 코드 리뷰 능력 |

:::tip[둘 다 필요합니다]
직접 코딩을 할 줄 알아야 AI가 생성한 코드를 이해하고 수정할 수 있습니다. 바이브 코딩만으로는 AI가 틀렸을 때 고칠 수 없습니다. **1막의 직접 코딩 경험이 2막의 바이브 코딩을 더 강하게** 만들어줍니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분은 방금 **말로 코드를 만들었습니다**.

자연어 설명
**→** AI가 코드로 변환
**→** Pico에서 실행
**→** 센서 데이터 출력

이것이 2025년 이후 소프트웨어 개발의 현실입니다. 전문 개발자들도 AI에게 코드를 시키고, 리뷰하고, 수정합니다. 여러분은 방금 그 과정을 직접 경험한 것입니다.

하지만 기억하세요 — AI는 **도구**이고, **무엇을 만들지 결정하고, 결과를 검증하는 것은 여러분**입니다.
