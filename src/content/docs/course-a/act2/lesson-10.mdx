## 준비물

| 준비물 | 설명 |
|--------|------|
| 컴퓨터 | 웹 브라우저와 Python이 사용 가능한 PC |
| 이전 차시 데이터 | 6~9차시에서 수집하고 정리한 센서 데이터. 없다면 아래 샘플 데이터를 사용합니다. |

:::tip[AI 챗봇 준비]
이 차시에서 AI는 **데이터를 분석하는 도구**로 사용합니다. 웹 브라우저에서 AI 챗봇 사이트에 접속하세요. 학교에서 허용된 서비스를 사용합니다. 선생님의 안내를 따라주세요.
:::

:::note[바이브 코딩과 AI 분석의 차이]
5~8차시의 바이브 코딩에서는 AI에게 **코드를 만들어달라고** 요청했습니다. 이번 차시에서는 AI에게 **데이터를 분석해달라고** 요청합니다. 같은 AI지만, 요청하는 것이 다릅니다. **프롬프트를 잘 쓰는 능력**이 여기서도 핵심입니다.
:::

## 활동 가이드

### Step 1. 데이터를 AI에게 전달하기 (10분)

1. [기본 코드](#기본-코드) 탭의 코드를 PC Python에서 실행합니다.
2. 코드가 센서 데이터를 **AI에게 보여줄 수 있는 텍스트 형식**으로 정리합니다.
3. 출력된 텍스트를 **복사**합니다. (Ctrl+A로 전체 선택, Ctrl+C로 복사)
4. AI 챗봇에 접속합니다.

:::tip[자기 데이터 사용하기]
9차시에서 만든 CSV 파일이 있다면, 코드의 샘플 데이터 대신 자기 데이터를 사용하세요. AI에게 더 의미 있는 분석을 받을 수 있습니다.
:::

### Step 2. AI에게 분석 요청하기 (15분)

AI 챗봇에 데이터를 붙여넣고, 아래 프롬프트 예시를 참고하여 분석을 요청합니다.

#### 프롬프트 예시

**프롬프트 1: 패턴 찾기**
> 이 CSV 데이터는 교실 온습도 24시간 기록입니다. 패턴, 이상점, 개선 제안을 분석해줘.
>
> [여기에 데이터 붙여넣기]

**프롬프트 2: 환기 효과 분석**
> 아래 데이터에서 5분째에 환기(창문 열기)를 시작했습니다. 환기 전후의 변화를 분석하고, 환기의 효과를 수치로 설명해줘.
>
> [여기에 데이터 붙여넣기]

**프롬프트 3: 비교 분석**
> 아래는 두 교실의 CO2 데이터입니다. A교실은 환기를 했고, B교실은 하지 않았습니다. 두 교실의 차이를 분석해줘.
>
> [여기에 데이터 붙여넣기]

**프롬프트 4: 개선 제안**
> 아래 교실 환경 데이터를 바탕으로, 학생들의 집중력을 높이기 위한 환경 개선 방안을 구체적으로 제안해줘.
>
> [여기에 데이터 붙여넣기]

:::note[좋은 프롬프트의 조건]
AI에게 질문할 때는 **맥락을 함께 알려주는 것**이 중요합니다.
- "이 데이터를 분석해줘" (나쁜 예)
- "25명이 있는 교실에서 30초 간격으로 측정한 CO2 데이터입니다. 5분째에 창문을 열었습니다." (좋은 예)

**상황을 설명**할수록 AI가 더 의미 있는 분석을 합니다.
:::

### Step 3. AI 분석 결과 해석하기 (15분)

AI의 분석 결과를 읽고, 자기 분석과 비교합니다.

1. **AI가 찾은 패턴**을 기록합니다.
2. **AI가 맞게 분석한 것**과 **잘못 분석한 것**을 구분합니다.
3. **AI가 몰랐던 맥락**을 찾아봅니다. (교실 크기, 학생 수, 계절, 창문 위치 등)
4. AI에게 **추가 질문**을 해봅니다: "왜 그렇게 생각하나요?", "다른 해석은 없을까요?"

#### AI가 찾을 수 있는 것

| 분석 유형 | 예시 |
|----------|------|
| 시간대별 패턴 | "오전 10시에 CO2가 가장 높습니다" |
| 변화 추세 | "환기 후 5분 내에 CO2가 30% 감소했습니다" |
| 상관관계 | "CO2가 올라갈 때 온도도 함께 올라갑니다" |
| 이상값 지적 | "3번째 데이터(9999)는 센서 오류로 보입니다" |
| 개선 제안 | "40분마다 5분간 환기하면 CO2를 800ppm 이하로 유지할 수 있습니다" |

#### AI의 한계

| 한계 | 예시 |
|------|------|
| 맥락 모름 | 교실에 에어컨이 있는지, 창문이 어디인지 모름 |
| 잘못된 추론 | 데이터의 원인을 잘못 추측할 수 있음 |
| 확신에 찬 오류 | 틀린 분석을 자신 있게 말할 수 있음 |
| 검증 불가 | AI 스스로 자기 분석이 맞는지 확인할 수 없음 |

### Step 4. AI vs 나 비교 토론 (10분)

모둠에서 다음 질문을 토론합니다.

1. **"AI가 나보다 더 잘 분석한 것은 무엇인가요?"** (패턴 인식, 수치 계산 등)
2. **"내가 AI보다 더 잘 판단한 것은 무엇인가요?"** (맥락 이해, 상식적 판단 등)
3. **"AI가 잘못 분석했다면, 어떻게 알 수 있었나요?"** 검증의 중요성
4. **"데이터 수집 → 정리 → AI 분석"에서 각 단계의 주인공은 누구인가요?"**

### AI에게 보낼 분석 프롬프트

이번 차시의 프롬프트는 "코드를 만들어줘"가 아니라 **"데이터를 분석해줘"**입니다.

#### 기본 분석 프롬프트 구조

```
[맥락 설명]
25명이 수업하는 교실에서 SCD41 CO2 센서로 30초 간격으로 측정한 데이터입니다.
5분째에 창문 2개를 열어 환기를 시작했고, 8분 30초에 닫았습니다.

[데이터]
(여기에 데이터 붙여넣기)

[분석 요청]
1. 이 데이터에서 발견되는 패턴을 알려주세요.
2. 이상한 값이 있다면 지적해주세요.
3. 환기의 효과를 수치로 설명해주세요.
4. 교실 환경을 개선하기 위한 구체적인 제안을 해주세요.
```

#### 심화 분석 프롬프트

```
위 분석에 추가로:
1. CO2 농도와 온도의 상관관계가 있나요?
2. 환기를 몇 분마다, 몇 분간 하는 것이 최적일까요?
3. 이 데이터를 학교 관리자에게 보고한다면, 핵심 메시지는 무엇인가요?
```

:::tip[프롬프트 반복 실험]
같은 데이터를 **다른 프롬프트**로 물어보면 AI가 다른 관점의 분석을 합니다. 여러 번 질문해보고, 가장 유용한 분석을 선택하세요. AI에게 "이 분석이 틀릴 수 있는 이유를 알려줘"라고 물어보는 것도 좋은 방법입니다.
:::

### 기본 코드: 데이터 준비

AI에게 보여줄 데이터를 텍스트로 정리하는 코드입니다. **PC Python**에서 실행하세요.

```python
# 센서 데이터를 AI에게 보여줄 텍스트로 정리하는 코드
# PC Python에서 실행합니다

# 샘플 데이터: 교실 CO2 측정 (30초 간격, 10분간)
data = [
    {"time": "0:00", "co2": 620, "temp": 23.1, "humi": 45, "note": "수업 시작"},
    {"time": "0:30", "co2": 635, "temp": 23.2, "humi": 46, "note": ""},
    {"time": "1:00", "co2": 658, "temp": 23.2, "humi": 47, "note": ""},
    {"time": "1:30", "co2": 680, "temp": 23.3, "humi": 48, "note": ""},
    {"time": "2:00", "co2": 715, "temp": 23.3, "humi": 49, "note": ""},
    {"time": "2:30", "co2": 748, "temp": 23.4, "humi": 50, "note": ""},
    {"time": "3:00", "co2": 790, "temp": 23.4, "humi": 51, "note": ""},
    {"time": "3:30", "co2": 835, "temp": 23.5, "humi": 52, "note": ""},
    {"time": "4:00", "co2": 880, "temp": 23.5, "humi": 53, "note": ""},
    {"time": "4:30", "co2": 925, "temp": 23.6, "humi": 54, "note": ""},
    {"time": "5:00", "co2": 910, "temp": 23.5, "humi": 53, "note": "환기 시작"},
    {"time": "5:30", "co2": 850, "temp": 23.3, "humi": 51, "note": ""},
    {"time": "6:00", "co2": 780, "temp": 23.0, "humi": 48, "note": ""},
    {"time": "6:30", "co2": 720, "temp": 22.8, "humi": 46, "note": ""},
    {"time": "7:00", "co2": 670, "temp": 22.5, "humi": 44, "note": ""},
    {"time": "7:30", "co2": 640, "temp": 22.3, "humi": 43, "note": ""},
    {"time": "8:00", "co2": 615, "temp": 22.2, "humi": 42, "note": ""},
    {"time": "8:30", "co2": 600, "temp": 22.1, "humi": 42, "note": "창문 닫음"},
    {"time": "9:00", "co2": 610, "temp": 22.2, "humi": 43, "note": ""},
    {"time": "9:30", "co2": 625, "temp": 22.3, "humi": 43, "note": ""},
]

# AI에게 보여줄 텍스트 생성
print("=" * 60)
print("교실 환경 측정 데이터")
print("측정 조건: 25명 교실, SCD41 센서, 30초 간격")
print("=" * 60)
print(f"{'시간':<8} {'CO2(ppm)':<10} {'온도(C)':<10} {'습도(%)':<10} {'비고'}")
print("-" * 60)

for d in data:
    print(f"{d['time']:<8} {d['co2']:<10} {d['temp']:<10.1f} {d['humi']:<10} {d['note']}")

print("-" * 60)

# 기본 통계도 함께 출력
co2_values = [d['co2'] for d in data]
temp_values = [d['temp'] for d in data]
humi_values = [d['humi'] for d in data]

print(f"\n=== 기본 통계 ===")
print(f"CO2  | 평균: {sum(co2_values)/len(co2_values):.0f}ppm, "
      f"최소: {min(co2_values)}, 최대: {max(co2_values)}")
print(f"온도 | 평균: {sum(temp_values)/len(temp_values):.1f}C, "
      f"최소: {min(temp_values)}, 최대: {max(temp_values)}")
print(f"습도 | 평균: {sum(humi_values)/len(humi_values):.0f}%, "
      f"최소: {min(humi_values)}, 최대: {max(humi_values)}")

# 환기 전후 비교
before = [d['co2'] for d in data[:10]]  # 0:00~4:30
after = [d['co2'] for d in data[10:]]   # 5:00~9:30

print(f"\n=== 환기 전후 비교 ===")
print(f"환기 전 평균: {sum(before)/len(before):.0f}ppm")
print(f"환기 후 평균: {sum(after)/len(after):.0f}ppm")
print(f"변화: {sum(before)/len(before) - sum(after)/len(after):.0f}ppm 감소")

print("\n" + "=" * 60)
print("위 전체 텍스트를 복사하여 AI 챗봇에 붙여넣으세요.")
print("=" * 60)
```

### 도전 코드: AI 분석 결과 기반 인사이트 정리

AI의 분석 결과를 받은 뒤, 핵심 인사이트를 정리하는 코드입니다.

```python
# AI 분석 결과를 정리하는 인사이트 대시보드
# AI에게 분석을 받은 뒤, 핵심 내용을 아래에 직접 입력하세요

print("=" * 60)
print("      AI 데이터 분석 인사이트 대시보드")
print("=" * 60)

# === 여기에 AI 분석 결과를 직접 입력하세요 ===
insights = {
    "사용한 AI": "Claude",  # 또는 ChatGPT, Gemini 등
    "분석 데이터": "교실 CO2 + 온습도 (10분간)",

    "AI가 찾은 패턴": [
        "환기 전 CO2가 분당 약 60ppm씩 상승",
        "환기 시작 후 급격히 감소 (분당 약 90ppm)",
        "CO2와 온도가 양의 상관관계",
    ],

    "AI의 정확한 분석": [
        "환기 효과를 수치로 정확히 계산",
        "이상값 없음을 올바르게 판단",
    ],

    "AI의 부정확한 분석": [
        "교실 크기를 모르면서 환기량을 추정",
        "계절을 모르면서 냉방 제안",
    ],

    "AI가 몰랐던 맥락": [
        "교실이 3층이고 창문이 북향",
        "측정 당일 외부 기온이 5도",
        "에어컨 없음",
    ],

    "핵심 인사이트": "40분 수업 후 5분 환기가 최적. CO2 800ppm이 넘기 전에 환기 시작 권장.",
}

# 인사이트 출력
print(f"\n사용한 AI: {insights['사용한 AI']}")
print(f"분석 데이터: {insights['분석 데이터']}")

print(f"\n--- AI가 찾은 패턴 ---")
for i, p in enumerate(insights['AI가 찾은 패턴'], 1):
    print(f"  {i}. {p}")

print(f"\n--- 정확한 분석 ---")
for item in insights['AI의 정확한 분석']:
    print(f"  O {item}")

print(f"\n--- 부정확한 분석 ---")
for item in insights['AI의 부정확한 분석']:
    print(f"  X {item}")

print(f"\n--- AI가 몰랐던 맥락 ---")
for item in insights['AI가 몰랐던 맥락']:
    print(f"  ? {item}")

print(f"\n{'='*60}")
print(f"핵심 인사이트: {insights['핵심 인사이트']}")
print(f"{'='*60}")

# 평가 점수
print("\n--- AI 분석 평가 ---")
accurate = len(insights['AI의 정확한 분석'])
inaccurate = len(insights['AI의 부정확한 분석'])
total = accurate + inaccurate
if total > 0:
    score = accurate / total * 100
    print(f"정확도: {accurate}/{total} ({score:.0f}%)")
    if score >= 80:
        print("평가: AI 분석이 꽤 정확했습니다!")
    elif score >= 50:
        print("평가: AI 분석에 주의가 필요합니다.")
    else:
        print("평가: AI 분석을 그대로 믿으면 안 됩니다!")

print("\n결론: AI는 강력한 도구이지만, 최종 판단은 사람이 합니다.")
```

:::note[코드 읽어보기]
이 코드는 AI가 분석한 결과를 **여러분이 직접 정리하고 평가**하는 도구입니다. `insights` 딕셔너리에 AI의 분석 결과를 직접 입력하고, 정확한 것과 부정확한 것을 구분합니다. AI를 맹목적으로 믿는 것이 아니라, **비판적으로 검토하는 습관**이 중요합니다.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

2막 전체에서 여러분이 해온 일의 전체 흐름을 다시 생각해보세요.

**5차시**: AI에게 **코드를 시키는 방법**(바이브 코딩)을 배웠고
**→ 6~8차시**: 바이브 코딩으로 **센서 시스템을 만들고 데이터를 수집**했고
**→ 9차시**: 수집한 데이터를 **정리하고 시각화**했고
**→ 10차시**: AI에게 데이터를 **분석시키고, 그 결과를 검증**했습니다.

이 흐름에서 핵심은, **어느 한 단계도 혼자서는 완벽하지 않다**는 것입니다.

- **센서**는 이상값을 보냅니다.
- **코드**는 맥락을 모릅니다.
- **AI**는 교실의 실제 상황을 모릅니다.
- **사람**은 수천 개의 숫자를 빠르게 분석할 수 없습니다.

**사람 + 코드 + AI가 각자의 강점을 발휘할 때** 비로소 의미 있는 결과가 완성됩니다. 이것이 2막의 핵심 교훈입니다.

#### 인사이트 대시보드 설계 (종이 활동)

A4 용지에 다음 내용을 포함하는 대시보드를 그려보세요:

1. **제목**: 데이터가 말해주는 우리 교실
2. **핵심 숫자 3개**: 가장 중요한 수치를 크게 표시
3. **변화 그래프**: 시간에 따른 변화를 간단히 스케치
4. **AI의 추천**: AI가 제안한 개선 방안 1~2개
5. **우리의 판단**: AI 추천에 대한 우리의 의견

:::note[AI는 도구입니다]
AI는 매우 강력한 도구이지만, "정답을 알려주는 존재"가 아닙니다. AI는 데이터에서 패턴을 빠르게 찾아주지만, 그 패턴이 의미 있는지, 분석이 맞는지 **판단하는 것은 사람**의 몫입니다. 특히 AI는 가끔 **확신에 찬 말투로 잘못된 정보를 말하기도** 합니다. 항상 비판적으로 검토하세요.
:::

## 마무리

### 오늘 배운 것

- **AI에게 데이터 분석 요청하기**: 맥락을 포함한 좋은 프롬프트가 더 좋은 분석을 이끌어냅니다.
- **AI의 가능성**: 빠른 패턴 인식, 수치 계산, 다양한 관점에서의 분석
- **AI의 한계**: 맥락 이해 부족, 확신에 찬 오류, 검증 불가 — 사람의 비판적 검토가 필수입니다.

### 2막 전체 정리

2막에서 여러분은 **바이브 코딩**이라는 새로운 도구를 배웠습니다:

| 차시 | 주제 | 핵심 |
|------|------|------|
| 5차시 | AI에게 코드 시키기 | 바이브 코딩의 기초 |
| 6차시 | 환경 모니터링 대시보드 | 멀티센서 + OLED |
| 7차시 | 내 몸의 데이터 | 생체 신호 + 파형 |
| 8차시 | 스마트 알림 시스템 | 입력→처리→출력 |
| 9차시 | 데이터 정리와 시각화 | 전처리 + 그래프 |
| 10차시 | AI 데이터 분석 | AI 활용 + 비판적 사고 |

### 다음 시간 미리보기

:::tip[3막: 프로젝트]
다음 시간부터 **3막 "프로젝트"**가 시작됩니다. 1막에서 배운 직접 코딩, 2막에서 배운 바이브 코딩과 데이터 분석을 모두 활용하여, 팀 프로젝트로 **종합 대시보드 + 머신러닝**에 도전합니다. 첫 번째 단계는 **"측정 가능한 좋은 질문"**을 세우는 것입니다.
:::
