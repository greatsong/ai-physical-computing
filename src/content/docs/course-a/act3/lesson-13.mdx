## 활동 가이드

### Step 1. 머신러닝이란? (5분)

머신러닝을 한 문장으로 설명하면: **데이터에서 패턴을 찾아 새로운 데이터를 예측하는 기술**이에요.

**ML 파이프라인:**

```
데이터 수집 → 라벨링 → 학습 → 테스트 → 배포
```

| 단계 | 하는 일 | 오늘의 예시 |
|------|---------|------------|
| 데이터 수집 | 센서/마이크로 데이터를 모아요 | 박수, 노크, 말소리를 각각 녹음 |
| 라벨링 | 각 데이터에 이름표를 붙여요 | "이것은 박수", "이것은 노크" |
| 학습 | 컴퓨터가 패턴을 찾아요 | Teachable Machine이 자동으로 학습 |
| 테스트 | 새로운 데이터로 확인해요 | 새 소리를 들려주고 맞추는지 확인 |
| 배포 | 실제 환경에서 사용해요 | 대시보드에 분류 결과 표시 |

:::note[AI, ML, DL의 관계]
**AI**(인공지능)는 가장 넓은 개념이에요. **ML**(머신러닝)은 AI의 한 방법이고, **DL**(딥러닝)은 ML의 한 방법이에요. Teachable Machine은 내부적으로 딥러닝을 사용하지만, 우리는 쉬운 인터페이스로 사용할 수 있어요.
:::

### Step 2. 예시 1: 소리 분류 (15분)

마이크를 사용하여 **소리를 분류하는 ML 모델**을 만들어봐요.

**Teachable Machine 소리 프로젝트 만들기:**

1. [teachablemachine.withgoogle.com](https://teachablemachine.withgoogle.com)에 접속해요
2. **"오디오 프로젝트"**를 선택해요
3. 클래스(카테고리) 이름을 정해요:
   - Class 1: "박수" (Clap)
   - Class 2: "노크" (Knock)
   - Class 3: "말소리" (Voice)
   - Background Noise: 배경 소음 (자동 생성)
4. 각 클래스마다 **"마이크" 버튼**을 눌러 8초 이상 녹음해요
   - 박수: 박수를 여러 번 쳐요 (다양한 세기로)
   - 노크: 책상을 여러 번 두드려요
   - 말소리: 자연스럽게 대화해요
5. **"모델 학습시키기"** 버튼을 눌러요 (약 30초~1분 소요)
6. 학습 완료 후, **실시간 테스트**로 소리를 들려줘서 정확도를 확인해요

**좋은 학습 데이터 만들기 팁:**
- 각 클래스당 최소 20개 이상의 샘플을 모아요
- 다양한 세기와 속도로 소리를 내요
- 조용할 때의 배경 소음도 반드시 녹음해요
- 여러 사람이 번갈아 녹음하면 더 좋아요

### Step 3. 예시 2: 센서 데이터 수집 (15분)

Pico의 센서 데이터를 CSV로 수집하여, 동작이나 환경 패턴을 분류할 준비를 해요.

**가속도 센서 데이터 수집 예시:**
- 동작 1: "가만히 있기" -- 3축 값이 거의 변하지 않음
- 동작 2: "걷기" -- 규칙적인 진동 패턴
- 동작 3: "흔들기" -- 큰 진폭의 불규칙 패턴

수집한 CSV 데이터를 Teachable Machine에 업로드하거나, 패턴을 직접 비교해보세요.

### Step 4. 모델 테스트 및 개선 (15분)

학습된 모델의 정확도를 테스트하고, 틀리는 경우를 분석해요.

**테스트 방법:**
1. 학습에 사용하지 않은 **새로운 소리/동작**을 입력해요
2. 모델이 올바른 카테고리를 예측하는지 확인해요
3. 틀리는 경우, **왜 틀렸을까?**를 분석해요
   - 학습 데이터가 부족했나?
   - 카테고리 간 구분이 모호했나?
   - 배경 소음이 영향을 줬나?
4. 데이터를 추가하거나 카테고리를 재정의하고, 다시 학습시켜요

### 가속도 센서 데이터를 CSV로 수집 (MicroPython)

```python
from machine import I2C, Pin
import time

# I2C 설정
i2c = I2C(1, sda=Pin(6), scl=Pin(7))
time.sleep(0.1)

# ADXL345 가속도 센서 설정
ADXL_ADDR = 0x53
i2c.writeto_mem(ADXL_ADDR, 0x2D, bytes([0x08]))  # 측정 모드 활성화

def read_accel():
    """가속도 센서에서 X, Y, Z 값을 읽습니다"""
    data = i2c.readfrom_mem(ADXL_ADDR, 0x32, 6)
    x = (data[1] << 8) | data[0]
    y = (data[3] << 8) | data[2]
    z = (data[5] << 8) | data[4]
    # 부호 처리 (16비트 2의 보수)
    if x > 32767: x -= 65536
    if y > 32767: y -= 65536
    if z > 32767: z -= 65536
    return x, y, z

# 데이터 수집 설정
LABEL = "walking"    # 동작 이름 (수집할 때마다 변경)
DURATION = 10        # 수집 시간 (초)
INTERVAL = 0.05      # 수집 간격 (초) = 20Hz
FILENAME = f"data_{LABEL}.csv"

# CSV 파일 생성
with open(FILENAME, "w") as f:
    f.write("time,x,y,z,label\n")

print(f"'{LABEL}' 데이터 수집을 시작합니다!")
print(f"  {DURATION}초 동안 동작을 수행하세요...")
print("  3... ", end="")
time.sleep(1)
print("2... ", end="")
time.sleep(1)
print("1... 시작!")
time.sleep(1)

start = time.time()
count = 0

while time.time() - start < DURATION:
    try:
        x, y, z = read_accel()
        elapsed = time.time() - start
        count += 1

        with open(FILENAME, "a") as f:
            f.write(f"{elapsed:.3f},{x},{y},{z},{LABEL}\n")

        # 진행 표시
        if count % 40 == 0:  # 2초마다
            print(f"  수집 중... {elapsed:.1f}초 / {count}개")

    except Exception as e:
        print(f"  오류: {e}")

    time.sleep(INTERVAL)

print(f"\n수집 완료! {count}개 데이터 -> {FILENAME}")
print("LABEL 값을 바꿔서 다른 동작도 수집하세요.")
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 24 | `LABEL = "walking"` | 현재 수집하는 동작의 이름이에요. "still", "walking", "shaking" 등으로 바꿔가며 수집해요. |
| 26 | `INTERVAL = 0.05` | 0.05초 = 50ms = 초당 20회(20Hz) 측정이에요. 동작 인식에는 이 정도면 충분해요. |
| 27 | `f"data_{LABEL}.csv"` | f-string으로 라벨 이름을 파일명에 포함시켜요. 동작별로 파일이 분리돼요. |
| 44 | `f"{elapsed:.3f}"` | 소수점 3자리까지 시간을 기록해요. ML에서 시간 축 정밀도가 중요할 수 있어요. |

### 간단한 소리 분류 시뮬레이션 (PC Python)

Teachable Machine 없이도 **규칙 기반 분류**의 원리를 이해할 수 있는 코드예요.

```python
# 소리 센서 데이터를 규칙 기반으로 분류하는 시뮬레이션
# 실제 ML은 이런 규칙을 데이터에서 자동으로 찾아냅니다

import random

def classify_sound(level, duration, pattern):
    """소리의 특성을 보고 종류를 분류합니다

    level: 소리 크기 (0~100)
    duration: 지속 시간 (초)
    pattern: 패턴 유형 ("spike"=순간, "steady"=지속, "rhythmic"=반복)
    """
    if level > 70 and pattern == "spike" and duration < 0.5:
        return "박수", 0.85
    elif level > 50 and pattern == "spike" and duration < 0.3:
        return "노크", 0.78
    elif level > 30 and pattern == "steady" and duration > 1.0:
        return "말소리", 0.72
    elif level < 20:
        return "배경소음", 0.95
    else:
        return "알 수 없음", 0.30

# 테스트 데이터 (실제로는 센서에서 수집)
test_sounds = [
    {"level": 82, "duration": 0.2, "pattern": "spike",   "actual": "박수"},
    {"level": 65, "duration": 0.1, "pattern": "spike",   "actual": "노크"},
    {"level": 45, "duration": 2.5, "pattern": "steady",  "actual": "말소리"},
    {"level": 12, "duration": 5.0, "pattern": "steady",  "actual": "배경소음"},
    {"level": 55, "duration": 1.0, "pattern": "rhythmic","actual": "박수"},
]

print("=== 소리 분류 테스트 ===\n")
correct = 0
total = len(test_sounds)

for i, sound in enumerate(test_sounds, 1):
    predicted, confidence = classify_sound(
        sound["level"], sound["duration"], sound["pattern"]
    )
    match = "O" if predicted == sound["actual"] else "X"
    if predicted == sound["actual"]:
        correct += 1

    print(f"테스트 {i}: 크기={sound['level']}, 지속={sound['duration']}s")
    print(f"  예측: {predicted} (신뢰도: {confidence:.0%})")
    print(f"  정답: {sound['actual']} [{match}]\n")

accuracy = correct / total
print(f"정확도: {correct}/{total} = {accuracy:.0%}")

if accuracy >= 0.8:
    print("좋은 분류 성능이에요!")
elif accuracy >= 0.6:
    print("보통이에요. 더 많은 데이터와 규칙이 필요해요.")
else:
    print("개선이 필요해요. 이것이 규칙 기반의 한계예요.")
    print("ML은 이런 규칙을 데이터에서 자동으로 학습합니다!")
```

:::note[규칙 기반 vs 머신러닝]
위 코드는 사람이 직접 규칙(`if level > 70 and ...`)을 만들었어요. 하지만 실제 소리는 이렇게 깔끔하게 나뉘지 않아요. **머신러닝**은 수천 개의 데이터에서 이런 규칙을 **자동으로** 찾아내요. 사람이 생각하지 못한 패턴도 발견할 수 있다는 것이 ML의 강점이에요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

박수를 쳤더니 컴퓨터가 **"이것은 박수입니다"**라고 말해요. 책상을 두드렸더니 **"이것은 노크입니다"**라고 말해요.

여러분이 한 일은 이것이에요:
1. **데이터를 모았어요** (소리 샘플)
2. **이름표를 붙였어요** ("박수", "노크", "말소리")
3. **학습 버튼을 눌렀어요**
4. 컴퓨터가 **스스로 패턴을 찾았어요**

이것이 **머신러닝의 전부**예요. 복잡한 수학이나 코드가 아니라, **좋은 데이터를 모으고 적절한 라벨을 붙이는 것**이 핵심이에요.

자율주행차가 사람과 신호등을 구분하는 것도, 음성 비서가 "시리야"를 알아듣는 것도, 스팸 필터가 광고 메일을 걸러내는 것도 -- 모두 같은 원리예요. **데이터 -> 패턴 학습 -> 예측.** 여러분은 방금 이 과정을 직접 경험했어요.

### ML 성능 개선 실험

모델의 정확도가 낮다면, 아래 방법을 시도해보세요:

1. **데이터 추가**: 각 카테고리의 샘플을 2배로 늘려요
2. **카테고리 조정**: 구분하기 어려운 카테고리를 합치거나 쪼개요
3. **노이즈 제거**: 배경 소음 샘플을 더 다양하게 수집해요
4. **다시 학습**: 개선된 데이터로 모델을 다시 학습시켜요

:::note[데이터가 전부]
ML에서 가장 중요한 것은 **알고리즘**이 아니라 **데이터의 질**이에요. 좋은 데이터 10개가 나쁜 데이터 1000개보다 나아요. "Garbage in, Garbage out" -- 쓰레기 데이터를 넣으면 쓰레기 결과가 나온다는 뜻이에요.
:::
