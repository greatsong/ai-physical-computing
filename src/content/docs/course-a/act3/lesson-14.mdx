## 활동 가이드

### Step 1. 발표 준비 (5분)

팀별로 발표 순서를 정하고, 마지막 점검을 해요. 발표는 아래 **5단계 구조**를 따르세요.

**발표 구조 (5분):**

| 단계 | 시간 | 내용 | 예시 |
|------|------|------|------|
| 1. 문제 | 30초 | 우리가 발견한 문제 | "교실이 답답한데 환기 타이밍을 모르겠어요" |
| 2. 센서/대시보드 | 1분 30초 | 센서 구성 + 대시보드 시연 | OLED 대시보드 실시간 동작 시연 |
| 3. 데이터 | 1분 | 수집한 데이터와 패턴 | "CO2가 수업 시작 30분 후 1000ppm을 넘어요" |
| 4. ML 결과 | 1분 | ML 모델 학습 결과 | "Teachable Machine으로 환기필요 상태를 87% 정확도로 감지" |
| 5. 의미 | 1분 | 이 프로젝트의 가치와 개선 방향 | "자동 환기 알림 시스템으로 발전 가능" |

:::tip[발표 팁]
**대시보드를 실제로 시연하세요.** 슬라이드보다 실시간으로 동작하는 대시보드가 훨씬 인상적이에요. OLED에 데이터가 나오는 모습, PC 대시보드의 실시간 그래프, Teachable Machine의 분류 결과를 직접 보여주세요.
:::

### Step 2. 팀별 발표 + 질의응답 (30분)

- 팀당 **5분 발표 + 2분 질의응답** = 7분
- 4~5팀 기준으로 약 30분이 소요돼요
- 발표를 듣는 팀은 아래 **피드백 양식**을 작성해요

#### 피드백 양식

각 발표를 듣고 1~5점으로 평가하세요. (1: 부족 ~ 5: 훌륭)

| 평가 항목 | 점수 (1~5) | 메모 |
|-----------|-----------|------|
| **대시보드 디자인** - 정보가 한눈에 들어오는가? | | |
| **데이터 신뢰성** - 센서 데이터가 합리적이고 충분한가? | | |
| **ML 활용도** - ML을 의미 있게 활용했는가? | | |
| **발표 전달력** - 설명이 명확하고 시연이 효과적인가? | | |
| **실용성** - 실제로 문제를 해결할 수 있는가? | | |
| **한 줄 코멘트** | | (자유롭게) |

### Step 3. 피드백 집계 (10분)

모든 발표가 끝난 후, 각 팀이 받은 피드백 점수를 아래 코드로 집계해요.

### Step 4. 피드백 결과 공유 (5분)

집계 결과를 전체 학급에 공유해요.
- 가장 높은 점수를 받은 **항목**은 무엇인가요?
- 가장 인상적이었던 **대시보드 디자인**은?
- 가장 창의적인 **ML 활용 사례**는?

### 피드백 점수 집계 코드 (PC Python)

```python
# 피드백 점수 집계 코드
# 우리 팀이 받은 점수를 입력하세요

# 각 평가자의 항목별 점수 (5개 항목, 1~5점)
feedback_data = [
    # [대시보드, 데이터, ML활용, 전달력, 실용성]
    [4, 5, 3, 4, 4],  # 1조에서 받은 점수
    [5, 4, 4, 5, 3],  # 2조에서 받은 점수
    [3, 4, 5, 4, 4],  # 3조에서 받은 점수
    [4, 5, 4, 3, 5],  # 4조에서 받은 점수
]

categories = [
    "대시보드 디자인",
    "데이터 신뢰성",
    "ML 활용도",
    "발표 전달력",
    "실용성",
]

# 항목별 평균 계산
print("=== 우리 팀 피드백 결과 ===\n")
total_avg = 0

for i, category in enumerate(categories):
    scores = [row[i] for row in feedback_data]
    avg = sum(scores) / len(scores)
    total_avg += avg

    # 막대 시각화 (텍스트)
    bar = "#" * int(avg * 4)  # 최대 20칸
    print(f"{category:14s} | {bar:20s} | {avg:.1f}점")

total_avg /= len(categories)
print(f"\n{'종합 평균':14s} | {'=' * int(total_avg * 4):20s} | {total_avg:.1f}점")

# 강점과 보완점
scores_by_cat = []
for i in range(len(categories)):
    avg = sum(row[i] for row in feedback_data) / len(feedback_data)
    scores_by_cat.append((categories[i], avg))

scores_by_cat.sort(key=lambda x: x[1], reverse=True)
print(f"\n강점: {scores_by_cat[0][0]} ({scores_by_cat[0][1]:.1f}점)")
print(f"보완: {scores_by_cat[-1][0]} ({scores_by_cat[-1][1]:.1f}점)")
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 5 | `feedback_data = [...]` | 2차원 리스트예요. 각 안쪽 리스트는 한 팀에서 받은 5개 항목의 점수예요. |
| 21 | `scores = [row[i] for row in feedback_data]` | **리스트 컴프리헨션**이에요. 모든 평가자의 i번째 항목 점수를 모아서 새 리스트를 만들어요. |
| 24 | `"#" * int(avg * 4)` | 문자열에 숫자를 곱하면 반복돼요. `"#" * 3`은 `"###"`이 돼요. |
| 33 | `.sort(key=lambda x: x[1], reverse=True)` | 점수 기준으로 내림차순 정렬해요. `lambda`는 짧은 함수를 한 줄로 만드는 문법이에요. |

### 도전 코드: 전체 학급 피드백 대시보드

```python
# 모든 팀의 피드백을 한눈에 비교하는 대시보드

all_teams = {
    "1조 교실환경": [
        [4, 5, 3, 4, 4],
        [5, 4, 4, 5, 3],
        [4, 4, 5, 4, 4],
    ],
    "2조 소음측정": [
        [5, 5, 3, 4, 5],
        [4, 4, 4, 3, 4],
        [5, 5, 4, 4, 5],
    ],
    "3조 통행량": [
        [3, 4, 4, 5, 3],
        [4, 3, 5, 4, 4],
        [3, 4, 4, 5, 3],
    ],
}

categories = ["대시보드", "데이터", "ML", "전달력", "실용성"]

print("=" * 60)
print(f"{'팀':>14s}", end="")
for cat in categories:
    print(f" | {cat:>6s}", end="")
print(f" | {'종합':>6s}")
print("=" * 60)

team_totals = {}

for team_name, feedbacks in all_teams.items():
    avgs = []
    for i in range(len(categories)):
        scores = [fb[i] for fb in feedbacks]
        avgs.append(sum(scores) / len(scores))

    total = sum(avgs) / len(avgs)
    team_totals[team_name] = total

    print(f"{team_name:>14s}", end="")
    for avg in avgs:
        print(f" | {avg:>5.1f}점", end="")
    print(f" | {total:>5.1f}점")

print("=" * 60)

# 종합 순위
print("\n=== 종합 순위 ===")
ranked = sorted(team_totals.items(), key=lambda x: x[1], reverse=True)
for rank, (team, score) in enumerate(ranked, 1):
    medal = ["1st", "2nd", "3rd"][rank - 1] if rank <= 3 else f"{rank}th"
    print(f"  {medal}. {team} - {score:.1f}점")

# 항목별 최고 팀
print("\n=== 항목별 최고 팀 ===")
for i, cat in enumerate(categories):
    best_team = ""
    best_score = 0
    for team_name, feedbacks in all_teams.items():
        avg = sum(fb[i] for fb in feedbacks) / len(feedbacks)
        if avg > best_score:
            best_score = avg
            best_team = team_name
    print(f"  {cat}: {best_team} ({best_score:.1f}점)")
```

:::note[코드 읽어보기]
`enumerate(ranked, 1)`은 순서 번호를 1부터 매겨줘요. `sorted()`는 원래 리스트를 바꾸지 않고 정렬된 새 리스트를 반환해요. `all_teams`의 점수를 여러분 학급의 실제 점수로 바꿔서 실행해보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

**문제 발견 -> 센서 선택 -> 대시보드 제작 -> ML 학습 -> 발표 -> 피드백**

이 전체 과정은 **IT 프로젝트의 실제 개발 사이클**과 동일해요.

- 스타트업이 새로운 서비스를 만들 때
- 데이터 과학자가 분석 결과를 보고할 때
- 엔지니어가 IoT 시스템을 제안할 때

모두 "문제 -> 프로토타입 -> 데이터 -> 발표 -> 피드백"을 반복해요. 여러분은 이 사이클을 4차시에 걸쳐 직접 경험한 거예요.

그리고 피드백도 결국 **데이터**예요. 숫자로 모으고, 평균을 내고, 패턴을 찾는 -- 센서 데이터를 다루는 것과 같은 원리로 피드백을 분석했어요.
