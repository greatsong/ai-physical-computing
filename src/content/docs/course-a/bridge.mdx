## 코스 A 역량과 코스 B 레벨의 연결

코스 A에서 배운 역량이 코스 B의 어느 레벨에서 활용되는지 확인해보세요.

| 레벨 | 코스 A에서 이미 연습한 것 | 새롭게 도전하는 것 |
|------|------------------------|------------------|
| **Lv.1 탐험가** (3시간) | 1차시 I2C 센서 연결, 2차시 ADC 읽기, 3차시 CSV 저장, 4차시 OLED 대시보드 | 원하는 센서를 **직접 선택**하여 나만의 대시보드 설계 |
| **Lv.2 발명가** (3시간) | 5차시 바이브 코딩, 6차시 복합 센서 대시보드, 8차시 스마트 알림 시스템 | 센서 2개 이상 조합 + 복합 조건 판단 + 프롬프트 기록 |
| **Lv.3 연구자** (3시간) | 9차시 PC Python 데이터 시각화, 10차시 AI 분석, 11차시 프로젝트 설계 | 가설 수립 → 장기 데이터 수집 → AI 분석으로 검증 |
| **Lv.4 마스터** (3시간) | 12차시 종합 대시보드, 13차시 ML 분류, 14차시 발표와 피드백 | 센서 3개+ ML 모델 + 실시간 대시보드 결합, 실제 문제 해결 |

## 코스 A vs 코스 B 비교

|  | 코스 A | 코스 B |
|---|--------|--------|
| **형태** | 수업형 (교사 주도) | 프로젝트형 (학생 주도) |
| **시간** | 50분 x 15주 | 3시간 x 레벨별 |
| **구조** | 정해진 차시 순서 (3막) | 레벨 자유 선택 |
| **센서** | 차시별 지정 | 학생이 선택 |
| **코딩 방식** | 직접 코딩 -> 바이브 코딩 | 자유 선택 (직접 + 바이브 혼합) |
| **AI/ML** | 바이브 코딩 + Teachable Machine 체험 | ML 모델 설계 + AI 분석 심화 |
| **대시보드** | OLED + PC 기초 | 종합 시스템 구축 |
| **목표** | 기초 역량 습득 | 실전 프로젝트 완성 |
| **평가** | 성장 레이더 차트 자기점검 | 산출물 기반 평가 |
| **적합 대상** | 처음 시작하는 학생 | 기초를 마친 학생 |

## 코스 B 레벨별 추천 시작점

여러분의 상태에 맞는 레벨에서 시작하세요.

:::tip[15차시 성장 레이더 차트를 참고하세요!]
[15차시](/ai-physical-computing/course-a/act3/lesson-15/)에서 작성한 **인터랙티브 성장 레이더 차트**의 점수가 여러분의 현재 실력을 보여줘요. 8개 역량 점수를 참고하여 시작 레벨을 선택하세요.
:::

### 코스 A를 모두 이수한 학생

| 자기 평가 | 추천 레벨 | 설명 |
|-----------|----------|------|
| "센서 연결은 되지만 코드가 아직 어려워요" | **Lv.1 탐험가** | 1~4차시에서 배운 것을 내 힘으로 다시 해보세요 |
| "바이브 코딩도 익숙하고 여러 센서를 써봤어요" | **Lv.2 발명가** | 5~8차시처럼 복합 센서 시스템에 도전하세요 |
| "데이터 분석이 재미있고 가설을 세워보고 싶어요" | **Lv.3 연구자** | 9~10차시의 분석 역량을 심화하세요 |
| "대시보드도 ML도 자신 있고 실제 문제를 해결하고 싶어요" | **Lv.4 마스터** | 11~14차시의 종합 프로젝트를 한 단계 넘어서세요 |

### 코스 A 없이 코스 B부터 시작하는 학생

:::caution[사전 준비가 필요해요]
코스 B는 코스 A의 기초 역량을 전제해요. 코스 A를 거치지 않았다면, 최소한 아래 내용을 먼저 확인하세요.

1. [시작 가이드](/ai-physical-computing/guide/setup/) - Pico 연결과 Thonny 설치
2. [1차시](/ai-physical-computing/course-a/act1/lesson-01/) - DHT20 센서 연결과 I2C 코드 실행
3. [3차시](/ai-physical-computing/course-a/act1/lesson-03/) - CSV 파일 저장 방법

이 세 차시를 마치면 Lv.1 탐험가부터 시작할 수 있어요.
:::
