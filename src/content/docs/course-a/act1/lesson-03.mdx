## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. WH 모델은 헤더 핀이 미리 납땜되어 있어 바로 사용할 수 있습니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |
| Grove 케이블 (4핀) | 센서와 Shield를 연결하는 케이블입니다. 양쪽 끝에 4핀 커넥터가 달려 있습니다. |
| 온습도 센서 (DHT20) | 1차시에서 사용한 온습도 센서를 다시 사용합니다. I2C 방식으로 통신합니다. ([센서 도감에서 자세히 보기](/ai-physical-computing/appendix/sensor-catalog/)) |
| USB-C 케이블 | Pico와 컴퓨터를 연결하여 전원을 공급하고 코드를 전송합니다. |

### 연결 방법

DHT20 센서를 Grove 케이블로 Shield의 **I2C 포트**(I2C0)에 연결하세요. 1차시와 동일한 연결입니다.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[1차시 복습]
DHT20은 **I2C 포트**에 연결합니다. 아날로그(A0~A2)나 디지털(D16~D20) 포트가 아닌, "I2C0" 또는 "I2C1"이라고 적힌 포트를 찾으세요.
:::

## 활동 가이드

### Step 1. DHT20 재연결 + 1차시 코드 복습 (5분)

1. **DHT20 센서를 I2C 포트에 연결합니다.** 1차시와 같은 연결입니다.
2. **USB-C 케이블로 컴퓨터와 연결합니다.**
3. **Thonny에서 1차시 기본 코드를 실행하여** 센서가 정상 동작하는지 확인합니다. 온도와 습도가 출력되면 준비 완료입니다.

:::tip
1차시 코드를 저장해두지 않았다면, [1차시 페이지](/ai-physical-computing/course-a/act1/lesson-01/)에서 기본 코드를 다시 가져올 수 있습니다.
:::

### Step 2. 파일 저장 코드 작성 (20분)

이제 센서 데이터를 화면에 출력하는 것에서 한 단계 더 나아가, **CSV 파일로 자동 저장**하는 코드를 작성합니다.

1. 아래 [기본 코드](#기본-코드)를 Thonny 편집기에 입력합니다.
2. 코드를 읽어보면서 각 줄이 무엇을 하는지 이해해봅니다. 특히 `open()`, `write()`, `flush()` 세 함수의 역할에 주목하세요.
3. 상단의 **실행(Run)** 버튼을 클릭합니다.
4. Shell 창에 측정값이 출력되면서 동시에 파일이 만들어지는 것을 확인합니다.

:::note[파일은 어디에 저장되나요?]
Pico에 코드를 저장하고 실행하면, CSV 파일도 **Pico 내부 저장소**에 만들어집니다. Thonny의 왼쪽 파일 탐색기에서 "Raspberry Pi Pico"를 클릭하면 생성된 파일을 볼 수 있습니다. 파일을 더블클릭하면 내용을 확인할 수 있습니다.
:::

:::note[CSV 파일이란?]
CSV는 Comma-Separated Values의 약자로, 데이터를 쉼표(,)로 구분하여 저장하는 형식입니다. 엑셀, 구글 스프레드시트 등에서 바로 열 수 있어서 데이터를 분석하기 편리합니다. 파일 한 줄이 데이터 한 건입니다.
:::

### Step 3. 5분간 자동 기록 실행 (15분)

1. **코드를 실행하고 5분간 기다립니다.** 10초 간격으로 측정하면 5분 동안 약 30개의 데이터가 쌓입니다.
2. 기다리는 동안 다음을 시도해보세요:
   - 센서 위에 손을 올려 온도를 올려보기
   - 입김을 불어 습도를 올려보기
   - 창문을 열어 환기해보기
3. **5분 후 코드를 정지합니다.** Thonny에서 **Stop** 버튼을 클릭하거나, Ctrl+C를 누릅니다.
4. Thonny 파일 탐색기에서 `data.csv` 파일을 열어봅니다.
5. CSV 파일의 내용을 확인합니다. 각 줄에 측정번호, 온도, 습도가 쉼표로 구분되어 있는지 살펴보세요.

:::tip
코드를 실행하고 기다리는 5분은 지루하지 않습니다. 센서에 다양한 자극을 주고 그 결과가 데이터로 기록되는 과정을 관찰하세요. 나중에 CSV 파일을 열었을 때 "아, 이때 내가 손을 올렸구나"라는 것이 데이터에 남아 있을 것입니다.
:::

### Step 4. 토론 - 자동 vs 수동 기록의 차이 (10분)

다음 질문을 모둠에서 이야기해보세요.

1. **"사람이 직접 기록하는 것과 코드가 자동으로 기록하는 것, 각각의 장점은 무엇인가요?"** 정확성, 편리함, 속도, 지속성 등의 관점에서 비교해봅시다.
2. **"자동 기록의 한계는 무엇인가요?"** 센서가 고장 나면? 저장 공간이 부족하면? 측정 간격이 너무 길면?
3. **"이 자동 기록 시스템을 실제로 어디에 활용할 수 있을까요?"** 학교, 가정, 농장, 공장 등 구체적인 사례를 생각해봅시다.

### 기본 코드

```python
from machine import I2C, Pin
import time

# I2C 통신 설정 (Grove Shield I2C0 포트)
i2c = I2C(0, sda=Pin(8), scl=Pin(9))

# DHT20 센서 초기화
sensor_addr = 0x38
time.sleep(0.1)

def read_dht20():
    """DHT20 센서에서 온도와 습도를 읽습니다"""
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

# CSV 파일 생성 (헤더 쓰기)
file = open("data.csv", "w")
file.write("측정번호,온도(C),습도(%)\n")

# 10초마다 측정하여 파일에 저장
count = 1
while True:
    temp, humi = read_dht20()

    # 화면에 출력
    print(f"[{count}] 온도: {temp}°C, 습도: {humi}%")

    # 파일에 저장
    file.write(f"{count},{temp},{humi}\n")
    file.flush()  # 버퍼를 비워 즉시 저장

    count += 1
    time.sleep(10)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 23 | `open("data.csv", "w")` | `data.csv`라는 파일을 쓰기 모드("w")로 엽니다. 파일이 없으면 새로 만들고, 있으면 내용을 지우고 새로 씁니다. |
| 24 | `file.write("측정번호,온도(C),습도(%)\n")` | CSV 파일의 첫 줄에 열 제목(헤더)을 씁니다. `\n`은 줄바꿈을 의미합니다. |
| 30 | `print(f"[{count}] ...")` | 화면에 현재 측정 상태를 보여줍니다. 파일 저장과 별도로 동작합니다. |
| 33 | `file.write(f"{count},{temp},{humi}\n")` | 측정번호, 온도, 습도를 쉼표로 구분하여 파일에 한 줄 추가합니다. 이것이 CSV 형식입니다. |
| 34 | `file.flush()` | 데이터를 즉시 파일에 기록합니다. 이 줄이 없으면 데이터가 메모리에만 남아 있다가, 전원이 꺼지면 사라질 수 있습니다. |
| 36 | `count += 1` | 측정 번호를 1씩 증가시킵니다. `count = count + 1`과 같은 뜻입니다. 이 번호가 간이 타임스탬프 역할을 합니다. |

### 도전 코드: 측정 횟수 제한 + 시작/종료 메시지

무한 반복 대신 정해진 횟수만 측정하고, 시작과 종료 메시지를 표시하며, 파일 크기를 확인해봅시다.

```python
from machine import I2C, Pin
import time
import os

# I2C 통신 설정
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
sensor_addr = 0x38
time.sleep(0.1)

def read_dht20():
    """DHT20 센서에서 온도와 습도를 읽습니다"""
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

# 설정
filename = "classroom_data.csv"
max_count = 30    # 30회 측정 후 자동 종료
interval = 10     # 10초 간격

# CSV 파일 생성
file = open(filename, "w")
file.write("측정번호,온도(C),습도(%)\n")

# 시작 메시지
print(f"=== {filename} 기록 시작 ===")
print(f"총 {max_count}회, {interval}초 간격으로 측정합니다.")
print(f"예상 소요 시간: {max_count * interval // 60}분 {max_count * interval % 60}초")
print()

# 정해진 횟수만큼 측정
for count in range(1, max_count + 1):
    temp, humi = read_dht20()
    print(f"[{count}/{max_count}] 온도: {temp}°C, 습도: {humi}%")
    file.write(f"{count},{temp},{humi}\n")
    file.flush()
    time.sleep(interval)

# 파일 닫기
file.close()

# 종료 메시지 + 파일 크기 확인
file_size = os.stat(filename)[6]  # 파일 크기(바이트)
print(f"\n=== 기록 완료! ===")
print(f"파일: {filename}")
print(f"데이터: {max_count}개")
print(f"파일 크기: {file_size} 바이트")
```

:::note[코드 읽어보기]
`for count in range(1, 31):`은 count를 1부터 30까지 반복하라는 뜻입니다. `while True:`와 달리 정해진 횟수만 반복하므로 자동으로 종료됩니다. `file.close()`는 파일을 안전하게 닫습니다. `os.stat()`으로 파일 크기를 확인할 수 있습니다. 파일명이나 측정 횟수를 바꿔서 다양한 실험을 해보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

센서가 공기의 온도와 습도를 측정하고
**→** 코드가 그 데이터를 읽어서
**→** CSV 형식으로 **파일에 자동 저장**하고
**→** 시간이 지나면서 **데이터셋**이 만들어졌습니다.

이것은 과학자가 실험 노트에 손으로 기록하던 일을 코드가 대신한 것입니다. 기상청의 자동 기상 관측소(AWS), 병원의 환자 모니터링 시스템, 공장의 품질 관리 센서 — 모두 이 원리의 확장입니다. 사람이 24시간 지켜볼 수 없는 것을 코드가 대신 기록합니다.

센서 **→** 코드 **→** 파일 **→** 데이터셋. 여러분이 만든 30줄짜리 코드가 바로 **자동 실험 기록 시스템**의 시작점입니다.

## 마무리

### 오늘 배운 것

- **파일 입출력**: `open()`, `write()`, `flush()`, `close()`를 사용하여 데이터를 파일로 저장하는 방법을 배웠습니다.
- **CSV 형식**: 쉼표로 데이터를 구분하는 표준 형식을 이해했습니다. 이 형식은 엑셀, 구글 시트에서 바로 열 수 있습니다.
- **자동 기록의 가치**: 사람이 일일이 기록하는 것과 코드가 자동으로 기록하는 것의 차이를 체험했습니다.
- **데이터셋 만들기**: 시간에 따라 축적된 데이터에서 패턴을 발견하는 경험을 했습니다.
