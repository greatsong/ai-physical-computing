## 준비물

| 준비물 | 설명 |
|--------|------|
| Raspberry Pi Pico 2 WH | MicroPython이 동작하는 마이크로컨트롤러입니다. WH 모델은 헤더 핀이 미리 납땜되어 있어 바로 사용할 수 있습니다. |
| Grove Shield for Pi Pico | Pico 위에 올려 꽂는 확장 보드입니다. Grove 커넥터 포트가 여러 개 있어 센서를 케이블 하나로 연결할 수 있습니다. |

### 연결 방법

빛 센서를 Grove 케이블로 Shield의 **A0 포트**(아날로그)에 연결하세요.

:::tip[Grove 커넥터 안내]
4핀 커넥터라 방향이 정해져 있어서, 홈에 맞춰 끝까지 밀어 넣으면 됩니다. 잘못 꽂을 걱정이 없습니다.
:::

:::note[포트 찾기]
빛 센서는 **아날로그 센서**입니다. 반드시 **"A0"이라고 적힌 포트**에 연결해야 합니다. I2C 포트나 디지털(D16~D20) 포트에 꽂으면 올바른 값을 읽을 수 없습니다. A0 포트는 Pico의 GP26 핀에 연결되어 있습니다.
:::

## 활동 가이드

### Step 1. 연결하기 (5분)

1. **Pico 2 WH가 Grove Shield 위에 올려져 있는지 확인합니다.** 1차시에서 이미 연결했다면 그대로 사용하세요.
2. **빛 센서를 Grove 케이블로 A0 포트에 연결합니다.** Shield에서 "A0"이라고 적힌 포트를 찾아 꽂으세요.
3. **USB-C 케이블로 컴퓨터와 연결합니다.**

:::tip
빛 센서의 앞면에 작은 원형 부품(포토레지스터)이 보입니다. 이 부분이 빛을 감지하는 핵심 소자입니다. 손으로 가려보면 값이 바뀌는 것을 확인할 수 있습니다.
:::

### Step 2. ADC로 밝기 읽기 (15분)

1. 컴퓨터에서 **Thonny**를 열어주세요.
2. 아래 [기본 코드](#기본-코드)를 Thonny 편집기에 입력합니다.
3. 상단의 **실행(Run)** 버튼을 클릭합니다.
4. 하단 **Shell** 창에 밝기 값이 2초마다 출력되는지 확인하세요.

:::note[생각해보기]
손으로 센서를 완전히 가리면 값이 어떻게 변하나요? 스마트폰 손전등을 센서에 비추면 어떤가요? 값의 범위를 탐색해보세요.
:::

### Step 3. Thonny Plotter로 실시간 그래프 그리기 (15분)

이제 숫자를 **그래프**로 바꿔봅시다. 이것이 여러분의 **첫 대시보드 경험**입니다.

1. Thonny 메뉴에서 **View > Plotter**를 선택합니다. 화면 하단에 그래프 영역이 나타납니다.
2. 코드를 다시 실행합니다. `print()`로 출력한 숫자가 자동으로 **실시간 그래프**가 됩니다.
3. 다음을 시도하며 그래프 변화를 관찰하세요:
   - 손으로 센서를 천천히 가리기 → 그래프가 내려갑니다
   - 손을 빠르게 떼기 → 그래프가 급격히 올라갑니다
   - 스마트폰 손전등을 가까이/멀리 → 거리에 따라 그래프가 변합니다

:::note[Thonny Plotter 사용법]
Thonny Plotter는 `print()`로 출력한 숫자를 자동으로 그래프로 그려줍니다. 핵심은 **숫자만 단독으로 출력**하는 것입니다. 여러 값을 동시에 그래프로 그리고 싶으면 `print(val1, val2)`처럼 쉼표로 구분하면 됩니다. 각 값이 서로 다른 색의 선으로 표시됩니다.
:::

### Step 4. 교실 곳곳 밝기 측정 (15분)

교실 안 **5곳 이상**의 위치에서 밝기를 측정해봅시다. 노트북과 Pico를 함께 들고 이동하면서, 각 위치에서 값이 안정될 때까지(약 5초) 기다린 뒤 기록합니다.

:::tip
측정할 때 센서가 같은 방향(예: 천장을 향하도록)을 가리키게 하세요. 센서의 방향에 따라 값이 달라질 수 있습니다.
:::

**측정 위치 예시:** 창가(햇빛 직사), 창가(커튼 친 곳), 교실 중앙, 복도 쪽, 사물함 앞

## 코드

### 기본 코드

```python
from machine import ADC, Pin
import time

# 아날로그 입력 설정 (Grove Shield A0 포트 = GP26)
light = ADC(Pin(26))

# 2초마다 밝기 측정하여 출력
while True:
    raw_value = light.read_u16()           # 0 ~ 65535
    brightness = round(raw_value / 65535 * 100, 1)  # 퍼센트로 변환
    print(f"밝기: {brightness}%")
    print(brightness)  # Plotter용 숫자만 출력
    time.sleep(2)
```

### 줄별 해설

| 줄 | 코드 | 설명 |
|----|------|------|
| 1 | `from machine import ADC, Pin` | `ADC`는 Analog-to-Digital Converter의 약자입니다. 아날로그 신호(연속적인 전압)를 디지털 숫자로 바꿔주는 장치입니다. |
| 5 | `ADC(Pin(26))` | GP26 핀에 연결된 아날로그 센서를 읽겠다는 뜻입니다. Grove Shield의 A0 포트가 GP26에 연결되어 있습니다. |
| 9 | `read_u16()` | 센서 값을 16비트 부호 없는 정수(0~65535)로 읽습니다. 어두우면 0에 가깝고, 밝으면 65535에 가까운 값이 나옵니다. |
| 10 | `raw_value / 65535 * 100` | 0~65535 범위를 0~100% 범위로 변환합니다. 사람이 이해하기 쉬운 퍼센트 값으로 바꾸는 과정입니다. |
| 12 | `print(brightness)` | 숫자만 출력하면 Thonny Plotter가 이 값을 자동으로 그래프에 표시합니다. |

### 도전 코드: 밝기 판정 + 내장 LED 연동

기본 코드를 바탕으로, 밝기에 따라 판정하고 어두우면 내장 LED를 켜봅시다.

```python
from machine import ADC, Pin
import time

# 아날로그 입력 설정
light = ADC(Pin(26))

# Pico 내장 LED 설정
led = Pin("LED", Pin.OUT)

# 밝기 판정 + LED 연동
while True:
    raw_value = light.read_u16()
    brightness = round(raw_value / 65535 * 100, 1)

    if brightness >= 60:
        status = "밝음"
        led.off()   # 밝으면 LED 끔
    elif brightness >= 30:
        status = "보통"
        led.off()
    else:
        status = "어두움"
        led.on()    # 어두우면 LED 켬

    print(f"밝기: {brightness}% → {status}")
    print(brightness)  # Plotter용
    time.sleep(2)
```

:::note[코드 읽어보기]
`Pin("LED", Pin.OUT)`은 Pico에 내장된 초록색 LED를 제어하는 코드입니다. `led.on()`과 `led.off()`로 켜고 끌 수 있습니다. `if-elif-else` 구조로 세 가지 경우를 나눕니다. 밝기 기준값(60, 30)을 여러분 교실에 맞게 조절해보세요.
:::

## 잠깐, 방금 무슨 일이 일어난 걸까요?

여러분이 방금 한 일을 다시 생각해보세요.

교실의 빛 에너지가
**→** 포토레지스터(빛에 반응하는 저항)의 **전기 저항**을 바꾸고
**→** 저항 변화가 **전압 변화**로 이어지고
**→** ADC가 그 아날로그 전압을 0~65535 사이의 **디지털 숫자**로 바꾸고
**→** 코드가 그 숫자를 **퍼센트**로 변환하고
**→** Plotter가 그 퍼센트를 **실시간 그래프**로 그렸습니다.

"밝다", "어둡다"라는 주관적인 느낌이 객관적인 숫자가 되고, 그 숫자가 눈에 보이는 그래프가 되었습니다. 스마트폰의 자동 밝기 조절, 가로등의 자동 점등, 카메라의 자동 노출 — 모두 이 원리로 동작합니다. 여러분은 방금 그 첫 단계를 직접 구현한 것입니다.
