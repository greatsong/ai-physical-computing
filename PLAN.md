# "AI 피지컬 컴퓨팅" 인터랙티브 웹 교재 프로젝트 계획서

> 마지막 업데이트: 2026-03-06

## 프로젝트 개요

센서+바이브코딩+대시보드+머신러닝 교육 커리큘럼 "AI 피지컬 컴퓨팅"의 인터랙티브 웹 교재.
GitHub Pages로 배포되는 정적 사이트.

- **대상**: 중고등학생 (초심자 대상, 표현은 성숙하게)
- **하드웨어**: Raspberry Pi Pico 2 WH + Grove Shield + Grove 센서 32종
- **핵심 가치**: 바이브 코딩으로 진입 장벽 낮추기, 대시보드로 성취감, ML로 확장

## 바이브 코딩 교육 전략

### 단계적 전환
1. **1막 (1~4차시)**: 직접 코딩으로 기초 체득 (변수, 반복문, 조건문, 파일 I/O)
2. **2막 (5~10차시)**: 바이브 코딩 도입
   - 5차시: 바이브 코딩이란? AI에게 프롬프트로 코드 요청하는 법
   - 6~8차시: 점점 복잡한 시스템을 바이브 코딩으로 제작
   - 학생 역할: 프롬프트 작성 → AI 코드 검토 → 수정 → 테스트
3. **3막 (11~15차시)**: 바이브 코딩 + ML, 대시보드 설계

### 바이브 코딩 도구
- **추천**: Claude, ChatGPT 등 AI 챗봇
- **프롬프트 패턴**: "Grove [센서명]을 Pico 2 WH의 [포트]에 연결했어. [원하는 기능]하는 MicroPython 코드 만들어줘"
- **학습 포인트**: AI가 생성한 코드를 읽고 이해하기, 오류 수정하기, 요구사항 변경하기

## 대시보드 활용 전략

### 대시보드 유형
1. **OLED 대시보드** (4차시~): SSD1306 128x64에 센서값 실시간 표시
2. **Thonny Plotter 대시보드** (2차시~): 실시간 그래프
3. **웹 대시보드** (9차시~): PC Python + matplotlib/plotly로 CSV 시각화
4. **종합 대시보드** (12차시): 여러 센서 데이터를 한 화면에

### 대시보드 학습 흐름
```
2차시: Thonny Plotter (가장 간단한 그래프)
  ↓
4차시: OLED에 숫자 표시 (첫 하드웨어 대시보드)
  ↓
6차시: OLED에 바 그래프 + 아이콘 (바이브 코딩으로 제작)
  ↓
9차시: PC 웹 대시보드 (matplotlib)
  ↓
12차시: 종합 실시간 대시보드 (OLED + 웹)
```

## 머신러닝 범위

### 3막에서 다루는 ML
1. **Teachable Machine** (Google): 브라우저에서 센서 데이터로 분류 모델 학습
   - 소리 분류: 박수 vs 말소리 vs 노크
   - 동작 분류: 가속도 센서로 걷기 vs 뛰기 vs 정지
2. **Python scikit-learn** (간단한 소개): CSV 데이터로 분류/회귀
3. **Edge Impulse** (선택): 임베디드 ML (Pico에서 직접 추론)

## 차시 페이지 8섹션 템플릿

모든 차시 페이지는 아래 8섹션으로 구성:

1. **헤더**: 차시 번호, 제목, "오늘의 질문", 소요시간, 센서, 난이도
2. **준비물**: 부품 카드 + 연결 다이어그램
3. **단계별 활동**: Step별 시간 배분, 팁/생각해보기
4. **코드 예시**: 기본/도전 탭, 줄별 해설 (2막부터는 "바이브 코딩 프롬프트" 탭 추가)
5. **와우 모먼트**: 핵심 개념의 흐름 다이어그램
6. **대시보드 활동**: 해당 차시의 대시보드 과제
7. **마무리**: 오늘 배운 것 + 다음 차시 미리보기
8. **체크리스트**: 달성도 체크
